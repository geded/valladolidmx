# Valladolid.mx Platform Blueprint

# 12C.0 -- Art Direction, Hero & Visual Experience

**Versión:** 1.0\
**Estado:** Oficial\
**Depende de:** 12, 12A, 12B

## Objetivo

Este documento define la Dirección de Arte oficial de la experiencia
inicial de Valladolid.mx. No modifica arquitectura, componentes ni
lógica de negocio.

## Principio Rector

**La portada debe inspirar antes de vender.**

El visitante debe sentir que llegó a un destino extraordinario antes de
comenzar a explorar.

## Filosofía Visual

-   El destino es el protagonista.
-   La plataforma acompaña.
-   La fotografía comunica más que los textos.
-   Menos elementos, mayor impacto.

## Hero

### H1 Oficial

**Despierta en Valladolid y descubre el Oriente Maya de Yucatán.**

Reducir ligeramente el tamaño del H1 para mantener una composición
elegante.

### Eslogan

**Experiencias que emocionan**

Debe actuar como firma editorial y no competir con el H1.

### Fotografías

Usar exclusivamente activos de:

`docs/brand-assets/photography/hero`

Si existen varias fotografías oficiales, utilizar un carrusel con
transición suave (6--8 segundos por imagen).

### Overlay

Aplicar un degradado oscuro suave para garantizar la legibilidad de
textos y botones.

## Header

-   Transparente al cargar.
-   Efecto glass y sombra ligera al hacer scroll.
-   El logotipo oficial siempre debe ser legible.

## Botones

### Principal

Explorar Destinos.

### Secundario

Arma tu Viaje.

El principal debe tener mayor jerarquía visual.

## Buscador

Debe acompañar la experiencia, nunca dominar el Hero.

## Uso obligatorio de Brand Assets

Todos los recursos visuales deberán provenir de:

-   docs/brand-assets/logos
-   docs/brand-assets/colors
-   docs/brand-assets/icons
-   docs/brand-assets/photography
-   docs/brand-assets/fonts

Queda prohibido crear activos alternativos.

## Mobile First

Diseñar primero para teléfonos móviles. Desktop será una adaptación
enriquecida.

## Responsive

Mantener la jerarquía visual, el contraste y la composición fotográfica
en móvil, tablet y escritorio.

## Accesibilidad

-   Contraste WCAG AA.
-   Botones mínimos de 44 px.
-   Tipografía legible.

## Errores prohibidos

-   Inventar logotipos.
-   Inventar fotografías.
-   Sobrecargar el Hero.
-   Dar más protagonismo al buscador que al destino.

## Criterios de aceptación

-   Hero inspirador.
-   H1 claro.
-   Fotografías oficiales.
-   Header elegante.
-   CTAs con jerarquía.
-   Experiencia Mobile First.
-   Identidad visual proveniente exclusivamente de `docs/brand-assets`.

**Fin del Documento 12C.0**
