# Valladolid.mx Platform Blueprint

# 12C.2 -- Responsive, Mobile First, Visual Performance & Final Acceptance

**Versión:** 1.0\
**Estado:** Oficial\
**Depende de:** 12, 12A, 12B, 12C.0 y 12C.1

## Objetivo

Definir las reglas oficiales para responsive, Mobile First, rendimiento
visual y criterios finales de aceptación de la Fase 0.

## Mobile First

Toda decisión de diseño deberá comenzar en móvil y posteriormente
escalar a tablet y escritorio.

### Resoluciones objetivo

-   390--430 px (iPhone)
-   768--834 px (tablet)
-   1024--1440 px (laptop)
-   1440 px en adelante (desktop)

## Hero Responsive

-   Mantener el sujeto principal visible.
-   Evitar recortes que oculten monumentos o elementos clave.
-   Reencuadrar imágenes cuando sea necesario.
-   El H1 deberá conservar buena legibilidad en todas las resoluciones.

## Imágenes

-   Utilizar únicamente activos de `docs/brand-assets/photography`.
-   Preparar soporte para formatos modernos (WebP/AVIF) cuando existan.
-   Lazy loading para contenido bajo el primer pliegue.

## Performance Visual

Objetivos:

-   LCP \< 2.5 s
-   CLS \< 0.1
-   INP dentro de buenas prácticas
-   Lighthouse:
    -   Performance ≥ 90
    -   Accessibility ≥ 95
    -   Best Practices ≥ 95
    -   SEO ≥ 95

## Motion

-   Transiciones suaves (150--250 ms).
-   Carrusel Hero: 6--8 s por imagen.
-   Sin animaciones que distraigan.

## Accesibilidad

-   Contraste WCAG AA o superior.
-   Navegación mediante teclado.
-   Focus visible.
-   Objetivos táctiles mínimos de 44 px.

## Consistencia Visual

Todo el proyecto deberá utilizar:

-   Design System oficial.
-   Paleta aprobada.
-   Activos de `docs/brand-assets`.
-   Componentes reutilizables.

## Criterios Finales de Aceptación

La Fase 0 se considerará concluida cuando:

-   La identidad visual respete los documentos 12, 12A, 12B, 12C.0 y
    12C.1.
-   El Hero transmita la identidad del Oriente Maya de Yucatán.
-   El sitio sea Mobile First.
-   La navegación sea clara y consistente.
-   Los activos oficiales provengan exclusivamente de
    `docs/brand-assets`.
-   No existan elementos gráficos creados por interpretación.
