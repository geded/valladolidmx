## 1. Alcance entregado
- **Server function** `resolveTravelerDestinationContext` (`src/lib/traveler/destination-context.functions.ts`): registra idempotentemente los Contributors iniciales (weather / hours / traffic) y devuelve `ResolvedDestinationContext` serializable (`{ signals, resolvedAt, contributors }`).
- **Weather efectivo** vía `weatherContributor` (Open-Meteo, server-only) — publicado como señal `weather` con `explain.rationale` y TTL 15 min.
- **Decision Center Contributors nuevos** (`src/lib/traveler/decision-center-destination.ts`), independientes entre sí, consumen ÚNICAMENTE `ResolvedDestinationContext`:
  - `destination.weather` → warning "Prepárate: probable lluvia" cuando `rainChanceNext6h ≥ 60`.
  - `destination.hours` → warning "Cierra pronto" / critical "Está cerrado ahora" con acciones.
  - `destination.traffic` → info/warning "Sal antes" para traslados ≥ 15 min.
- **NowNextLaterSurface** (`src/components/traveler/NowNextLaterSurface.tsx`) resuelve contexto vía TanStack Query (server fn), inyecta `destination` en `deriveDecisionCenter` y compone contributors: `[...CV6_2_BASE_CONTRIBUTORS, ...CV6_5_DESTINATION_CONTRIBUTORS]`.

## 3. Founder Experience First — declaración obligatoria
- **Cuándo aporta valor:** durante fase `onsite`, cuando alguna señal del destino sugiere acción (lluvia probable, cierre inminente, traslado largo).
- **Auto-ocultamiento:** sin señales relevantes → contributors devuelven `[]`; sin tarjetas → `NowNextLaterSurface` no renderiza (`center.empty`). Fuera de `onsite`, la resolución no se ejecuta (`enabled: false`).
- **Acción concreta:** cada tarjeta prioriza acción (`Iniciar navegación`, `Ver alternativas`) o comunica preparación explícita.
