
# 13.4 – CMS & Content Operations
**Versión:** 1.0  
**Estado:** Aprobado para ejecución  
**Blueprint:** Valladolid.mx v2.0

# 1. Propósito

Este documento gobierna la implementación del CMS de Valladolid.mx.
El CMS es el núcleo operativo de la plataforma. Toda creación, edición,
publicación y retiro de contenido deberá pasar por este modelo.

Lovable implementa este documento; no redefine su comportamiento.

# 2. Principios Rectores

1. El CMS es Headless y API First.
2. Todo contenido es reutilizable.
3. Todo contenido es versionable.
4. Todo contenido soporta múltiples idiomas.
5. Toda publicación deja auditoría.
6. Ningún contenido se publica sin flujo editorial.
7. El Blueprint es la única fuente de verdad.

# 3. Capacidades obligatorias

El CMS deberá administrar:

- Países
- Estados
- Regiones
- Destinos
- Zonas
- Puntos de interés
- Empresas
- Productos / Experiencias
- Eventos
- Rutas
- Artículos
- Páginas
- FAQ
- Fotografías
- Videos
- Documentos
- Promociones
- Banners
- SEO
- Traducciones

# 4. Modelo editorial

Estados obligatorios:

Draft
↓
In Review
↓
Approved
↓
Published
↓
Archived

No se permitirá publicar contenido omitiendo etapas.

# 5. Roles editoriales

Administrador
Editor
Revisor
Autor
Colaborador

Los permisos provienen del documento 13.3.

# 6. Multimedia

El CMS administrará:

- Logos
- Hero Images
- Galerías
- Fotografías
- Videos
- PDFs

Cada archivo almacenará:

- UUID
- propietario
- organización
- destino
- idioma
- fecha
- estado

# 7. Traducciones

Todo contenido soportará:

- idioma principal
- traducciones
- fallback
- SEO independiente por idioma

# 8. SEO

Cada entidad deberá administrar:

- slug
- title
- meta description
- Open Graph
- Canonical
- Indexación

# 9. Integraciones

El CMS alimentará:

- Home
- Destinos
- Empresas
- Marketplace
- Alux IA
- Arma tu Viaje
- APIs públicas

No existirá contenido duplicado fuera del CMS.

# 10. Auditoría

Registrar:

- creación
- modificación
- publicación
- despublicación
- eliminación
- usuario responsable

# 11. Restricciones

- No hardcodear contenido.
- No almacenar textos en componentes.
- Toda información pública proviene del CMS.
- Ningún módulo implementará su propio gestor de contenido.

# 12. Entregables

- CMS operativo
- Panel editorial
- Gestión multimedia
- Flujo editorial
- Versionado
- Traducciones
- Evidencias QA
- Documentación

# 13. Definition of Done

El bloque concluye cuando:

- Todo contenido público proviene del CMS.
- Los flujos editoriales funcionan.
- Los permisos respetan 13.3.
- SEO operativo.
- Traducciones operativas.
- QA aprobado.
