## 2. Diseño del Preloader

`preloadShadowAssetBundles(ids[])`:

1. Deduplica `ids` y filtra por `SHADOW_ALLOWLIST` **antes** de tocar la DB.
   Cualquier id ajeno se descarta silenciosamente ⇒ 0 queries.
2. Ejecuta **dos** queries batched como máximo:
   - `media_assets.select("id, original_width, original_height, pipeline_status, original_checksum").in("id", ids)`
   - `media_asset_variants.select("asset_id, format, width, height, bucket, path, variant_key, usage_context").in("asset_id", ids).eq("is_current", true).eq("status", "ready")`
3. Envuelve cada query con `withTimeout(1500ms)` — timeout duro.
4. Materializa `has_original_checksum: boolean` y descarta el valor crudo
   del checksum antes de exponer el bundle. Ningún consumidor lo lee.
5. Ante `timeout`/`error` devuelve `{assets: {}, error}` y el evaluador cae a
   `fallback_reason='storage_unreachable'`; jamás afecta el render.

**Garantía anti N+1**: los tests V9 verifican que `N` ids elegibles se
resuelven en exactamente 2 queries (una por tabla), y que 0 ids elegibles
no toca la DB.

## 3. Integración con el evaluador

`evaluateMediaSourceShadow(asset, {preloaded, preloadTelemetry, …}, ctx)`:

- Si hay bundle → **no** toca DB; usa variantes en memoria.
- Preflights: `asset.id` distinto → `no_variants_for_context`; checksum
  faltante → `variant_key_missing`; `preloadTelemetry.error` presente →
  `storage_unreachable`.
- **Parity check con el resolver puro**: tras elegir variante, se corre
  `resolveMediaSource()` con las mismas variantes (URLs opacas `opaque://vk`)
  y se verifica que también habría elegido pipeline. Si difiere, la decisión
  cae a legacy con `fallback_reason='no_variants_for_context'`.
- Emite el evento sanitizado extendido con:
  `preload_latency_ms`, `preload_query_count`, `preload_error`.

## 6. Evidencia de cero exposición al cliente

- El módulo `shadow-preloader.server.ts` termina en `.server.ts` (blocked por el import graph — ver `tanstack-supabase-import-graph`).
- El evaluador se importa **dinámicamente** dentro del route handler
  `/api/dev/media-shadow-eval` (`await import(...)`), fuera del bundle cliente.
- El endpoint devuelve exclusivamente telemetría (decisión, latencias,
  banderas booleanas). Test F verifica que ninguna URL/token/bucket/path
  aparece en el shape del `ShadowDecision`.
- El preloader NO expone `original_checksum` crudo — sólo el booleano
  `has_original_checksum` (test V10/V11).
- Ningún componente renderizador (`<img>`, `<picture>`) llama al evaluador
  ni al preloader. `grep` confirma cero call sites fuera de `scripts/` y
  `src/routes/api/dev/`.
