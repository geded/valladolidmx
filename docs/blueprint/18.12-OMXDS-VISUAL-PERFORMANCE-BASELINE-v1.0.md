## 1. Método

### 1.1 Herramientas

- `curl -sSL` con `-w` para HTTP status, `num_redirects`, `time_starttransfer` (TTFB), `time_total`, `size_download`, `content-type`, `cache-control`.
- Playwright Chromium headless (perfil desktop y móvil emulado por viewport, sin throttling de red/CPU).
- `PerformanceObserver` en el runtime del navegador para Web Vitals de laboratorio (LCP, FCP, CLS, Long Tasks → TBT aproximado) sobre una sola carga.
- Recolector de recursos vía `performance.getEntriesByType('resource')` para pesos, conteos y hosts terceros.

- **Diagnóstico HTTP** sobre las **17 superficies** S01–S17 en el perfil de red del sandbox.

Ejecutar Web Vitals sobre las 17 × 2 = 34 combinaciones se descartó por producir datos comparables sólo con multitud de trials y throttling reproducible (fuera de alcance V0).

**Riesgos observables sin activar el pipeline:**

- Ausencia global de variantes WebP/AVIF en los assets críticos auditados.

- Riesgo de layout shift bajo en la muestra (CLS máx 0.069) pero **elevado en potencia** cuando la galería ZT reemplace stock por assets reales sin dimensiones declaradas.
- Recursos rotos: no observados en muestra.

## 7. Cross-cutting observations

### 7.1 Servidor

- SSR consistentemente devuelve HTML en `<110 KB`, pero TTFB medio 1.7 s en el sandbox. Coincide con arquitectura Cloudflare Worker + Supabase Data API sin caché CDN.

### 7.2 Caché

- HTML: `Cache-Control: no-cache, must-revalidate, max-age=0` en todas las 17 superficies (confirma SE-05 e informa **PF-03**).
- Imágenes signed URLs: expiran (`sign?token=…`) → invalidación forzada.

### 7.3 Compresión

- `Content-Encoding` observado como `br`/`gzip` en HTML (peso HTML pequeño).
- No hay pruebas de que las imágenes JPEG estén siendo transformadas por CDN.

### 7.4 Preload / Preconnect

- No se observa `<link rel="preload" as="image">` para el LCP en ninguna superficie con hero image.
- No se observa `<link rel="preconnect">` a `qjlfwsqsrzikjilicoad.supabase.co` (host crítico) ni a `maps.googleapis.com`. **PF-09**.

### 7.5 Code splitting e hidratación

- Solicitudes por página elevadas (91–183). Consistente con chunk splitting granular de TanStack Start.
- Hidratación no medida como INP; no hay long tasks > 50 ms en lab sin throttling.
- **PF-07**: número de solicitudes por encima del presupuesto en todas las páginas del árbol territorial. Recomendación futura: consolidar chunks compartidos.

### 7.6 Cargas duplicadas

- No se detectaron duplicados obvios en la muestra.

### 7.7 Errores de red

- Cero HTTP 4xx/5xx en la muestra.

### 7.8 Offline observable / PWA

- No se registró Service Worker propio para app-shell en la sesión de laboratorio (consistente con las guardas de PWA para `beta.lovable.dev` / previews).
- El worker de mensajería para Web Push existe en repositorio (`public/push-sw.js`) pero no se activó offline por la sesión sandbox.
- Comportamiento offline no auditable como “app-shell offline” en V0.

### 7.9 Diferencias móvil vs escritorio

- Escritorio muestra galerías completas en el primer paint (más peso), móvil las difiere (mejor peso, peor riesgo de CLS cuando carguen).
- Categorías (S04) tienen el mapa above-the-fold en desktop, provocando LCP = tile de Google.
