# 15.10.6.4 — Push Notifications (infraestructura)

Estado: **INFRAESTRUCTURA OFICIAL — SIN ADOPCIÓN AUTOMÁTICA**

## Alcance
- Service Worker dedicado `public/push-sw.js` en scope acotado
  `/push/`, aislado del SW de app-shell (`/sw.js`, scope `/`) para no
  colisionar con vite-plugin-pwa.
- API cliente `src/pwa/push.ts`:
  - `getPushSupport()` / `getPermissionStatus()`
  - `registerPushServiceWorker()` / `unregisterPushServiceWorker()`
  - `requestPushPermission()`
  - `subscribePush()` / `unsubscribePush()` / `getCurrentPushSubscription()`
- VAPID public key opcional vía `VITE_VAPID_PUBLIC_KEY`. Sin key, la
  suscripción devuelve `null` y emite `pwa:push:error` con
  `missing-vapid-key`. Comportamiento intencional: nunca suscribir a
  ciegas.

## Gobernanza aplicada
- **Cero adopción automática**: ningún componente invoca
  `subscribePush()` sin acción explícita del usuario. Esta adenda no
  integra push en ninguna superficie funcional.
- **Denylist absoluta**: el payload de las notificaciones nunca debe
  incluir tokens, sesiones, roles, permissions, billing ni pagos.
  `push-sw.js` sólo lee `title`, `body`, `icon`, `badge`, `tag`, `url`.
- **Kill Switch**: `?sw=off`, iframe, dev y preview Lovable bloquean
  registro y suscripción. `unregisterPushServiceWorker()` +
  `unsubscribePush()` revierten estado completo.
- **Observability First**: eventos `pwa:push:*` (`skipped`,
  `sw_registered`, `sw_unregistered`, `permission_requested`,
  `permission_granted`, `permission_denied`, `subscribed`,
  `unsubscribed`, `error`).
- **Reversibilidad**: el usuario puede revocar en cualquier momento;
  `notificationclick` reutiliza pestaña abierta cuando existe.
- **Compatibilidad Preview Lovable**: guardas idénticas a
  `register-sw.ts` (dev / iframe / preview hostname / `?sw=off`).
- **Compatibilidad Sync Queue (15.10.6.3)**: eventos `pwa:sync:*` y
  `pwa:push:*` viven en namespaces separados y no interfieren.

## Compatibilidad con Discovery y Workspace
- Discovery Layer: no monta banner alguno; UI de suscripción se
  añadirá superficie por superficie en sub-adendas posteriores.
- Workspace Engine: la API es transversal; cualquier módulo puede
  invocarla desde un handler de usuario autorizado.

## Rollback
1. Eliminar `public/push-sw.js` y `src/pwa/push.ts`.
2. Publicar release: navegadores dejarán de recibir push al no
   encontrar SW; existente puede limpiarse llamando
   `unregisterPushServiceWorker()` desde un release intermedio.
3. `?sw=off` continúa siendo kill-switch inmediato del stack PWA.
