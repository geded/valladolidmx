# 14.40.X — Payment Lifecycle (Backlog Arquitectónico)

**Tipo:** Lineamiento arquitectónico — no se implementa en Etapa 5.
**Origen:** Aprobación formal del Reporte 14.40.5.
**Reutiliza:** `payment_events`, `order_events`, `PaymentProvider`.

## Objetivo

Definir el ciclo completo de vida del pago dentro del Marketplace y
evolucionarlo de forma incremental sobre la infraestructura ya aprobada
en la Etapa 5, sin modificar la capa de abstracción `PaymentProvider`.

## Estados del ciclo

```text
intent_created → processing → succeeded ─┬─► consumed
                            │             ├─► refunded (total | parcial)
                            │             ├─► disputed → resolved | lost
                            │             └─► no_show
                            └─► failed | expired | cancelled
```

## Eventos mínimos a soportar

- `payment_intent_created`
- `payment_succeeded`
- `payment_failed`
- `payment_cancelled`
- `payment_expired`
- `payment_refunded` (total y parcial)
- `payment_disputed` / `dispute_resolved` / `dispute_lost`
- `experience_consumed` (consumo del bien/servicio adquirido)
- `booking_no_show`
- `reconciliation_matched` / `reconciliation_discrepancy`

## Extensiones esperadas al contrato `PaymentProvider`

- `refund(intentId, amount?, reason?)`
- `capture(intentId, amount?)` (para flujos auth + capture)
- `cancel(intentId)`
- `retrieve(intentId)` (verificación bajo demanda / conciliación)

Cada extensión debe permanecer agnóstica del proveedor; Stripe,
Paddle, Mercado Pago y PayPal implementarán la misma interfaz.

## Auditoría y conciliación

- Toda transición de estado se registra en `order_events` con `payload`
  normalizado.
- `payment_events` mantiene la huella cruda del proveedor (idempotente
  por `provider_event_id`).
- Job de conciliación periódico compara `orders.payment_status` vs.
  estado remoto del proveedor; discrepancias generan
  `reconciliation_discrepancy` para revisión en `/cms/pagos`.

## Fuera de alcance de Etapa 5

Esta especificación no introduce código en la Etapa 5. Se planifica para
olas posteriores y se referencia desde el Plan 14.40.
