## 1. Propósito

Definir criterios verificables de responsive, accesibilidad y consistencia entre superficies sin fijar breakpoints finales ni valores definitivos (fuera de contratos técnicos vigentes).

## 2. Reglas responsive

Viewports canónicos de referencia (RV0.1): mobile 390 px, desktop 1440 px. Breakpoints finales se definen en RV0.3.

### 2.2 Límites vinculantes

- Barra inferior en mobile: **máximo 3 acciones** (VP-03).
- Nivel-1 de tarjeta: **máximo 5 bloques** en mobile, 6 en desktop.
- Insignias primarias visibles simultáneamente en mobile: **1** (VP-06).
- Sticky CTA: **uno** por superficie.

## 4. Accesibilidad

| ID | Criterio | Umbral |
|---|---|---|
| AC-01 | Contraste texto | WCAG AA (4.5:1 texto normal, 3:1 texto grande) |
| AC-02 | Tamaño táctil | ≥44 × 44 px |
| AC-03 | Foco visible | Anillo perceptible con contraste AA |
| AC-04 | Navegación teclado | Toda acción accesible sin puntero |
| AC-05 | Lector de pantalla | Labels y roles semánticos verificables |
| AC-06 | Etiquetas de iconos | Icono + etiqueta o `aria-label` claro para acciones críticas |
| AC-07 | Estados perceptibles | Loading, éxito, error, offline con texto y visual |
| AC-08 | Mensajes de error | Comprensibles, con acción sugerida |
| AC-09 | Contenido alternativo | `alt` semántico obligatorio en fotografía funcional |
| AC-10 | Movimiento reducido | Respetar `prefers-reduced-motion` |
| AC-11 | Jerarquía semántica | H1 único por página; landmarks correctos |
| AC-12 | Lenguaje comprensible | 6 idiomas oficiales; sin jerga |
| AC-13 | Icono decorativo vs funcional | Decorativos marcados `aria-hidden` |
| AC-14 | Comprensión intercultural | Símbolos neutros o explicados |

**Regla vinculante:** los iconos solos no sostienen acciones críticas (VP-10 + AC-06).

## 5. Consistencia sin uniformidad

- Mismas reglas (VP + taxonomías) producen resultados coherentes, no idénticos.
- Prohibido “un componente para todo”.
- Prohibido borrar diferencias Premium / estándar / onsite bajo una monoquilla visual.
