# H3·A4 · M1 · First-Vertical Derivation — Completion Report v1.0

**Estado:** Cerrado técnicamente · pendiente validación funcional
**Ola:** H3 · A4 · M1
**Motor:** `sharp` self-hosted (decisión M0)
**Feature flag:** `media_pipeline_enabled = false` (permanece OFF)
**Superficies públicas afectadas:** ninguna

## 1. Alcance entregado

M1 implementa la **primera vertical productiva controlada** del pipeline
de imágenes: derivación real con `sharp` self-hosted ejecutada **fuera
del Worker** (Cloudflare Workers no ejecuta binarios nativos —
restricción arquitectónica confirmada en M0), publicación de variantes
en el bucket privado `media-derived` y persistencia trazable en
`media_asset_variants`.

**Componentes:**

- `scripts/media-pipeline/derive.mjs` — CLI de derivación por asset.
- `scripts/media-pipeline/README.md` — operación, matriz y rollback.
- `scripts/media-pipeline/out/{runId}.json` — reporte por corrida.

La infraestructura M0 (`media_asset_variants`, `media-derived`, trigger
de inmutabilidad, `resolveMediaSource`, `isMediaPipelineEnabled`) se
reutiliza sin modificaciones.

## 2. Cumplimiento de reglas vinculantes

| # | Regla | Evidencia |
|---|-------|-----------|
| 1 | Original inmutable | El CLI sólo hace `download` de `media-original`; el trigger `media_assets_protect_original` sigue activo. |
| 2 | Derivación reproducible | Cada variante persiste `engine`, `engine_version` (`sharp@<ver>`), `format`, `width`, `quality`, `source_checksum`, `content_hash`, `generated_at`, `processing_ms`. |
| 3 | Idempotencia | Match por `(asset_id, format, width, engine)` + comparación de `metadata.engine_version + quality + source_checksum`. Repetición ⇒ `skipped-idempotent`, 0 bytes de egress. |
| 4 | Contrato público estable | `media_pipeline_enabled = false`. `resolveMediaSource()` devuelve `file_url` legacy. Cero cambios en componentes. |
| 5 | Fallback obligatorio | Errores por variante ⇒ `status='failed'` + `pipeline_status='failed'`. El resolver sirve el legacy (`usedPipeline=false`). |
| 6 | Activación controlada | El CLI exige `--asset-id` o `--file`. Sin `--all`, sin cron, sin backfill. |
| 7 | Observabilidad | Reporte JSON completo por corrida + columnas DB (`bytes`, `checksum`, `processing_ms`, `error`, `metadata`). |

## 3. Outcome Validation (Founder Product Vision)

- **Fidelidad visual preservada** — calidades por contexto derivan de la
  adenda M0 (SSIM ≥ 0.985 en hero/gallery, ≥ 0.975 en card/thumbnail).
  Fotografías decisivas (hero de destino/hotel/experiencia) usan Q≥50
  AVIF / Q≥75 WebP / Q≥82 JPEG.
- **Velocidad de descubrimiento** — cuando el flag se active para el
  asset piloto, `srcSet` moderno reduce ~57% vs JPEG (medido M0),
  mejorando LCP móvil sin regresión perceptual.
- **Cero riesgo comercial hoy** — flag OFF garantiza que ningún
  visitante ve una imagen distinta a la actual mientras se valida.

## 5. Benchmark contra M0

El CLI reproduce los parámetros del harness M0 (mismo motor `sharp`,
mismas calidades, mismos anchos por contexto), por lo que las
reducciones y tiempos por variante deben caer dentro del ±10% de la
adenda `18.H3-A4-M0-ADDENDUM-BENCHMARK-RESULTS-v1.0.md`. El reporte
JSON de la primera corrida piloto se archivará junto con esta adenda.

## 6. Prueba controlada (protocolo)

1. Elegir 1–3 asset IDs piloto (recomendado: 1 hero, 1 card, 1 gallery).
2. Poblar `media_assets.original_bucket` y `original_path` para esos
   IDs (si el asset es legacy y sólo tiene `file_url`, el CLI usa el
   `file_url` como fallback de lectura, sin escribir en el original).
3. `node scripts/media-pipeline/derive.mjs --asset-id=<id> --dry-run`
   → validar plan de variantes sin escribir.
4. Repetir sin `--dry-run` → variantes reales en `media-derived`.
5. Repetir la misma corrida → todas las variantes `skipped-idempotent`.
6. Inspeccionar `media_asset_variants` y el reporte JSON.
7. Verificar que el sitio público sigue sirviendo la URL legacy
   (flag OFF).
8. **NO activar el flag** hasta autorización explícita del Founder.

## 7. Rollback

Documentado en `scripts/media-pipeline/README.md` §Rollback:
`UPDATE media_assets SET pipeline_status='disabled'` +
`DELETE FROM media_asset_variants WHERE asset_id=...`. Los binarios en
`media-derived` pueden purgarse sin impacto (no son URLs canónicas —
Founder Stable Public Asset Contract).
