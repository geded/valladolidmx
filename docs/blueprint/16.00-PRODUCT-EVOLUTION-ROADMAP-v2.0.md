## Reglas Oficiales de Gobernanza

- **Single Roadmap Principle** — Este documento es la única fuente oficial de planificación. Prohibido generar roadmaps independientes; toda nueva épica/programa/versión/iniciativa se incorpora aquí mediante actualización incremental.
- **Priorización por Roadmap** — Al cerrar una épica, la siguiente se determina exclusivamente contra este Roadmap declarando: Programa Estratégico · Carril · Versión de producto · Dependencias resueltas · Dependencias pendientes · Valor al producto · Justificación vs alternativas. Ninguna épica fuera del Roadmap sin autorización expresa del Founder.
- **Two-Rails** — Carril A · Producto (prioridad principal, acerca a la versión comercial). Carril B · Hardening (mejora continua, no bloquea). Todo lo del Carril A pasa por Carril B antes de considerarse terminado al nivel del Design System.
- **Product Changelog obligatorio** — Cada épica cerrada registra entrada al final de este documento con: versión · programa · carril · fecha · impacto funcional · Founder Decisions incorporadas · % actualizado del Roadmap.
- **Forma de trabajo** — Una épica → una historia → una ola → una sub-ola a la vez, con Reconciliation Report → aprobación Founder → implementación → typecheck → build → smoke → auditoría no-regresión → Completion Report → actualización de Changelog + Roadmap.
- **Freeze de infraestructura** — Prioridad: ensamblar el producto comercial sobre la infraestructura construida. No se abren nuevas líneas de infraestructura salvo bloqueo crítico.

> Este roadmap consolida la evolución real del producto desde la Fase 1 (Serie 13) hasta el cierre de la Ola de Navegación N3 (Serie 15.11). Reorganiza el trabajo en **Programas Estratégicos** y en **Versiones de Producto**, incorpora las **Founder Decisions** aprobadas durante la construcción y define la ruta hasta **Beta** y **Comercial**.

## FASE 2 · Reconciliación (Original vs Actual)

### Lo que permanece
- Visión de plataforma: Destino → Categoría → Empresa → Producto.
- Marketplace, Portal Empresarial, CMS, Concierge, Alux.
- Fase 1 (Serie 13) como marco normativo.

### Lo que cambió
- **CMS** dejó de ser un editor de campos y se unificó en **Experience Builder** (Studio único, WYSIWYG por defecto).
- **Navegación** pasó de rutas ad-hoc a **Navigation Contract v1.0** con resolver canónico y sessionStorage.
- **Superficies autenticadas** dejaron de tener layouts propios y viven en **Workspace Engine v2**.
- **Superficies públicas** se unificaron en **Discovery Layer** con PublicShell.
- **Diseño** pasó de "estilo genérico" a **Valladolid Colonial Design System v1.0** (migración progresiva M1..M10).
- **Alux** dejó de ser un asistente aislado y es capa transversal con contrato explicable (rationale/sources/effect/reversibility).

### Lo que nació durante la construcción
- Experience Builder unificado + Biblioteca Oficial + Plantillas Madre.
- Context Engine + Discovery Navigator.
- Navigation Blueprint / Contract / Intelligence.
- Design System Colonial + Institutional Badges (registry).
- Trust Engine (declarado, pendiente de construcción).
- Carril A (Producto) / Carril B (Hardening).
- Two-Rails Roadmap Policy.
- Single Studio Principle.
- Product First Validation Policy.

### Lo que desapareció / se absorbió
- Editores paralelos de páginas (unificados en Studio único).
- Sistemas de layout específicos por módulo (absorbidos por Workspace + Discovery).
- Rutas legacy independientes (siguen como shims; se retirarán en Fase 2 de 301, no autorizada aún).
- Terminología "Marketplace" en superficies públicas (historia futura, ver backlog).

## FASE 3 · Programas Estratégicos

Se adoptan **8 Programas** (siete originales + Trust como programa propio por su peso transversal).

### Programa A · Experiencia del Visitante
Discovery Layer, superficies públicas (Home, Destino, Categoría, Empresa, Producto), Marketplace visitante, Arma tu Viaje, Perfil del Viajero, Related, Recomendaciones contextuales, PWA offline, Push visitante.

### Programa B · Experiencia del Empresario
Portal Empresarial, onboarding multiempresa, gestión de fichas, medios, promociones, reservas, ventas, comisiones, notificaciones de negocio, panel de rendimiento.

### Programa C · Experiencia Editorial (Staff)
Experience Builder Studio (Visual + Profesional), Biblioteca Oficial, Plantillas Madre, Versionado + Preview + Publish + Restore, Redirects, Sitemaps, i18n editorial, Landings/Campañas.

### Programa D · Administración
Founder Cockpit, Roles + Zone Scopes, Moderación, Observabilidad, Exportes, Auditoría, Panel Alux operativo, Health del ecosistema.

### Programa E · Alux IA
Alux Traveler, Alux Concierge, Alux Recommendations, Alux Sources/Explainability, Alux ↔ Context Engine, Alux ↔ Discovery Navigator, Alux ↔ Trust Engine.

### Programa F · Core Platform
Workspace Engine, Context Engine, Discovery Layer, Navigation Contract, Experience Builder core, Lovable Cloud, AI Gateway, PWA, i18n runtime.

### Programa G · Trust Engine (nuevo)
Reviews, Ratings, Verificaciones oficiales, Institutional Badges dinámicos, Reputación empresarial, Señales sociales, Anti-fraude editorial.

### Programa H · Hardening
DSL Colonial M2..M10, a11y AA, performance, responsive, animaciones, microinteracciones, SEO técnico, seguridad continua, pruebas y observabilidad.

## FASE 8 · Ruta hasta el Lanzamiento

### Camino óptimo desde v2.0 (hoy) hasta v4.0 Comercial

#### Bloque 1 · v2.0 → v2.5 (Consumer Rails · Carril A)
Orden recomendado, una iniciativa activa a la vez (Policy 02):
1. **Alux Traveler productivo sobre Context Engine + Navigation Session** (E + F).
2. **Related Collection transversal** en Empresa/Producto/Destino/Categoría (A + E).
3. **Arma tu Viaje productivo** con demo pack real (A).
4. **Recomendaciones contextuales** basadas en `useNavigationSession()` (A + E).
5. **Perfil del Viajero público** (A).

Criterio de salida v2.5: los cinco consumidores explotan la infraestructura N2/N3 sin abrir nuevas piezas de navegación.

#### Bloque 2 · v2.5 → v3.0 (Beta Cerrada)
Dependencias externas:
- **Google Auth productivo** (bloqueador de sign-in real).
- **Push Notifications productivo** (visitante).
- **Trust Engine v1** (Programa G): reviews reales + verificaciones básicas + badges dinámicos activos.
- **Contenido real semilla** en Valladolid, Izamal, Espita.
- **DSL Colonial M2..M5** aplicado a superficies públicas críticas.

#### Bloque 3 · v3.0 → v3.5 (Beta Abierta)
- **Portal Empresarial productivo** (onboarding multiempresa real).
- **Reservas + Carrito** estabilizados.
- **Alux Concierge productivo** con handoff humano.
- **DSL Colonial M6..M8**.

#### Bloque 4 · v3.5 → v4.0 (Comercial)
- **Stripe productivo** (cobros + payouts + comisiones).
- **CRM empresarial básico**.
- **Trust Engine v2** (reputación + anti-fraude).
- **Header & Navigation Builder (15.10.8)** activado.
- **DSL Colonial M9..M10**.
- **Fase 2 de 301** (retirar rutas legacy) autorizada.

### Bloqueadores estratégicos
| Bloqueador | Impacto | Mitigación |
|---|---|---|
| Google Auth no productivo | v3.0 imposible | Configurar provider + verificación de dominio. |
| Stripe no productivo | v4.0 imposible | Onboarding Stripe + Connect. |
| Trust Engine inexistente | v3.0 y v4.0 débiles | Iniciar Programa G tras v2.5. |
| Contenido real insuficiente | Toda superficie pública | Plan de contenidos por destino, previo a v3.0. |
| DSL migración lenta | Percepción de marca | Priorizar M2..M5 antes de v3.0. |

### Dependencias críticas
- Programa G (Trust) depende de Programa A (perfiles) y Programa D (moderación).
- Programa B (empresario) depende de Programa G (reputación) y de Stripe.
- Programa H (hardening) es continuo y bloquea la percepción de calidad en cada versión.

### Riesgos residuales
1. **Ampliación de alcance no autorizada** — mitigado por Infrastructure Freeze reforzada.
2. **Dispersión de la Biblioteca Oficial** — mitigado por Evolutionary Compatibility Rule (H-03).
3. **Regresión visual durante migración DSL** — mitigado por Safe Visual Migration Rule.
4. **Deriva de navegación** — mitigado por Navigation Contract v1.0 congelado.
5. **Fatiga editorial** — mitigado por Plantillas Madre + Studio único.
