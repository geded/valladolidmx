## 1. Entregables

### 1.1 Auditoría MCP (R2)
- Tabla `public.mcp_tool_invocations` con RLS activa. Sólo admin/super_admin leen. GRANTs sólo a `service_role`.
- RPC `public.mcp_log_invocation(...)` `SECURITY DEFINER` con `search_path = public`. `EXECUTE` revocado a `PUBLIC/anon/authenticated`; sólo `service_role`.
- Campos: `invocation_id`, `tool_name`, `contract_version`, `client_id`, `user_id`, `input_hash` (SHA-256 truncado 16 hex), `locale`, `duration_ms`, `success`, `error_code`, `result_count`, `created_at`.
- **No** registra tokens, ni datos personales completos, ni payloads. Sólo hash e IDs.

### 1.2 Rate Limiting (R1)
- Tabla `public.mcp_rate_buckets` (contadores por ventana alineada al múltiplo de `window_seconds` desde epoch). GRANTs sólo a `service_role`. RLS activa sin políticas (acceso vía RPC).
- RPC atómico `public.mcp_rate_hit(scope_key, tool_name, window_seconds, limit)` con UPSERT `ON CONFLICT` — sin condiciones de carrera.
- Helper `src/lib/mcp/lib/rate-limit.ts`:
  - Scope preferente: `user:<uuid>` → `client:<id>` → `ip:unknown`.
  - Políticas por defecto: **autenticado 60/min · 1000/h**, **anónimo 20/min · 200/h**.
  - Fail-open ante error del limiter (no bloquea al usuario legítimo por fallas internas).
  - Respuesta 429 con `retry_after_seconds` en `structuredContent`.

### 1.3 Protección de búsqueda (R3)
- Helper `src/lib/mcp/lib/sanitize.ts`:
  - Mínimo 3 caracteres alfanuméricos Unicode (`\p{L}\p{N}`).
  - Rechaza queries compuestas exclusivamente por `%`, `_`, `*`, `?`, `\`.
  - Cap 60 chars, escape LIKE (`\`, `%`, `_`).
  - `SEARCH_RESULT_HARD_CAP = 25`.
- Aplicado a `search_businesses`. Base lista para FTS multi-idioma en M2.

### 1.4 Output Schemas Zod tipados
Las 3 tools existentes ahora declaran `outputSchema` validado con `OutputSchema.parse(...)` antes de responder:
- `search_businesses` → `results[]`, `result_count`, `contract_version`, `explain{ rationale, sources, limitations, locale_used, locale_fallback }`.
- `get_my_traveler_profile` → `profile`, `contract_version`, `explain`.
- `list_my_travel_plans` → `plans[]`, `result_count`, `contract_version`, `explain`.

### 1.5 Localización (`locale`)
- Todas las tools aceptan `locale` (`es|en|fr|de|it|pt`) con fallback explícito a `es`.
- Cada respuesta declara `locale_used` y `locale_fallback` en `explain` — auditable.
- Contenido multi-idioma real se activa por tool en M1.1/M1.2 al conectar `platform_locales`/`translations`.

### 1.6 Consent Screen (`/.lovable/oauth/consent`)
- Lenguaje humano; se eliminaron scopes técnicos como explicación al usuario.
- Enumera capacidades reales: consultar perfil, consultar planes, buscar información turística pública.
- Declara qué **no** puede: reservar, pagar, modificar planes ni contactar empresas.
- Enlace visible al dominio canónico `https://quehacerenvalladolid.com/mcp`.

### 1.7 Helpers compartidos (`src/lib/mcp/lib/`)
- `contracts.ts` — `LocaleSchema`, `GeoPointSchema`, `SourceCitationSchema`, `ExplainSchema`.
- `ctx-supabase.ts` — cliente Supabase por token (RLS del usuario) o anón; strip de `sb_` bearer.
- `sanitize.ts` — `sanitizeSearchQuery`, `SEARCH_RESULT_HARD_CAP`.
- `hash.ts` — `inputHash` SHA-256 truncado.
- `audit.ts` — `logInvocation`.
- `rate-limit.ts` — `enforceRateLimit`, `DEFAULT_POLICIES`.
- `wrap.ts` — `withMcpGuardrails` (auditoría + rate limit + locale + hash + captura de excepciones; obligatorio en toda tool nueva).

## 3. Seguridad

- `src/lib/mcp/tools/*` **no** referencian `SUPABASE_SERVICE_ROLE_KEY`. El único punto donde se carga `supabaseAdmin` es en `lib/audit.ts` y `lib/rate-limit.ts` para escribir en las tablas internas de gobernanza (nunca para leer datos de usuario). RLS del usuario se preserva en cada consulta a través de `supabaseFor(ctx)`.
- RPCs `mcp_log_invocation` y `mcp_rate_hit`: `EXECUTE` revocado a `PUBLIC/anon/authenticated`; sólo `service_role`.
- Sin cambios en scopes de OAuth ni en tokens.
- Consent screen no expone tokens ni identificadores internos.
