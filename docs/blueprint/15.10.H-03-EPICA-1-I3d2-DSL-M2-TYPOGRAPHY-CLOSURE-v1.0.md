## 1. Objetivo

Consolidar la **escala tipográfica oficial** del Valladolid Colonial Design System v1.0 (§5) como utilidades reutilizables en todo el proyecto, sin refactor destructivo. Primer paso rentable del carril de Hardening, con impacto visible en el Header/mega-menú.

## 2. Cambios entregados

### 2.1 `src/styles.css` (additive-only)
- Reforzado reset base `h1..h4`: se añade `line-height: 1.05` (ya tenía `font-family: var(--font-display)`, `font-feature-settings: "ss01"`, `letter-spacing: -0.015em`).
- **9 nuevas utilidades DSL** vía `@utility` (Tailwind v4):
  - Display: `text-display-xs` (24px), `text-display-sm` (36px), `text-display-md` (48px), `text-display-lg` (64px), `text-display-xl` (88px).
    - `font-family: var(--font-display)`, `font-feature-settings: "ss01"`, tracking negativo escalonado.
  - Body: `text-body-sm` (14px), `text-body-md` (16px), `text-body-lg` (18px). `font-family: var(--font-sans)`, `line-height: 1.5`.
  - Caption: `text-caption` (11px, 600, uppercase, tracking 0.14em).

### 2.2 `src/components/layout/PrimaryMegaMenu.tsx`
- Eyebrows del dropdown desktop (`col.title`) y del acordeón mobile ahora usan `text-caption` (antes: `text-[11px] font-semibold uppercase tracking-[0.14em]`). **Visualmente idéntico**; ahora gobernado por el DSL, no por clases arbitrarias.

### 2.3 No tocado (deliberadamente)
- `SiteHeader.tsx`, `BrandLogo`, `UserMenu`, `LanguageSwitcher`, `SiteFooter`, todos los bloques del EB.
- Colores, radios, sombras, spacing, breakpoints, z-index.
- Base de datos, RLS, server functions, tipos generados.
