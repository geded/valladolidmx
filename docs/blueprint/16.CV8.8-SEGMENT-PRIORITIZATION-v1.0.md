# CV8.8 · Priorización por Segmento — Blueprint v1.0

**Programa:** Visitor Intelligence Platform · **Carril A · Producto**
**Dependencias:** CV8.0 Journey Contracts · CV8.1 Ingesta · CV8.2 Projection · CV8.4 Segmentation · CV8.6 Recommendation Validation Loop · CV8.7 Prescriptive Decision Prioritization.

## 1. Behavioral Change Statement (Founder Behavioral First)

**Antes:** el equipo priorizaba oportunidades globales sin distinguir cómo afectaban a cada grupo de viajeros, corriendo el riesgo de concentrar esfuerzo sólo en los segmentos ya convertidores.

**Después:** el Visitor Intelligence Center señala, para cada segmento con evidencia suficiente, si estamos ante una **oportunidad comercial**, **de experiencia**, **de inclusión**, un **riesgo de abandono** o **evidencia insuficiente** — con muestra, delta contra baseline, confianza, impacto esperado, riesgo de sesgo y explicaciones alternativas antes de decidir.

## 2. Founder Business Value

- **Objetivo:** aumentar conversión y permanencia sin sacrificar confianza, inclusión ni calidad de experiencia.
- **Hipótesis:** al distinguir tipos de oportunidad por segmento, priorizaremos con equidad y evitaremos mejorar sólo a quienes ya convierten.
- **North Star:** JPR-30d — el mismo — desagregado ahora por segmento con riesgo de sesgo declarado.

## 3. Alcance (DoD)

- Priorización por **locale**, **destination** y **capability**.
- **country**, **channel** y **device** permanecen `contract_pending` mientras la ingesta no capture su contrato.
- Respeto estricto de `MIN_SEGMENT_POPULATION = 25`; segmentos menores agrupados como *Otros* o marcados `insufficient_data`.
- Comparación contra baseline global y contra el propio segmento.
- Matriz **Segmento → Hallazgo → Prioridad → Acción → Impacto esperado → Riesgo de sesgo**.
- Typecheck + build limpios.
- Completion Report con Outcome Validation.

## 4. Arquitectura

### 4.1 Módulo puro

`src/lib/visitor-intel/segment-prioritization.ts` — `prioritizeSegments({ segments, validation })` recibe los `JourneySegmentSnapshot` (v1.0.0) oficiales de CV8.4 y la confianza aprendida (`FamilyLearningSignal[]`) de CV8.6. No hace I/O, no persiste nada.

### 4.2 Pesos oficiales (auditables)

| Factor | Peso |
| --- | --- |
| impact | 20% |
| confidence | 15% |
| urgency | 15% |
| reach | 15% |
| friction | 15% |
| traveler_benefit | 10% |
| ecosystem_benefit | 10% |

El score **nunca depende sólo de conversión o revenue** (Founder Fair Opportunity Principle).

### 4.3 Clasificación

| Tipo | Regla |
| --- | --- |
| `abandonment_risk` | delta ≤ −20% vs baseline |
| `inclusion_opportunity` | delta ≤ −5% vs baseline |
| `commercial_opportunity` | delta ≥ +5% vs baseline |
| `experience_opportunity` | intent/active ≥ 25% y JPR ≤ baseline |
| `insufficient_data` | población < `MIN_SEGMENT_POPULATION` o sin señal accionable |

### 4.4 Fairness metadata (obligatoria por hallazgo)

`sample_size` · `delta_vs_baseline` · `confidence` (low/medium/high/unknown) · `expected_impact` · `bias_risk` (low/medium/high) · `alternative_explanations` (idioma, contenido, accesibilidad, confianza, precio, canal, atención, estacionalidad).

### 4.5 UI

Nueva sección **CV8.8 · Priorización por Segmento** en `/cms/visitor-intel`. Renderiza la matriz completa: chips por tipo, tarjeta por segmento con acción, impacto esperado, explicaciones alternativas y factores del score expandibles.

## 5. Métricas de éxito

- ≥ 1 hallazgo `inclusion_opportunity` o `abandonment_risk` visible cuando existen segmentos con baja progresión.
- 100% de hallazgos con `bias_risk` y ≥ 3 explicaciones alternativas.
- Cero snapshots persistidos; cero identidades individuales expuestas.
