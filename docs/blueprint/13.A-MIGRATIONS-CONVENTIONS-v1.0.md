# 13.A – Migrations Conventions
**Versión:** 1.0
**Estado:** Activo desde Fase 1 – Bloque A
**Blueprint refs:** 13.0A · 13.1 · 13.2 · 11.5

## 1. Convención de nombres

Los archivos los genera automáticamente Lovable Cloud con timestamp + UUID:

```
YYYYMMDDHHMMSS_<uuid>.sql
```

Para hacerlas legibles, **cada migración debe abrir con un encabezado
normalizado** que indique fase, bloque y propósito:

```sql
-- ============================================================
-- Phase <N> · Block <X> · <Short purpose>
-- Migration: phase<N>_block<X>_<short_snake_case>
-- Blueprint refs: <doc1>, <doc2>, ...
-- ============================================================
```

El campo `Migration:` actúa como **slug lógico** independiente del
timestamp. Es el identificador citado en el reporte de cada Decision Gate.

## 3. Reglas obligatorias (resumen Blueprint 11.5)

1. **UUID** como PK (`gen_random_uuid()` – nativo PG13+; no usar `pgcrypto`).
2. **Timestamps** `created_at` + `updated_at` con `DEFAULT now()`.
3. Trigger `BEFORE UPDATE` con `public.set_updated_at()` en toda tabla con `updated_at`.
4. **Toda tabla pública** debe llevar, en este orden y en la **misma migración**:
   1. `CREATE TABLE public.<t>(...)`
   2. `GRANT` a los roles que las políticas permitan (siempre `service_role`).
   3. `ALTER TABLE ... ENABLE ROW LEVEL SECURITY`.
   4. `CREATE POLICY ...`
5. RLS **nunca recursiva**: usar `SECURITY DEFINER` (`has_role()`, etc.).
6. Roles **nunca** se guardan en `profiles` (Blueprint 11.2 / 13.3).
7. `auth.uid()` como referencia de propietario; FK directa a `auth.users` solo en `profiles`/`user_roles`.
8. Slugs case-insensitive (`citext` desde schema `extensions`).
9. Soft delete vía `deleted_at TIMESTAMPTZ` cuando lo exija el dominio (no por defecto).
10. Estados editoriales: enum `content_status` (`draft`, `in_review`, `approved`, `published`, `archived`).

## 4. Extensiones disponibles

Instaladas en schema `extensions` (`phase1_blockA_hardening`):

- `unaccent` – normalización para slugs y búsqueda.
- `pg_trgm` – índices trigram, fuzzy search.
- `citext` – texto case-insensitive.

`gen_random_uuid()` es **nativo** en PG13+; `pgcrypto` no se utiliza.

## 5. Helpers compartidos

| Función                       | Estado            | Propósito                              |
| ----------------------------- | ----------------- | -------------------------------------- |
| `public.set_updated_at()`     | ✅ Bloque A       | Trigger genérico de `updated_at`.      |
| `public.has_role(uuid, role)` | ⏳ Bloque C       | Chequeo de rol (SECURITY DEFINER).     |
| `public.current_org_ids()`    | ⏳ Bloque C       | Organizaciones del usuario.            |

## 6. Datos semilla

Las semillas oficiales (países, estados, región Oriente Maya, destinos
iniciales) viven en migraciones `phase1_seed_*` y son **reproducibles e
idempotentes** (`ON CONFLICT DO NOTHING`). Datos demo/QA van en
`phase1_seed_demo_*` y deben poder borrarse sin afectar producción.

## 7. Regla de oro

Una migración no se considera terminada hasta que:

- compila sin errores,
- el linter de seguridad queda sin WARN nuevos,
- la tabla afectada tiene `GRANT` + RLS + policies coherentes,
- el slug queda referenciado en el reporte del Decision Gate.
