## Alcance entregado

- **DB:** columnas `share_token uuid unique` + `shared_at timestamptz` en
  `travel_plans` + índice parcial. RPC `travel_plan_get_shared(_token uuid)`
  `SECURITY DEFINER` que devuelve `{plan, items}` filtrando campos
  sensibles (no `user_id`, no `notes` del plan). Sin políticas `TO anon`
  sobre las tablas: única puerta pública es el RPC.
- **Server fns** (`src/lib/traveler/travel-plans.functions.ts`):
  - `enableShareLink({ planId })` — owner-only, idempotente.
  - `disableShareLink({ planId })` — owner-only, limpia token.
  - `getSharedPlan({ token })` — **pública** (sin `requireSupabaseAuth`),
    cliente publishable + RPC.
- **UI pública:** `src/routes/viaje-compartido.$token.tsx` con SSR loader,
  `head()` con `noindex/nofollow`, hero + lista numerada de items, botón
  "Imprimir / Guardar PDF" que dispara `window.print()` con `@media print`
  embebido, fallbacks 404 y error.
- **UI Workspace** (`/cuenta/mi-viaje`): sección "Compartir y exportar" con
  generar/copiar/abrir/revocar link + acceso a impresión.

## Fuera de alcance (por diseño)

PDF server-side, sharing colaborativo, redes sociales, export Excel/QR,
revocación granular por viewer.

## Reutilización

- `travel_plans` / `travel_plan_items` (Iniciativa 7).
- Cliente publishable + `SECURITY DEFINER` (mismo patrón que otras RPC
  públicas).
- `window.print()` nativo — cero dependencias nuevas.

## Seguridad

- Token = UUID v4 (128 bits) generado con `crypto.randomUUID()`.
- RPC filtra campos: excluye `user_id`, `notes` del plan, `meta`, `case_id`.
- Ruta pública emite `robots: noindex, nofollow`.
- Revocación instantánea (`share_token = NULL`).
- Linter warnings existentes sobre "Public Can Execute SECURITY DEFINER
  Function" son **esperados y deseados** para esta RPC (es su contrato).
