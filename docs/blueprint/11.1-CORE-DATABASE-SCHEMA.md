# Valladolid.mx Platform Blueprint

# 11.1 – CORE DATABASE SCHEMA

Versión 1.0

## Objetivo
Definir el modelo principal de datos sobre el cual se construirá toda la plataforma.

## Convenciones
- UUID como llave primaria.
- created_at, updated_at, created_by, updated_by.
- Soft delete: deleted_at, deleted_by.
- Slug único para entidades públicas.
- Soporte multilenguaje.

## Dominios
### Geografía
countries
states
tourism_regions
destinations
destination_zones

### Empresas
businesses
business_categories
business_locations
business_contacts
business_media
business_hours
business_social_links

### Productos
products
Tipos: experiencia, hotel, restaurante, evento, tour, transporte, servicio y producto artesanal.

### Multimedia
media_assets

### Reseñas
reviews

### Usuarios
users (roles definidos en 11.2)

### Arma tu Viaje
travel_plans

### Concierge
concierge_requests

### IA
knowledge_sources
embeddings_queue
ai_summaries

## Reglas
- No duplicar información.
- Usar claves foráneas.
- Tablas puente para relaciones muchos-a-muchos.
- No almacenar binarios en PostgreSQL.

## Índices
slug, destination_id, business_id, product_id, category_id, status, created_at.

## Criterio de aceptación
Modelo preparado para CMS, IA, multilenguaje y expansión internacional.
