## 2. Archivos modificados

**Modificados (2):**

- `src/components/layout/BreadcrumbTerritorial.tsx`
  - `crumbs` pasa de requerido a opcional.
  - Nueva prop `useContextCrumbs?: boolean` (default `false`).
  - Cuando `useContextCrumbs` es `true` **y** `useResolvedContext()` retorna contexto, deriva migas de `ancestors + current`. En cualquier otro caso usa `crumbs` (comportamiento previo).
  - Retorna `null` si no hay migas efectivas — igual al render vacío previo bajo `PublicShell` (que ya filtraba con `hasCrumbs`).
- `src/components/discovery/PublicShell.tsx`
  - Nuevas props `contextDeclaration?: RouteContextDeclaration` y `useContextCrumbs?: boolean`.

  - Cuando ambas props son `undefined`/`false` (todas las rutas actuales), el render es **byte-idéntico** al previo (mismo JSX, mismos className, mismos condicionales).

## 3. Confirmación de no cambio observable

### 3.1 Análisis de retrocompatibilidad

Las 25+ llamadas actuales a `PublicShell` no pasan `contextDeclaration` ni `useContextCrumbs`. Por diseño:

```tsx
// PublicShell sin contextDeclaration:
//   → NO se monta ContextEngineProvider
//   → useContextCrumbs default false
//   → hasCrumbs = useContextCrumbs || Boolean(crumbs && crumbs.length > 0)
//                = false || Boolean(...)   // igual al anterior
//   → BreadcrumbTerritorial recibe crumbs={...} y useContextCrumbs=false
//     → useResolvedContext() retorna null (no hay provider) → cae a crumbs
```

El código produce **exactamente el mismo árbol JSX** que antes de I2 cuando no se usan las nuevas props.

## 5. Comportamiento del playground

Ruta: `/lovable/context-engine-preview` (`noindex,nofollow`).

Tres escenarios seleccionables:

1. **Destino (Valladolid)** — declaración explícita con ancestro `region`. Persiste como `previous`.
2. **Categoría (Hoteles) desde destino** — `inherit: ["region","destination"]`. Al llegar desde el escenario 1, el resolver hereda vía whitelist (`destination → category`) y el breadcrumb del `PublicShell` real muestra `Inicio › Oriente Maya › Valladolid › Hoteles`.
3. **Detalle (Hacienda Selva) desde categoría** — `inherit: ["region","destination","category"]`. Encadena herencia multinivel usando `previous.ancestors`. El breadcrumb muestra `Inicio › Oriente Maya › Valladolid › Hoteles › Hacienda Selva`.

Bloques del playground:

- **Panel de debug** — muestra `source`, `canonical`, `ancestors`, `region/destination/category`, `previous`.
- **PublicShell real** — con `contextDeclaration` + `useContextCrumbs`, valida que el breadcrumb de producción consume el contexto correctamente.
- **Log de eventos** — captura en vivo `context_engine.resolved`, `context_engine.inherited`, `context_engine.previous_saved`, `context_engine.fallback_kind_default`.
- **Reset** — limpia `sessionStorage` y el log para reproducir el flujo desde cero.

## 6. Riesgos encontrados y estado

| # | Riesgo | Estado |
|---|---|---|
| R1 | `useResolvedContext()` en `BreadcrumbTerritorial` añade un `useContext` en todas las rutas | **Aceptable.** El hook siempre corre; sólo altera comportamiento cuando `useContextCrumbs` es `true`. Costo despreciable. |
| R2 | Provider montado en `PublicShell` podría activar `sessionStorage` en rutas no diseñadas para ello | **Mitigado.** El provider **sólo se monta** si `contextDeclaration` está presente. Hoy ninguna ruta real lo pasa → cero rutas escriben `sessionStorage`. |
| R3 | Cambio de `crumbs` (requerido → opcional) rompe llamadores | **Descartado.** Los 25+ llamadores pasan `crumbs`; opcional es superset compatible. Typecheck verde. |
| R4 | Doble render/flash cuando I3+ active herencia | **Diferido a I3.** `useSyncExternalStore` con `getServerSnapshot` ya está previsto para prevenirlo. |
| R5 | Reintroducción accidental de `crumbs={crumbs!}` con `!` no-null assertion | **Eliminada.** Ahora `PublicShell` pasa `crumbs={crumbs}` sin `!` y `BreadcrumbTerritorial` maneja opcional. |

## 7. Limitaciones conocidas

- Aún **ningún consumidor real** deriva migas del contexto — es exactamente el objetivo de I2.
- El playground sigue simulando `previous` mediante los tres botones; la verificación end-to-end con `previous` real entre rutas ocurrirá cuando I3 migre la primera ruta piloto.
- `variant="hero"` y `variant="minimal"` en `PublicShell` no renderizan header/breadcrumb; el provider igualmente se monta si hay `contextDeclaration` (útil para consumidores no visuales como Alux/Analytics).
