## 3. Detalle por dominio

### 3.1 Paleta — ✓ Alineado

- `--background` (Crema Cálido), `--foreground` (Noche Maya), `--primary` (Ocre Valladolid), `--secondary`/`--caliza` (Piedra Caliza), `--accent` (Cenote Profundo accesible), `--selva`, `--cenote`, `--atardecer`: **coinciden 1:1** con la paleta oficial del DSL §4.
- Charts 1–5 respetan el orden semántico (ocre, selva, cenote profundo, cenote micro, verde oscuro).
- **Acción:** ninguna. Mantener como fuente única.

### 3.2 Tipografía — ✓ Alineado

- `--font-display: Fraunces`, `--font-sans: Inter`, `--font-script: Tangerine` presentes.
- Reglas base h1..h4 aplican Fraunces + tracking editorial (`letter-spacing: -0.015em`) y `font-feature-settings: "ss01"` — coherente con §5 del DSL.
- **Ajuste menor pendiente (I3.d.2):** formalizar escala editorial en tokens (`--text-display-2xl`, `--text-display-xl`, `--text-editorial-lg`, `--text-body-lg`, `--text-caption`) para que los bloques no la re-hardcodeen.

### 3.3 Radios — ✓ Alineado

- Escala `radius-sm..4xl` derivada de `--radius: 0.75rem`. Compatible con el DSL §7 (soft, medium, generous).
- **Acción:** ninguna en I3.d.1. Documentar mapeo Fig→Token en I3.d.2.

### 3.4 Sombras — ✓ Alineado

- `--shadow-card`, `--shadow-raised`, `--shadow-overlay`, `--shadow-focus`: tono neutro cálido implícito vía oklch.
- **Ajuste menor recomendado (I3.d.2):** teñir levemente hacia oklch(0 0 0 / warm) usando `color-mix` con `--foreground` para reforzar calidez colonial en superficies claras.

### 3.5 Motion — ✓ Alineado

- 5 velocidades (`instant`..`deliberate`) + 2 easings (`standard`, `emphasized`). Coherente con §17 del DSL.
- **Acción:** ninguna.

### 3.6 Estados funcionales — ✓ Alineado

- success/warning/info/destructive definidos claro y oscuro con contraste AA.
- **Acción:** ninguna.

### 3.7 Superficies — ✓ Alineado

- `--surface`, `--surface-raised`, `--surface-overlay`, `--border-strong`, `--divider`, `--scrim`: cubren la jerarquía del DSL §6.
- **Acción:** ninguna.

### 3.8 Institutional Badges — ✗ Falta (crítico para I3.c)

El DSL §20 define un sistema de tokens de badges que **aún no existe** en `styles.css`. Requerido antes de I3.c:

```
--color-badge-pueblo-magico
--color-badge-pueblo-magico-foreground
--color-badge-patrimonio
--color-badge-patrimonio-foreground
--color-badge-oriente-maya
--color-badge-oriente-maya-foreground
--color-badge-despierta
--color-badge-despierta-foreground
--color-badge-certification
--color-badge-award
--radius-badge
--shadow-badge
```

- **Acción:** añadir en I3.d.2 como bloque atómico (una PR dedicada) sin tocar componentes existentes.

### 3.9 Escala editorial y espaciado — △ Extensión

Actualmente los bloques usan clases Tailwind directas (`text-4xl`, `py-24`). Para blindar el ADN editorial y evitar drift:

```
--text-display-2xl / xl / lg
--text-editorial-lg / md
--text-body-lg / md / sm
--text-caption / overline
--space-editorial-2xl / xl / lg / md / sm
--aspect-hero (16/7) / --aspect-cover (16/9) / --aspect-card (4/5) / --aspect-portrait (3/4)
```

- **Acción:** formalizar en I3.d.2 como capa aditiva; los bloques migran uno a uno.

### 3.10 Iconografía — △ Documentar

- No hay tokens de icono explícitos. Contrato pendiente: `stroke-width: 1.5`, tamaños 16/20/24/32, familia Lucide (ya en uso).
- **Acción:** documentar en I3.d.2, sin cambios de código si Lucide 1.5 ya es default.

### 3.11 Modo oscuro colonial — ~ Ajuste menor

- Base actual `oklch(0.329 0.048 129.7)` (Selva Oscuro) es válida para el DSL §4.5.
- **Riesgo detectado:** contraste `--muted-foreground` sobre `--muted` en dark está en el límite AA (≈4.6:1). Verificar en I3.d.2 con herramienta y ajustar `--muted-foreground` +0.02 L si necesario.

## 6. Principio de Experiencia — check por bloque

Todos los bloques a migrar en I3.d.3 deberán responder afirmativamente:

> "¿Ayuda al visitante a imaginar, descubrir o planear mejor su viaje?"

Se añade esta pregunta al DoR de cada migración. Si un bloque no la responde, se replantea antes de migrarse.

---

## 7. Riesgos y mitigaciones

- **Drift visual entre bloques migrados y no migrados** → mitigar publicando migraciones en lotes coherentes por superficie (ej. terminar Home antes de tocar Destination).
- **Regresión en modo oscuro** → cada PR incluye captura light + dark antes/después.
- **Rework en bloques cerrados** → nulo, porque el DSL ya está aprobado; sólo se retocan variantes/tokens, contratos intactos.
