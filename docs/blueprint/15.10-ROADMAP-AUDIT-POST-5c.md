**Conclusión:** todas las superficies *operativas autenticadas* declaradas en el Workspace Migration Blueprint están migradas. La serie 15.10.5c está cerrada formalmente.

## 2. Superficies públicas detectadas (no migradas, no pendientes de Workspace)

Rutas SSR anónimas que hoy usan `PageShell` + `SiteHeader` + `SiteFooter`:

- `/` (Home pública · composición Experience Builder)
- `/marketplace`, `/marketplace/buscar`, `/marketplace/$slug`
- `/empresas`, `/experiencias`, `/hoteles`, `/restaurantes`, `/eventos`
- `/oriente-maya`, `/oriente-maya/$destino`
- `/arma-tu-viaje`, `/alux`
- `/l/$slug`, `/preview/$token`
- `/auth`, `/reset-password`

Estas superficies **no pertenecen al Workspace Engine** por diseño:

- Son SSR-first para anónimos · SEO/OG por ruta · sin sesión, sin sidebar, sin inspector, sin command palette, sin selección/contexto operativo.
- Aplicar `WorkspaceShell` rompería SEO, accesibilidad pública y la propia política "Workspace para operación, no para descubrimiento".

Por tanto **no son candidatas a 15.10.5c ni a una "Ola 6" del Workspace Engine**. Pertenecen a un *Public Discovery Layer* gobernado por separado.

## 3. Hallazgos · gobernanza pendiente

Aunque las superficies públicas están fuera del Workspace Engine, sí existen inconsistencias entre ellas que merecen una adenda dedicada antes de PWA Offline:

- Header/footer públicos no declarados como contrato (`SiteHeader`/`SiteFooter` se referencian ad-hoc).
- Cards públicos duplican estilos (Categoría, Destino, Empresa, Reseña, Ruta) sin un Registry compartido.
- Metadata SEO/OG inconsistente entre rutas (algunas con `og:image` derivado, otras sin).
- `BreadcrumbTerritorial` y `LanguageSwitcher` sin integración uniforme.
- Experience Builder ya entrega Home pero el resto de superficies públicas no consumen aún composiciones EB.

Estas inconsistencias no bloquean PWA Offline, pero conviene resolverlas antes para no cachear superficies con shells distintos.

## 4. Propuesta de actualización del Roadmap 15.10

Insertar una nueva adenda **15.10.5d — Discovery Layer (Public Surface Consolidation)** antes de iniciar la infraestructura PWA Offline (15.10.6):

```
15.10.5  Workspace Engine v2.0
  ├─ 15.10.5a  Foundations          ✅
  ├─ 15.10.5b  Contextual Layer     ✅
  └─ 15.10.5c  Workspace Migration  ✅ (5 olas cerradas)

15.10.5d  Discovery Layer            ← NUEVA, recomendada antes de PWA
  · Public Shell contract (Header/Footer/Breadcrumb/LanguageSwitcher)
  · Public Cards Registry
  · SEO/OG metadata por ruta como contrato
  · Consumo opcional de composiciones EB en superficies de descubrimiento
  · Cero cambios funcionales · paridad 1:1

15.10.6  PWA Offline + Installability  ← después de 5d
  · Manifest + iconos + theme
  · Service Worker controlado (NetworkFirst HTML, CacheFirst hashed)
  · Guards de preview Lovable
  · Kill-switch worker

15.10.7+  Migración de capacidades sobre Workspace (Workspace Evolution Policy)
```

## 5. Secuencia recomendada

1. **15.10.5d Discovery Layer** — consolidar el shell público, sin tocar funcionalidad.
2. **15.10.6 PWA Offline** — sólo cuando shells (Workspace y Discovery) sean estables y declarativos, para evitar cachear superficies inconsistentes.
3. Adendas posteriores: migración de capacidades existentes sobre Workspace (Workspace Evolution Policy).

## 6. Recomendación operativa

**No iniciar PWA Offline hasta aprobar y cerrar 15.10.5d.** Cachear superficies públicas con shells inconsistentes generaría regresiones SEO difíciles de revertir una vez instalada la app.
