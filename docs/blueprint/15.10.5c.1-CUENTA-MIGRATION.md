# 15.10.5c.1 · Ola 1 — Migración de `/cuenta/*` al Workspace Engine

Estado: **Implementación entregada — pendiente de aprobación formal de cierre.**

## Alcance ejecutado (1:1, sin cambios funcionales)

- Reemplazado `src/routes/_authenticated/cuenta/route.tsx`: el layout local
  (sidebar + nav propia + tarjeta de cierre de sesión) se eliminó.
  La ruta ahora envuelve `<Outlet />` en
  `<WorkspaceProvider initialWorkspaceId="cuenta"><WorkspaceShell>` —
  única vía oficial.
- Extendida la `WorkspaceDefinition` `cuenta` en
  `src/lib/workspace/definitions/index.ts` para cubrir todas las rutas
  reales del módulo: `Resumen`, `Favoritos`, `Carrito`, `Historial`,
  `Actividad`, `Concierge`, `Notificaciones`, `Perfil`. La navegación se
  declara únicamente en el Navigation Registry.
- Declarado el contrato contextual (`context`) del workspace `cuenta` con
  entidades `favorite`, `cart-item`, `case`, `order`, selección `single`
  y vistas `summary` / `favorites` / `history`.
- Declarada `aluxCapabilities` y enriquecido `alux` con `summary`,
  `suggestedActions`, `rationale`, `sources`, `effect`, `reversible` y
  `confirm` para cumplir Explainable by Default.
- Ajuste mínimo en `WorkspaceProvider`: cuando un route fija
  `initialWorkspaceId`, las prefs persistidas no pueden sobrescribir el
  workspace activo (la URL manda). Necesario para que `/cuenta` muestre
  siempre el workspace `cuenta` independientemente del último usado.

## Cero duplicación

- ❌ Eliminado: sidebar local de `cuenta/route.tsx`, navegación local,
  tarjeta de cierre de sesión propia, contenedor `max-w-[1400px]` propio.
- ✅ Reutilizado: `WorkspaceShell`, `WorkspaceSidebar`, `WorkspaceTopbar`,
  `WorkspaceBottomNav`, `WorkspaceInspector`, `CommandPalette`,
  `WorkspaceContextProvider`, `SheetStackProvider`, `Toaster`.
- Ningún archivo migrado introduce nuevos componentes de layout, sidebar,
  topbar, drawer, command palette o bottom sheet.
- Las páginas hijas (`/cuenta/index`, `/cuenta/perfil`, `/cuenta/favoritos`,
  `/cuenta/carrito`, `/cuenta/historial`, `/cuenta/actividad`,
  `/cuenta/notificaciones`, `/cuenta/concierge`,
  `/cuenta/concierge/$caseId`, `/cuenta/concierge/$caseId/evaluar`,
  `/cuenta/pagos.exito`, `/cuenta/pagos.error`) no se tocaron; siguen
  funcionando como children del nuevo shell.

## Cumplimiento de políticas

- **Workspace First / Build Once, Reuse Everywhere**: cumplido. Toda la
  estructura visual procede del Workspace Engine.
- **Migration Before Evolution**: ningún flujo, regla de negocio o
  capacidad funcional fue modificado.
- **Definition of Migration Done**: `cuenta` consume sólo Workspace Engine
  + Registries oficiales; sin layouts/navegación paralelos; sin código
  legacy huérfano introducido por esta ola.
- **One Wave, One Validation**: typecheck en verde (`tsgo --noEmit`).
  Smoke tests visuales pendientes de validación por aprobador.
- **Migration Granularity**: esta etapa migra una superficie (Cuenta) y
  sólo una; Concierge/Portal/CMS/Founder no se tocan.

## Smoke tests propuestos

- `/cuenta` carga dentro de `WorkspaceShell`, sidebar muestra grupos
  `viaje` + `config`, switcher activo en "Mi Cuenta".
- `/cuenta/favoritos`, `/cuenta/carrito`, `/cuenta/historial`,
  `/cuenta/actividad`, `/cuenta/notificaciones`, `/cuenta/perfil`
  navegan sin reload y mantienen sidebar activo.
- `/cuenta/concierge/$caseId` y `/cuenta/concierge/$caseId/evaluar` abren
  expediente conservando el shell.
- `/cuenta/pagos.exito?...` y `/cuenta/pagos.error?...` preservan query
  params al renderizar.
- `⌘K` abre Command Palette y lista nav items del workspace cuenta.
- Bottom-nav visible en breakpoints xs/sm con Resumen · Favoritos ·
  Carrito · Concierge.
- Topbar Copiloto abre Inspector con el headline declarativo.
- Modo oscuro y viewports iPhone/iPad/Desktop sin regresiones visuales.
