## Historias entregadas

### US-E5.1 · Migración: handle + visibilidad
Añadido a `traveler_profiles`:
- `public_handle text` (unique, case-insensitive, 3–24 chars, `^[a-z0-9_]+$`)
- `is_public boolean default false`
- `public_display_name text` (≤60)
- `public_bio text` (≤200)
- `avatar_url text` (path relativo dentro del bucket `avatars`)

Objetos DB nuevos:
- `is_reserved_traveler_handle(text)` — fuente única de handles reservados.
- Trigger `trg_validate_traveler_public_handle` — normaliza a minúsculas, valida longitud/formato/reserva y recorta bio/display_name.
- RPC pública `get_public_traveler_profile(text)` — SECURITY DEFINER, sólo devuelve datos cuando `is_public=true`. Sólo campos seguros.
- RPC pública `check_traveler_handle_available(text)` — validez + disponibilidad.
- Bucket privado `avatars` + policies de Storage (owner escribe en su prefijo `<uid>/...`).

### US-E5.2 · Server functions
`src/lib/traveler/traveler-public.functions.ts`:
- `getPublicTravelerProfile({ handle })` — pública, publishable client + RPC + firma de avatar (7 días).
- `checkHandleAvailability({ handle })` — pública.
- `getMyPublicProfile()` — protegida (`requireSupabaseAuth`).
- `updateMyPublicProfile(input)` — protegida, whitelist estricta, valida que `is_public=true` requiera handle existente.

### US-E5.3 · Superficie pública `/viajero/:handle`
- Ruta `src/routes/viajero.$handle.tsx` con `head()` dinámico (title, description, `og:type: profile`, `og:image = avatar` cuando existe).
- 404 `notFound()` cuando la RPC devuelve `null` (perfil inexistente o privado).
- Componente `TravelerPublicProfileSurface` en Discovery Layer (PublicShell): avatar/iniciales, nombre público, handle, bio, chips país+idiomas.

### US-E5.4 · Panel `/cuenta/perfil-publico`
- Ruta autenticada bajo el Workspace `cuenta`.
- Form completo: avatar upload (bucket `avatars`, prefijo `<uid>/...`, límite 3 MB, tipo image/*), handle con verificación debounced en vivo, nombre, bio (contador 200), país, idiomas, toggle `is_public` (bloqueado si no hay handle válido), CTA "Ver mi perfil público".
- Nav item añadido en `src/lib/workspace/definitions/index.ts` (group `config`, order 10).
- Feedback humano para todos los errores del trigger (`reserved_handle`, `invalid_handle`, `handle_taken`).

## Decisiones fijadas
- **Visibilidad:** privado por defecto (opt-in explícito).
- **URL:** `/viajero/:handle` elegido por el viajero, unicidad case-insensitive.
- **V1 no muestra:** intereses, favoritos, plan de viaje. Reservados a épicas futuras con toggles granulares.

## Cumplimiento de reglas
- ✅ Cero infra nueva (reusa `traveler_profiles`, PublicShell, Workspace Engine, Storage).
- ✅ Loader público llama server fn pública (no `requireSupabaseAuth`) → SSR/prerender seguro.
- ✅ Grants explícitos en cada RPC (`REVOKE ALL … FROM PUBLIC` + `GRANT EXECUTE TO anon, authenticated`).
- ✅ Storage bucket privado + signed URLs desde servidor (no exposición directa).
- ✅ Design System: reusa tokens y PublicShell del DSL colonial.
