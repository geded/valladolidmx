## 1. Objetivo

Habilitar a los editores de cada empresa a administrar desde el Portal su
catálogo (productos) y promociones asociadas, manteniendo el workflow
editorial: el Portal puede crear borradores, editar, archivar y
solicitar/retirar revisión. Aprobación y publicación permanecen
reservadas al CMS Studio.

## 3. Entregables técnicos

### 3.1 Migración (aditiva)

- **Índice único parcial** sobre `business_media(business_id) WHERE role='logo'`.
  Garantiza a nivel de base de datos que jamás puedan coexistir dos logos
  activos para una misma empresa, aun bajo subidas concurrentes
  (cumple requisito explícito de la autorización).
- **Extensión** de `transition_content_status` para incluir
  `entity_kind = 'product'` (mapeo a `public.products`), conservando el
  resto del cuerpo intacto.
- **RPCs nuevas** (`SECURITY DEFINER`, `search_path = public`, EXECUTE
  revocado a `PUBLIC` y otorgado sólo a `authenticated`):
  - Productos: `create_business_product`, `update_business_product`,
    `archive_business_product`, `request_product_review`,
    `withdraw_product_review`.
  - Promociones: `create_business_promotion`, `update_business_promotion`,
    `archive_business_promotion`, `request_promotion_review`,
    `withdraw_promotion_review`.
- 0 cambios en tablas, columnas, RLS, triggers o buckets de Storage.

### 3.2 Server functions (TanStack)

`src/lib/portal/business-catalog.functions.ts`:

| Función | Rol mínimo | Notas |
|---|---|---|
| `listBusinessProducts` / `listBusinessPromotions` | viewer | Pre-check `has_business_access` antes de proyectar columnas. |
| `createBusinessProduct` | editor | Whitelist: `name`, `slug`, `product_type`, `tagline`, `description`, `price_amount`, `price_currency`, `duration_minutes`, `capacity`. Estado inicial `draft`. |
| `updateBusinessProduct` | editor | Whitelist. Soporta `clearPrice` para nulificar. Nunca toca `status`, `slug`, `business_id`, `published_at`. |
| `archiveBusinessProduct` | editor | Soft-delete + status `archived`. |
| `requestProductReview` / `withdrawProductReview` | editor | Transición controlada `draft↔in_review`. |
| `createBusinessPromotion` | editor | Whitelist + validación de rango de fechas y producto perteneciente a la empresa. |
| `updateBusinessPromotion` | editor | Whitelist. Soporta `clearDiscount` / `clearDates`. |
| `archiveBusinessPromotion` | editor | Soft-delete + status `archived`. |
| `requestPromotionReview` / `withdrawPromotionReview` | editor | Transición controlada `draft↔in_review`. |

Reglas comunes:
- `.middleware([requireSupabaseAuth])` en todas las server fns.
- `inputValidator` rechaza datos malformados antes del handler.
- Doble validación: la server fn revalida acceso vía `has_business_access`
  cuando aplica; la RPC SECURITY DEFINER vuelve a validarlo dentro.
- Aprobación y publicación quedan fuera del Portal (no se exponen).

### 3.3 UI del Portal

- Nueva ruta `/portal/catalogo` (`src/routes/_authenticated/portal/catalogo.tsx`)

- Listado con badge de estado, edición inline de campos whitelisted, y

- Enlace "Catálogo" añadido al nav del layout del Portal.

## 6. Integridad Referencial

- `products.business_id → businesses.id` y
  `promotions.business_id → businesses.id` permanecen estables y validadas
  por las RPCs antes de cualquier escritura.
- `promotions.product_id → products.id` validada cross-empresa antes de
  insertar.
- Auditoría coherente con Fase 1: `content_audit_log.entity_kind ∈
  {'product','promotion'}` y `entity_id` apuntando al registro
  correspondiente.
- `business_media`: nuevo índice parcial es aditivo, no rompe contratos
  existentes.

## 7. Mitigación de riesgos

| Riesgo | Mitigación |
|--------|-----------|
| RW3-1 Service role en cliente | No se usa. Todas las escrituras pasan por RPCs SECURITY DEFINER. |
| RW3-2 Cross-empresa | `has_business_access('editor')` en server fn y dentro de cada RPC. `product_id` de una promoción se valida contra el `business_id` del Portal. |
| RW3-3 Escalada editorial | El Portal solo expone `request_*_review` y `withdraw_*_review`. `transition_content_status` requiere `is_editor_or_admin` global — fuera del alcance del Portal. |
| RW3-Catalog Datos inválidos | Validación dura en `inputValidator` y en las RPCs (slug, precio, descuento, rango de fechas, longitudes). |
| RW3-Audit Auditoría inyectable | Acciones limitadas a un dominio fijo (`product.*`, `promotion.*`, `transition`). |
| RW3-Logo concurrencia (autorización Etapa 6) | Índice único parcial `business_media(business_id) WHERE role='logo'` + reemplazo transaccional dentro de `register_business_media`. |
