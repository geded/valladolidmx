## 1 · Cambios entregados

### 1.1 Base de datos (migración)
- Nueva tabla `public.page_redirects` con RLS: lectura pública sólo de activos; escritura restringida a admin/super_admin/editor.
- Índice único parcial: un solo redirect activo por `from_path`.
- Nuevas columnas en `page_compositions`: `canonical_override`, `robots_directive` (default `index,follow`), `sitemap_changefreq`, `sitemap_priority`, `previous_slug`. Constraints declarativos.
- Trigger `eb_page_slug_redirect_trigger`: al renombrar el slug de una composición se crea automáticamente un redirect 301 y se colapsan cadenas apuntando al slug anterior.
- RPC `eb_resolve_public_route(_path)`: resuelve un pathname a `redirect` o `composition` publicada. Colapsa hasta 5 hops, `SECURITY DEFINER`, `EXECUTE` a `anon` y `authenticated`.

### 1.2 Server functions
- `src/lib/experience-builder/eb-route-resolver.functions.ts` · `resolvePublicRoute({ path })` — envoltorio de la RPC con contrato tipado (`redirect | composition | miss`). Falla cerrada. No usada aún por rutas públicas.
- `src/lib/experience-builder/eb-redirects.functions.ts` · `listRedirects`, `createRedirect`, `deactivateRedirect`. Todas con `requireSupabaseAuth` + RLS. Detección de ciclos triviales y desactivación implícita del redirect activo previo al mismo origen.

### 1.3 Page Kind Registry (extensión)
- `PageKindDefinition.defaults` opcional: `jsonLdType`, `sitemapChangefreq`, `sitemapPriority`, `cacheControl`, `unpublishFallbackPath`.
- `resolvePageKindDefaults(kind)` con matriz por kind (Home, Marketplace, Destinos, Experiencias, Hoteles, Restaurantes, Eventos, Empresas, Productos, Landings, Micrositios, Institucionales, etc.) + fallback conservador.
- `assertRegisteredKind(kind)` — enforcement de R3.43 (toda superficie pública debe registrarse).

### 1.4 No modificado (garantía)
- `page_compositions` schema existente, Block Registry, Layout Engine, i18n overlay, Workflow, Soft Lock, Comments, Preview firmado, Studio, PagesPanel, rutas públicas.
- URLs públicas no cambian. Renderer público no cambia. Sitemap no cambia.
