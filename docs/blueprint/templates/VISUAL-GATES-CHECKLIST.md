## Visual Decision Gates (cap. 12)

- [ ] **1. Identidad** — usa paleta, tipografía y tono oficiales (sin literales fuera del Design System).
- [ ] **2. UX** — un objetivo por pantalla, jerarquía clara, descubrimiento progresivo.
- [ ] **3. Componentes** — solo componentes oficiales del Design System (sin duplicación).
- [ ] **4. Responsive** — verificado en Mobile / Tablet / Desktop.
- [ ] **5. Accesibilidad** — teclado, lector de pantalla, contraste, multilenguaje (WCAG 2.2 AA).
- [ ] **6. Rendimiento** — sin regresiones (imágenes optimizadas, carga progresiva).
- [ ] **7. CMS** — todo contenido administrable proviene del CMS (sin hardcode).
- [ ] **8. IA** — respeta gobernanza para IA (no redefine arquitectura ni publica fuera del flujo editorial).
- [ ] **9. Calidad Visual** — alineada con Canon Visual y Dirección de Arte.

## Definition of Done Visual (cap. 13)

- [ ] Cumple su objetivo funcional.
- [ ] Respeta el Canon Visual.
- [ ] Usa componentes oficiales.
- [ ] Supera QA.
- [ ] Supera los 9 Decision Gates de arriba.
- [ ] Funciona en Mobile, Tablet y Desktop.
- [ ] Mantiene accesibilidad.
- [ ] Documentación actualizada (Blueprint + reporte del Gate).
