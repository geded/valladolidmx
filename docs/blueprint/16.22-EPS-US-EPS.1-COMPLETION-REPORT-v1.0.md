# Épica E-PS · US-EPS.1 · Profile Switcher — Migración + RPC · Completion Report v1.0

**Programa:** A (Visitante) + B (Empresario) · **Carril A** · **Versión:** v2.6 → v3.0
**Fecha:** 2026-07-05 · **Dependencias:** ninguna (usa tablas existentes)

## 1. Objetivo
Habilitar el backend mínimo para que un mismo `auth.users` alterne entre modos disponibles (Viajero / Empresa / Concierge / Staff) sin cerrar sesión, sin infraestructura nueva.

## 2. Alcance entregado
### 2.1 Backend (una migración)
- **RPC `public.profile_get_available_modes()`** — SECURITY DEFINER, `GRANT authenticated`. Devuelve una fila por modo con `available`, `primary_label`, `secondary_label`, `entity_count`. Fuentes:
  - traveler → `traveler_profiles`
  - business → `business_users` (cuenta empresas distintas)
  - concierge → `concierge_profiles`
  - staff → `user_roles` in (`super_admin`,`admin`,`editor`)
- **RPC `public.profile_set_active_mode(_mode profile_mode)`** — SECURITY DEFINER, `GRANT authenticated`. Valida disponibilidad antes de escribir `profiles.active_mode`. Códigos de error canónicos: `not_authenticated` (28000), `mode_not_available` (42501).
- **Reutiliza**: enum `profile_mode` y columna `profiles.active_mode` preexistentes. Cero cambios destructivos.

### 2.2 Server functions (`src/lib/profile/switcher.functions.ts`)
- `getAvailableModes` (GET, `requireSupabaseAuth`) → `AvailableModeRow[]`.
- `setActiveMode` (POST, `requireSupabaseAuth`, Zod input) → `{ mode }`.

## 3. Product Vision Rule
- **Vender más al empresario:** desbloquea el patrón "convierte tu negocio en anfitrión" para US-EPS.3.
- **Reducir trabajo operativo:** un login, múltiples contextos.
- **Vivir mejor al turista:** cambio instantáneo entre modos sin re-autenticar.

## 4. Founder Decisions
- Freeze de infraestructura reforzado (cero tablas nuevas).
- Single Source of Truth (`profiles.active_mode` única).
- Discovery First + Workspace First respetados (el switcher vive en Workspace, no en Discovery).

## 5. Cero regresiones
- Ninguna superficie existente lee `active_mode` todavía; contrato pasivo hasta US-EPS.2.
- RPCs restringidas a `authenticated`; anon no puede ejecutar.
