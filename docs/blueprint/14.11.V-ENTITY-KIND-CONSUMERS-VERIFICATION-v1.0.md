# 14.11.V — Verificación de Consumidores de `entity_kind` v1.0

**Estado:** Completada · Pre-requisito para migración ADR 14.11 (Alternativa A)
**Fecha:** 2026-06-29
**Alcance:** Inventario exhaustivo de consumidores del enum `public.entity_kind` previo a `ALTER TYPE ... ADD VALUE 'review'`.

---

## 1. Valores actuales del enum

`country, state, tourism_region, destination, destination_zone, point_of_interest, business_category, business, product, media_asset, article, page, event, route, faq, banner, promotion` (17 valores). `review` **no** está presente.

## 2. Consumidores en la base de datos

### 2.1 Funciones (`pg_proc`)
| Función | Uso | Impacto de añadir `review` |
|---|---|---|
| `public.transition_content_status(_entity_kind, _entity_id, _to_status, _notes)` | Recibe `entity_kind` como parámetro y mapea a tabla mediante `CASE` explícito (`article→articles`, `page→pages`, `event→events`, `route→editorial_routes`, `faq→faqs`, `banner→banners`, `promotion→promotions`). Cualquier otro valor produce `RAISE EXCEPTION 'entity_kind % is not editable via this function'`. | **Seguro.** El `CASE` es allow-list explícito; recibir `review` resultará en excepción controlada idéntica a la actual para `country/state/etc.` No requiere modificación. La transición de status para reseñas continúa por `moderation.functions.ts` (Serie 14 / Etapa 5). |

No se localizaron triggers, RPC adicionales ni funciones SECURITY DEFINER que enumeren exhaustivamente los valores del enum.

### 2.2 Triggers
Búsqueda `pg_trigger`: ningún trigger referencia `entity_kind`. Todos los triggers existentes son `update_updated_at` o auditoría de roles.

### 2.3 Políticas RLS
Búsqueda `pg_policies` con `qual ILIKE '%entity_kind%' OR with_check ILIKE '%entity_kind%'`: **0 políticas**. Ninguna RLS depende del valor del enum.

### 2.4 Tablas que tipan columnas como `entity_kind`
| Tabla | Columna | Nullable | Restricción de dominio |
|---|---|---|---|
| `content_audit_log` | `entity_kind` | NOT NULL | Ninguna restricción adicional. Aceptará `review` tras `ALTER TYPE`. |
| `seo_metadata` | `entity_kind` | NOT NULL | UNIQUE `(entity_kind, entity_id, locale)`. Aceptará `review` sin colisión. |

| `reviews` | `subject_kind` (tipo `entity_kind`) | NOT NULL | Discrimina el sujeto reseñado (ej. `business`, `destination`). No es el sujeto-reseña; el agregado del enum es transparente. |

- Línea 2364: literal-union del enum. Será regenerada por el linter de tipos de Supabase.

### 3.3 Componentes UI / loaders / dashboards
Búsqueda `rg "entity_kind" src/`: solo aparecen ocurrencias en `types.ts` (generado) y en las dos server functions del CMS. **Ningún componente React, hook, loader, route ni dashboard enumera exhaustivamente los valores del enum.** Los listados del CMS Studio (Etapa 2/3) operan tabla-por-tabla y no dependen de la cardinalidad del enum.

### 3.4 Migraciones existentes
- `20260629031346_*`: creación original del enum (17 valores actuales).
- `20260629033650_*`: ampliación previa (`faq`, `banner`, `promotion`) — patrón a replicar.

## 4. Conclusión de la verificación

- **Allow-lists explícitos en BD y código** (`transition_content_status`, `TABLE_TO_ENTITY_KIND`) garantizan que la incorporación de `'review'` al enum es **estrictamente aditiva** y no altera ningún flujo existente.
- **Cero políticas RLS** dependen del enum.
- **Cero triggers** dependen del enum.
- **Cero componentes UI** enumeran los valores del enum.
- **Tipos generados** se regenerarán automáticamente tras la migración Supabase.
- **Reviews seguirán operando** con `metadata.moderation_history` hasta que la etapa correspondiente del plan 14.20 instrumente el doble-write/cutover a `content_audit_log`.

## 5. Autorización para ejecutar la migración

Se autoriza la ejecución de la migración **14.11.M — Ampliación de `entity_kind` con `review`** consistente con la Alternativa A del ADR 14.11. La migración será puramente aditiva:

```sql
ALTER TYPE public.entity_kind ADD VALUE IF NOT EXISTS 'review';
```

Sin cambios a RLS, sin uso de `SERVICE_ROLE`, sin modificación del modelo de dominio aprobado, sin alteración del Blueprint ni de las Series 12D/13/14.

## 6. Plan de contingencia

Si tras la migración se detecta cualquier consumidor no documentado que rompa por la presencia de `'review'`, se activará la **Alternativa C** del ADR 14.11 (bitácora dedicada `review_moderation_log`) y la entrada del enum quedará reservada sin uso productivo hasta resolución.
