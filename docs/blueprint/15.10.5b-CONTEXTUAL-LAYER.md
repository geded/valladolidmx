# Adenda 15.10.5b — Contextual Layer

**Estado:** entregada.
**Predecesor:** 15.10.5a · Workspace Foundations.
**Siguiente:** 15.10.5c · Migración 1:1 de superficies existentes.

## Principios de gobernanza aplicados

1. **Context is a Platform Capability** — el contexto operativo se consume vía un contrato declarativo único (`WorkspaceContextDefinition`). Queda prohibido implementar estados contextuales aislados fuera del Context Registry.
2. **Explainable by Default** — `AluxAction` lleva `rationale`, `sources`, `effect`, `reversible` y `undo` como parte del contrato; el panel los renderiza por defecto.
3. **Interaction Consistency Policy** — Gestos, Sheets, Inspector, Selection, Palette, Undo y Toasts viven en `src/components/workspace` y se comportan idénticos en toda la plataforma.
4. **State Preservation Policy** — `WorkspaceContextProvider` persiste un `ContextSnapshot` por workspace en `sessionStorage` (`vmx.workspace.ctx.<id>`): vista, selección, foco, scroll, filtros.
5. **Workspace Evolution Policy** — el núcleo del Engine sigue estable; sólo se agregan registries y providers reutilizables.

## Contratos nuevos

| Contrato | Archivo | Propósito |

| `WorkspaceContextDefinition` | `src/lib/workspace/types.ts` | Declaración de entidades, modos de selección, inspectores, quick actions y vistas por workspace. |

| `Inspector Registry` | `src/components/workspace/inspector/registry.ts` | `registerInspectorView` runtime sobre el contrato declarativo. |
| `Toast & Undo Bus` | `src/lib/workspace/toast-bus.ts` | `workspaceToast.success/error/info/promise` + stack de undo accesible a Alux. |

## Componentes nuevos (infraestructura)

- `WorkspaceContextProvider` — selección · foco · viewId · filtros · scroll + snapshot/restore.
- `SheetStackProvider` / `useSheetStack` — push/pop programático de bottom sheets (snaps `peek/half/full`).
- `EntityInspector` · `SelectionInspector` · `EmptyInspector` — vistas reutilizables del Inspector contextual.
- `useSwipeActions` · `usePullToRefresh` · `useLongPress` — capa universal de gestos.
- `AluxCopilotPanel` v2 — render Explainable by Default + toasts/undo integrados.
- `WorkspaceShell` — monta `WorkspaceContextProvider`, `SheetStackProvider` y `<Toaster/>` por defecto.
- `CommandPalette` — consume `useAvailableActions()` y expone acciones contextuales junto con navegación y comandos del workspace.

## Showcase

Ruta `/lovable/workspace-foundations` extendida con la sección **Contextual Layer**:

- Visualizador del Context Registry (entidades + quick actions).
- Selection Model (single/multi + long-press para entrar en multi).
- Gestos universales (swipe izq/der demo).
- Sheet Stack (peek/half/full y apilamiento).
- Toast & Undo bus (acción reversible + undo automático).
- Context Snapshot en vivo (Zero Context Loss).

## Criterios de aceptación cumplidos

- Build verde, sin regresiones en superficies existentes.
- Showcase renderiza en iPhone (xs), iPad portrait (md), Desktop (xl), light/dark, densidades cómoda/compacta.
- Context Registry consumido únicamente vía contrato declarativo.
- Inspector vistas (Entity/Selection/Empty) operan sin código del workspace.
- Sheet Stack push/pop/clear con 3 snaps.
- Toast/Undo ejecuta acción + deshace estado.
- Alux v2 muestra rationale + sources + effect + undo cuando aplica.
- Snapshot de contexto persiste y restaura.

## Riesgos cubiertos

- `sessionStorage` SSR-safe (acceso sólo dentro de `useEffect`).
- Gestos sin bloquear scroll nativo (`touch-action: pan-y`).
- Stack de undo acotado a 5 entradas.
- Snapshot truncable; `sources` y `filters` opacos para evitar crecimiento incontrolado.
