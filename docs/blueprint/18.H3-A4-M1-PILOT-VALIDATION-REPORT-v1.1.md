## 8. Fallo real de derivación y camino de failure (cerrado)

**Diagnóstico del gap v1.0:** el CLI v1.0 lanzaba la excepción de `sharp().metadata()` DENTRO del `deriveOne`, pero el `catch` externo en `main()` sólo la anotaba en el reporte y **no** marcaba el asset como `failed`. Además, `process.exit(1)` sólo se disparaba si `totals.failed > 0` (contador de variantes fallidas), no si el asset entero abortaba antes de iterar la matriz. Resultado: `exit=0` engañoso.

**Corrección aditiva aplicada** (`scripts/media-pipeline/derive.mjs` · sin cambios de contrato):

1. Envolturas `try/catch` alrededor de la descarga, `sharp.metadata()` y toda la fase de preparación.
2. Función `markAssetFailed(msg)`: actualiza `media_assets.pipeline_status='failed'`, `pipeline_engine='sharp'`, `pipeline_processed_at=now()`, `pipeline_last_error='M1: …'` truncado a 500 chars.
3. `main()` captura excepciones que escapan de `deriveOne` y también marca el asset como failed.
4. Nuevo contador `asset_failed` en el reporte y en la línea final del CLI.
5. Exit code: `1` si `totals.failed > 0` **o** `totals.asset_failed > 0`.

**Prueba controlada** con asset válido en DB (`09536ae3-f865-4ee3-98d5-1dddd41966dd`) y binario no-imagen:

| Verificación | Esperado | Obtenido |
|--------------|----------|----------|

## 10. Rediseño coherente de idempotencia y versionado (propuesta — NO aplicada)

### 10.1 · Diagnóstico del enfoque v1.0

La propuesta v1.0 combinaba:

- `UNIQUE (asset_id, format, width, engine)` (activa desde M0), que impone **exactamente una fila por dimensión × engine**, y
- una `variant_key` **más amplia** (incluyendo `engine_version` y `quality`), y
- una marca `status='superseded'` para versiones anteriores.

Contradicción real: no se pueden almacenar simultáneamente dos filas con distinto `engine_version` para la misma `(asset_id, format, width, engine)` mientras la UNIQUE actual siga vigente. La `variant_key` extra queda inerte y el `superseded` no puede coexistir con el activo.

### 10.2 · Cinco preocupaciones a separar

| Preocupación | Vehículo dedicado |
|--------------|-------------------|
| Identidad reproducible de cada derivación | `variant_key` determinística (columna generada, ver §10.3) |
| Variante activa por dimensión × engine (para servir) | flag `is_current` **parcial** único |
| Historial completo de versiones | filas coexistentes en la misma tabla, sin borrar |
| Prevención de duplicados exactos | `UNIQUE(variant_key)` |
| Limpieza segura de huérfanos en Storage | job programable + política de retención, con doble borrado (DB → Storage) |

### 10.3 · Migración aditiva (SQL — pendiente de autorización)

```sql
-- Aditiva, sin borrar la UNIQUE actual EN EL PRIMER PASO.
-- Se corre en 4 pasos con validación entre cada uno.

-- PASO 1 · Estructura de versionado.
ALTER TABLE public.media_asset_variants
  ADD COLUMN engine_version    text,
  ADD COLUMN quality_effective int,
  ADD COLUMN is_current        boolean NOT NULL DEFAULT true,
  ADD COLUMN superseded_at     timestamptz,
  ADD COLUMN superseded_by     uuid REFERENCES public.media_asset_variants(id) ON DELETE SET NULL,
  ADD COLUMN variant_key       text
    GENERATED ALWAYS AS (
      asset_id::text || ':' || format || ':' || width::text || ':' || engine ||
      ':' || COALESCE(engine_version,'-') ||
      ':' || COALESCE(quality_effective::text,'-') ||
      ':' || COALESCE(usage_context,'generic')
    ) STORED;

-- PASO 2 · Backfill idempotente desde metadata jsonb.
UPDATE public.media_asset_variants
   SET engine_version    = COALESCE(engine_version, metadata->>'engine_version'),
       quality_effective = COALESCE(quality_effective, NULLIF(metadata->>'quality','')::int)
 WHERE engine_version IS NULL OR quality_effective IS NULL;

-- PASO 3 · Índices dedicados: identidad reproducible + variante activa.
CREATE UNIQUE INDEX media_asset_variants_variant_key_ux
  ON public.media_asset_variants(variant_key);

CREATE UNIQUE INDEX media_asset_variants_current_ux
  ON public.media_asset_variants(asset_id, format, width, engine, COALESCE(usage_context,'generic'))
  WHERE is_current = true AND status = 'ready';

-- PASO 4 · Retiro de la UNIQUE antigua (destructiva: se agenda como
-- migración separada, tras un despliegue estable con los índices nuevos).
-- ALTER TABLE public.media_asset_variants
--   DROP CONSTRAINT media_asset_variants_asset_id_format_width_engine_key;
```

### 10.4 · Cambios de flujo en el CLI (sin ejecutar)

Cuando cambia `engine_version` o `quality_effective` (o cualquier campo del `variant_key`):

1. Se INSERTA una nueva fila con `is_current=true`.
2. Un trigger `AFTER INSERT` marca la fila anterior con `is_current=false, superseded_at=now(), superseded_by=<nuevo id>`.
3. El path en Storage sigue incluyendo `engine_version`, por lo que la fila antigua **conserva** su binario hasta la política de retención.

### 10.5 · Limpieza segura de huérfanos

Job programable `media-pipeline · orphan-sweeper` (agendable por cron post-M2):

1. Lista objetos de `media-derived/{assetId}/**` cuya combinación `{engine, engine_version, format, width, usage_context}` **no** aparezca en ninguna fila `media_asset_variants` con `is_current=true`.
2. Retención: **≥30 días** desde `superseded_at`.
3. Borrado en dos fases: (a) `DELETE FROM media_asset_variants WHERE is_current=false AND superseded_at < now() - interval '30 days'`; (b) `storage.remove()` de los paths físicos. Nunca al revés.
4. Reporte JSON con conteo por asset y bytes liberados. `pipeline_status` del asset no cambia.

### 10.6 · Rollback (pendiente de autorización)

```sql
DROP INDEX IF EXISTS public.media_asset_variants_current_ux;
DROP INDEX IF EXISTS public.media_asset_variants_variant_key_ux;
ALTER TABLE public.media_asset_variants
  DROP COLUMN IF EXISTS variant_key,
  DROP COLUMN IF EXISTS superseded_by,
  DROP COLUMN IF EXISTS superseded_at,
  DROP COLUMN IF EXISTS is_current,
  DROP COLUMN IF EXISTS quality_effective,
  DROP COLUMN IF EXISTS engine_version;
-- La UNIQUE original nunca se retira si no se completó el PASO 4.
```

**No se aplica nada de §10 con este reporte.** Se somete como propuesta consolidada para M2.
