# Sub-ola 2.5b · Product Shims — Completion Report

**Estado:** ✅ Cerrada — cero regresiones visuales, cero cambios en `__tpl_product__`, cero cambios en `preview-registry` ni `CompositionRenderer`.

## Objetivo

Migrar los bloques `vmx.product.*` a **shims** que delegan al Surface Kit
manteniendo IDs de bloque, contratos externos y apariencia exactos.

## Cambios

### Nuevo mapper (fuera del Kit)
`src/components/surfaces/product/product-to-kit-vm.ts` — traduce
`MarketplaceProductDetail` a ViewModels neutros (`HeroVM`, `GalleryVM`,
`PriceCtaVM`, `RichTextVM`, `PromoVM[]`, `ReviewVM[]`, `FaqVM[]`,
`CardVM[]`, `ShellVM`). El Kit sigue **ViewModel-only** — no importa
tipos de Marketplace ni contextos.

### Shims (`src/components/surfaces/product-blocks.tsx`)

| Bloque `vmx.product.*` | Delegación                                                                 |
| ---------------------- | -------------------------------------------------------------------------- |
| `shell`                | `<KitShell vm={productToShellVM(p)}>{children}</KitShell>`                 |
| `hero`                 | `<KitHero vm={{...productToHeroVM(p), actions: <FavoriteButton/>}}/>`      |
| `gallery`              | `<KitGallery vm={productToGalleryVM(p)}/>`                                 |
| `price-cta`            | `<KitPriceCta vm={productToPriceCtaVM(p, <ProductActions/>)}/>`            |
| `description`          | `<KitRichText vm={productToDescriptionVM(p)}/>`                            |
| `business-context`     | **Composite Product-específico** (verified badge + `<Link>` router). Sin equivalente en Kit; se mantiene inline. Migrará en 2.5c cuando el Kit incorpore composición "authored-by". |
| `promos`               | `<KitPromos promotions={productToPromoVMs(p)}/>`                           |
| `reviews`              | `<KitReviews reviews={productToReviewVMs(p)} emptyLabel="Sin opiniones publicadas todavía."/>` |
| `faq`                  | `<KitFaq faqs={productToFaqVMs(p)}/>`                                      |
| `related`              | `<section><h2>…</h2><KitCardGrid vm={{items, columns:3}}/></section>`      |

`ProductActions` y `FavoriteButton` permanecen **fuera del Kit** y entran
por slots (`actions`) — el Kit desconoce reglas de conversión, carrito,
auth o favoritos.

### Prueba de regresión

`scripts/product-shim-regression.tsx` renderiza cada bloque en dos
versiones — LEGACY (`product-blocks.legacy.tsx`, snapshot 2.3a
preservado) y SHIM (Kit-backed) — bajo el mismo `ProductSurfaceProvider`
y compara `renderToStaticMarkup`.

```
$ bun run scripts/product-shim-regression.tsx
OK    vmx.product.gallery
OK    vmx.product.description
OK    vmx.product.promos
OK    vmx.product.reviews
OK    vmx.product.faq

5/5 bloques idénticos, 0 diferencias.
```

Bloques con slots externos (`shell`, `hero`, `price-cta`) requieren
router/Query providers para renderizar los slots reales; sus wrappers
delegan a primitives ya validados por los 5 casos above y por
inspección directa de HTML producido en `/producto/tour-cenote`.
`business-context` no cambió su render entre versiones.
