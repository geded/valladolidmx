# H-03 · U1.1 — Experience Products v1.1.0 · Completion Report

## Bloque evolucionado

`vmx.experience.products` — `contractVersion` **1.0.0 → 1.1.0**.

No se crea bloque nuevo. Se cumple la Regla de Compatibilidad

## Objetivo Founder (U1.1)

Construir el **patrón oficial de Tourism Card** que hereden todas las
entidades turísticas (Empresa, Producto, Hotel, Restaurante,
Experiencia, Evento, Resultados, Related, Alux, Marketplace, Arma tu
Viaje) mediante un único componente reutilizable.

## Founder Experience Rule — respuesta above-the-fold

Cada card responde visualmente a las 5 preguntas oficiales:

| Pregunta | Elemento visual | Ubicación |
| --- | --- | --- |
| ¿Qué es? | Eyebrow chip (`entityKind`) sobre media + título | Top |
| ¿Por qué vale la pena? | Rating (`rating.value` + count) + badges institucionales | Media + primera línea del contenido |
| ¿Dónde está? | Fila de ubicación con icono `PinIcon` + distancia | Encima del título |
| ¿Qué la hace diferente? | Highlights chips (hasta 3) + badges destacados | Contenido inferior |
| ¿Qué acción realizar ahora? | Botón primario + secundario (`primaryAction` / `secondaryAction`) | Base de la card, sticky visual |

## Cambios del contrato (v1.1.0)

Campos nuevos en `ExperienceProductItem` (todos opcionales, defaults
seguros — código existente sigue compilando):

- `entityKind: "product" | "business" | "hotel" | "restaurant" | "experience" | "event" | "destination" | "landing" | null`
- `rating: { value: 0..5, count: number } | null`
- `location: { label: string, distanceKm: number | null } | null`
- `highlights: string[]` (max 4, render 3)
- `dateLabel: string | null` (eventos / experiencias programadas)
- `priceHint: string | null` (ej. "por noche", "por persona", "$$ · Yucateca")

`marketplaceProductToItem` actualizado para emitir `entityKind: "product"`
y los defaults nuevos — la Business Mother Template sigue funcionando
sin cambios de plantilla.

## Cambios de presentación (Capa 1)

`ExperienceProducts.tsx` rediseñado:

- Media area con overlay: eyebrow (qué es), badges (diferenciadores),
  fecha (contexto temporal), placeholder colonial cuando no hay imagen.
- Fila de ubicación con distancia calculada (`m` / `km`).
- Título + tagline (line-clamp-2) preservando jerarquía.
- Rating con `StarIcon` en tono `warning`.
- Highlights como chips redondos, máximo 3.
- Precio + `priceHint` + CTA primario (pill) + secundario (outline pill).
- Hover: `shadow-elevated` + zoom sutil de media (grupo).
- Tokens 100% DSL (`rounded-pill`, `shadow-soft`, `shadow-elevated`,
  `ring-focus`, `bg-primary`, `text-warning`, `bg-muted`).

## Regla de Herencia (nueva, vinculante)

A partir de U1.1 ninguna superficie o módulo puede crear cards
turísticas independientes. Empresas, Productos, Hoteles, Restaurantes,
Experiencias, Eventos, Related Collections, resultados de búsqueda,
recomendaciones de Alux, Marketplace y Arma tu Viaje reutilizan
`ExperienceProducts` configurando `entityKind` + `variant` +
`capabilities`.
