## 1. Entregable

**Bloque oficial del Experience Builder** (`vmx.discovery.navigator`)
registrado en la Block Library, con contrato editable en Studio,
preview neutral y renderer de producción. Reutilizable en cualquier
superficie pública actual y futura sin código nuevo.

El bloque se integra automáticamente en la Plantilla Destino:
`DestinationSurface` reemplaza el `<aside>` estático "Explora más del
destino" por el nuevo bloque, manteniendo paridad de accesos y sumando
capacidades (categorías dinámicas + conteos reales + CTA universal).

## 3. Contrato del bloque (Experience Builder)

```ts
{
  type: "vmx.discovery.navigator",
  version: "1.0.0",
  category: "static",
  schema: {
    title, variant (panel|list|grid), showCounts,
    ctaLabel, ctaHref, emptyLabel,
    scope (auto|destination|region),
    manualDestinationSlug, manualRegionSlug
  }
}
```

**Contrato de datos (server function)**

```ts
getDiscoveryNavigator({ destinationSlug?, regionSlug? }) → {
  scope:      { kind, slug, label },
  categories: DiscoveryCategoryItem[],   // slug, label, iconKey, count, href
  extensions: []                          // reservado v2+
}
```

**Preparado para evolucionar** sin romper compatibilidad hacia:
promociones del destino, eventos próximos, experiencias destacadas,
rutas sugeridas, negocios patrocinados, recomendaciones de Alux,
campañas temporales, contenido editorial, módulos IA — todo cabe en
`extensions[]` sin renombrar campos actuales.

## 5. Comparativa antes / después

| Aspecto | Antes | Después |
|---|---|---|
| Componente | `<aside>` estático hardcodeado en `DestinationSurface` | Bloque oficial del EB, reutilizable |
| Accesos | 4 fijos (Hoteles, Restaurantes, Experiencias, Marketplace) | N dinámicos (todos los con datos) + Marketplace CTA |
| Conteos | Ninguno | Reales, calculados server-side por destino |
| Iconografía | Sólo texto | Lucide + fondo circular semántico |
| Editabilidad | Requiere PR de código | Editable en Studio (título, variante, CTA, alcance) |
| Reutilización | Sólo `DestinationSurface` | Cualquier superficie pública (Destinos, Regiones, Micrositios, Landings, Marketplace contextual) |
| Evolución | Rediseño para añadir promociones/eventos | Slots `extensions[]` sin romper contrato |
| Contexto territorial | Se preservaba (search=destino) | Igual — sumado a persistencia del Context Engine |
| Fallback sin JS | Sin fallback | `<noscript>` con los 4 accesos originales |

**Ninguna capacidad se perdió.** La UX mejora (iconografía + conteos),
la arquitectura mejora (bloque oficial reutilizable) y la evolución
queda abierta sin romper compatibilidad.
