## Ajustes de arquitectura incorporados

1. **Multiempresa desde el origen** — enum `eb_scope = global | tenant | marketplace` aplicado a *themes / templates / sections / pages / variants*. Constraint `scope_tenant_chk` impide combinaciones inválidas y permite agregar nuevos alcances sin migrar datos.
2. **Templates como producto** — `eb_templates` contempla `config`, `pages`, `navigation`, `theme_id`, `assets`, `variables`, `recommended_blocks`, `seo_defaults`, `version`, `is_listed`. Listo para instalación 1-click por empresa o publicación en futura marketplace.
3. **Landing Pages preparadas para conversión** — `eb_pages` incluye desde el origen `seo`, `open_graph`, `schema_org`, `marketing` (Pixel Meta, GTM, GA4, otros), `conversion`, `experiments` (A/B futuro) y `cache`.
4. **Theme Manager** — `eb_themes.tokens` modela colores, tipografías, radios, espaciados, sombras, iconografía, modo claro/oscuro y variables CSS bajo un único jsonb extensible.
5. **Responsive declarativo** — el árbol vive en `tree`; las variantes responsivas se aplican vía `responsive.overridable_fields` del Block Contract + `eb_variants.overrides` (sin duplicar árboles por dispositivo).
6. **Preview = producción** — `eb_preview_tokens` y `eb_preview_resolve(token)` devuelven exactamente la composición a renderizar; el front Fase 3 usará el mismo renderer público para garantizar paridad.
7. **Versionado completo** — `eb_page_versions` / `eb_section_versions` conservan autor (`created_by`), fecha (`created_at`), comentario (`note`), versión origen (`parent_version_id`) y permiten restauración (`eb_page_restore_version`).
8. **Auditoría** — `eb_audit_log` registra `create | update | clone | publish | unpublish | restore | delete | preview_issue` con actor, scope y tenant, alimentado automáticamente por cada RPC.
9. **Performance / cache** — `eb_pages.cache` declara estrategia y TTL por página; `published_version_id` permite servir snapshots inmutables sin recomponer el árbol en cada render.
10. **Demo de Fase 1** — incluida abajo (SQL reproducible) hasta que la Fase 2 entregue UI.

## Inventario de tablas

| Tabla | Propósito | Scope | RLS |
|---|---|---|---|
| `eb_themes` | Tokens de tema (colores/tipografía/radios/sombras/dark+light/CSS vars) | global/tenant/marketplace | lectura pública; escritura por editor del scope |
| `eb_templates` | Template como producto instalable | global/tenant/marketplace | lectura: `published` o editor; escritura por editor |
| `eb_sections` | Partials reutilizables | global/tenant/marketplace | igual a templates |
| `eb_section_versions` | Historial de secciones | — | lectura solo a editores del section |
| `eb_pages` | Landings / institucionales / campañas / secciones | global/tenant/marketplace | lectura: `published` o editor |
| `eb_page_versions` | Historial de páginas (autor/fecha/nota/parent) | — | lectura solo a editores de la page |
| `eb_variants` | Composition Variants (A/B, idioma, segmento) | — | lectura si `is_active` o editor |
| `eb_preview_tokens` | Tokens cortos firmados para vista previa | — | lectura: creador o editor de la page |
| `eb_audit_log` | Bitácora editorial | — | Founder/Admin global; tenant solo su empresa |

## Inventario de RPCs (`SECURITY DEFINER`, `search_path=public`, `EXECUTE` solo a `authenticated`)

- `eb_can_edit_scope(scope, tenant_id) → boolean` — helper de autorización (super_admin/admin global, business_users para tenant).
- `eb_theme_upsert(payload)` · `eb_template_upsert(payload)` · `eb_template_clone(id, scope, tenant, slug)`
- `eb_section_upsert(payload)` · `eb_section_publish(id, note)`
- `eb_page_upsert(payload)` · `eb_page_save_version(id, note)` · `eb_page_publish(id, note)` · `eb_page_unpublish(id)` · `eb_page_restore_version(id, version_id)` · `eb_page_delete(id)`
- `eb_variant_upsert(payload)`
- `eb_preview_token_issue(page, version, variant, ttl_min) → token` · `eb_preview_resolve(token) → jsonb`
- `eb__audit(...)` (interno; emite registros a `eb_audit_log`)

Todas verifican `eb_can_edit_scope` antes de mutar y registran auditoría. Cualquier intento sin permiso devuelve `42501 / forbidden`.

## Pruebas RLS (resumen)

| Caso | Resultado esperado | Cómo se garantiza |
|---|---|---|
| Anónimo lee `eb_pages` no publicada | denegado | policy `eb_pages_read_public` exige `status='published'` |
| Empresa A intenta editar página de empresa B | denegado | `eb_can_edit_scope` valida `business_users` |
| Empresa edita su página | permitido | `business_users.business_id = tenant_id` |
| Empresa publica template global | denegado | `eb_can_edit_scope('global', NULL)` exige admin/super_admin |
| Admin/Founder hace clone global → tenant | permitido | rol admin/super_admin |
| Token de preview expirado | resuelve `NULL` | `expires_at < now()` en `eb_preview_resolve` |
| Variante `is_active=false` no visible al público | oculta | policy `eb_variants_read_public` |

## Demo funcional de la Fase 1 (sin UI)

Reproducible end-to-end vía RPCs (la Fase 2 envolverá esto en Studio):

```sql
-- 1. Empresa crea su tema
select public.eb_theme_upsert(jsonb_build_object(
  'scope','tenant','tenant_id', :empresa_id,
  'name','Marca Maya', 'slug','marca-maya',
  'tokens', jsonb_build_object(
    'colors', jsonb_build_object('primary','#0E5E5C','accent','#E0A14F'),
    'typography', jsonb_build_object('heading','Fraunces','body','Inter'),
    'radii', jsonb_build_object('md','12px'),
    'mode', 'auto'
  )
));

-- 2. Empresa crea una landing con SEO + GA4
select public.eb_page_upsert(jsonb_build_object(
  'scope','tenant','tenant_id', :empresa_id,
  'kind','landing', 'name','Ruta Cenotes', 'slug','ruta-cenotes',
  'theme_id', :theme_id,
  'tree', '{"root":{"children":[]}}'::jsonb,
  'seo', jsonb_build_object('title','Ruta Cenotes — Yucatán','description','Vive 4 cenotes en un día.'),
  'open_graph', jsonb_build_object('og:title','Ruta Cenotes','og:type','website'),
  'marketing', jsonb_build_object('ga4','G-XXXXXX','meta_pixel','1234567890')
));

-- 3. Guarda versión y publica
select public.eb_page_save_version(:page_id, 'borrador inicial');
select public.eb_page_publish(:page_id, 'go-live v1');

-- 4. Emite preview token (válido 30 min)
select public.eb_preview_token_issue(:page_id, NULL, NULL, 30);
select public.eb_preview_resolve(:token);

-- 5. Restaurar a versión anterior
select public.eb_page_restore_version(:page_id, :prev_version_id);

-- 6. Auditoría
select action, entity, created_at from public.eb_audit_log
 where entity='page' and entity_id = :page_id order by created_at;
```

## Deuda técnica / observaciones

- Las variantes hoy se modelan; el resolver SSR llega en Fase 3.
