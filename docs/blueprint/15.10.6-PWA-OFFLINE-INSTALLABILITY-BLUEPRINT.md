# 15.10.6 — PWA Offline & Installability · Blueprint

## Objetivo
Habilitar capacidades de ejecución PWA sobre la experiencia ya
unificada (Discovery + Workspace) sin modificar su arquitectura.
Implementación exclusivamente en la capa de runtime del navegador
(Service Worker + manifest + estrategias de caché).

## Restricciones de gobernanza

- `Discovery Architecture Freeze` — no se modifica Discovery.
- `Workspace First Policy` — no se modifica Workspace Engine.
- `Single Public Component Policy` — sin nuevos shells o variantes UI.
- `Performance First` — no degradar SSR, SEO ni CWV.
- Cumplimiento estricto de la **PWA skill de Lovable**:
  - `vite-plugin-pwa` con estrategia `generateSW`.
  - Wrapper único de registro (`src/pwa/register-sw.ts`).
  - `devOptions.enabled = false`, `injectRegister: null`,
    `registerType: "autoUpdate"`.
  - SW path `/sw.js`.
  - `NetworkFirst` para navegaciones HTML.
  - `CacheFirst` sólo para assets hasheados same-origin.
  - Exclusión de `/~oauth` del navigation fallback.
  - Bloqueo de registro en dev / iframe / preview de Lovable y
    soporte de kill-switch `?sw=off`.

## Plan por sub-adendas

### 15.10.6.1 — Service Worker base + Installability completa
- Adopción de `vite-plugin-pwa` (`generateSW`).
- Wrapper `register-sw.ts` activado para registro real en producción.
- Precache del shell mínimo (HTML root + assets hasheados emitidos
  por Vite).
- Estrategias:
  - HTML: `NetworkFirst` (timeout corto, fallback a precache).
  - JS/CSS hasheados: `CacheFirst`.
  - Imágenes locales: `StaleWhileRevalidate`.
  - Fuentes Google: `StaleWhileRevalidate` con expiración.
- Manifest revisado (íconos PWA reales, screenshots, shortcuts).
- Auto-update seguro (`registerType: "autoUpdate"` + skipWaiting
  controlado).
- Auditoría Lighthouse PWA + Performance.

### 15.10.6.2 — Offline First & Estrategias de caché extendidas
- Página de fallback offline (`/offline`) servida por el SW.
- Estrategias finas por dominio funcional (Discovery vs Workspace).
- Exclusión explícita de superficies con datos sensibles
  (Concierge, Pagos, Cuenta) del precache.
- Estrategia diferenciada para datos read-only públicos
  (Marketplace listing) vs autenticados.

### 15.10.6.3 — Cola de sincronización & Background Sync
- Sync Queue persistente (IndexedDB) para mutaciones diferidas.
- Background Sync API (con fallback por reintentos en foreground).
- Reintentos exponenciales + UX de "pendiente de sincronizar".
- Integración exclusiva con server functions ya existentes (sin
  crear endpoints nuevos).

### 15.10.6.4 — Push Notifications
- Service Worker dedicado para mensajería (no app-shell) conforme
  a la PWA skill.
- Gestión de permisos (opt-in explícito desde superficies Workspace
  relevantes: Concierge, Portal, Cuenta).
- Integración con el módulo de notificaciones ya existente.
- Política de payloads y deep-linking a rutas internas.

### 15.10.6.5 — Versionado seguro & gestión de activos
- Estrategia de versión del SW (autoUpdate + prompt no bloqueante).
- Limpieza determinística de caches obsoletos (allowlist por
  precache manifest).
- Validación post-deploy contra "white screen tras deploy".
- Documentación de rollback (kill-switch SW y `?sw=off`).

### 15.10.6.6 — Validación final & Completion Report
- Lighthouse: PWA Installable + Best Practices + SEO.
- Validación visual y funcional en iPhone, iPad y Desktop.
- Pruebas de actualización entre versiones.
- Pruebas de cold start offline.
- Reporte ejecutivo de cierre de la Serie 15.10.6.

## Criterios de cierre por sub-adenda
- Typecheck en verde.
- Build exitoso.
- Cero regresiones funcionales/visuales/SEO.
- Rollback documentado.
- Auditoría Lighthouse adjunta.
- Validación en iPhone / iPad / Desktop.

## Rollback global de la serie
1. Revertir `vite.config.ts` (remoción del plugin PWA).
2. Reemplazar `/sw.js` por el kill-switch oficial de la PWA skill
   durante un ciclo de release.
3. Volver `register-sw.ts` al modo inerte (Fase 0 actual).
4. Restaurar el manifest a su versión previa si fue ampliado.
