# U-VISUAL · V1 — Destination Microsite Parity · Completion Report v1.0

**Épica:** U-VISUAL · Cierre Turístico Integral
**Sub-ola:** V1 · Destination Microsite Parity
**Carril:** A · Producto
**Estado:** Implementado. Pendiente validación visual Founder.

## Alcance ejecutado

1. **Tourist Hero cinematic en micrositios de destino**
   `destinationToHeroDTO` emite ahora `variant: cinematic` cuando el
   destino tiene `hero_url`, con:
   - `eyebrowStyle: "script"` (identidad colonial),
   - `mediaSlides: [heroUrl]` (preparado para carrusel V1.1),
   - `overlapHeader: true`,
   - CTA primario **"Explorar el destino"** → ancla `#descubre`,
   - CTA secundario **"Ver en el mapa"** → ancla `#mapa` (V4).
   Sin `hero_url`, fallback `editorial` retrocompatible.
2. **Institutional Badges en micrositios de destino**
   Nuevo adapter `destinationToBadgeItems(input)` deriva distintivos
   desde el registry oficial (jamás hardcodeados):
   - `pueblo-magico` sólo para `valladolid` / `izamal` / `espita`
     (autorización del registry),
   - `oriente-maya` para todo destino de la región,
   - `despierta-en-valladolid` sólo cuando aplica.
   `DestinationSurface` renderiza `InstitutionalBadgesBlock`
   inmediatamente bajo el Hero con `variant: soft`, `layout: strip`,
   `mobileVisibleMax: 3`.

## Compatibilidad y reglas

- **Tourist Hero Policy:** ✅ evolución vía `variant` + `capabilities`
  del contrato oficial `vmx.experience.hero@1.1.0`. Sin heros paralelos.
- **Institutional Badges Rule:** ✅ contenido derivado del registry, no
  hardcodeado en la superficie.
- **Founder Design Principle:** ✅ identidad territorial + distintivos
  institucionales presentes above-the-fold.
- **Safe Visual Migration Rule:** ✅ único punto de cambio visible,
  fallback retrocompatible para destinos sin `hero_url`.
- **Evolutionary Compatibility Rule:** ✅ cero bloques nuevos.
