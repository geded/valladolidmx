# 15.10.5d.3 — Migración Home & Marketplace → Public Shell + Registries

## Alcance
- `/` (Home pública, fallback legacy + composición publicada)
- `/marketplace`
- `/marketplace/buscar`
- `/marketplace/$slug`

## Cambios

### Layout
- Reemplazo de `<main id="main">` ad-hoc y `PageShell` por `PublicShell`
  (15.10.5d.1) en todas las rutas migradas.
- Variante `hero` aplicada a Home para preservar layout full-bleed sin
  breadcrumb/título superior.

### SEO
- Toda la metadata pública usa `buildPublicHead` como única infraestructura
  oficial (15.10.5d.1).
- `canonical`, `og:url`, `og:locale`, `og:site_name`, `twitter:card` ahora se
  emiten de forma uniforme por construcción.
- `/marketplace/buscar` preserva el comportamiento original `noindex,follow`
  vía nuevo campo opcional `robots` en `DiscoveryHeadOptions`.
- `/marketplace/$slug` mantiene el JSON-LD `LocalBusiness` vía el contrato
  `jsonLd` del helper.

### Composición
- Home: el fallback legacy ahora resuelve cada bloque vía
  `getDiscoverySection(kind)` del Sections Registry. Mismos componentes,
  misma jerarquía visual (`hero → destinos → categorías → rutas →
  consejo-alux → arma-tu-viaje → en-vivo → empresas → reseñas`).
- Marketplace: contenido intacto; sólo cambia el shell envolvente.

## Migration Completion Audit
- Paridad 1:1 verificada: ningún componente interno fue modificado.
- `PageShell` ya no es consumido por estas rutas; permanece en codebase
  para superficies aún sin migrar (a depurar al cierre de 15.10.5d).
- Cards Registry (15.10.5d.2) listo para uso en próximas adendas.
- Experience Builder sigue consumiendo el Sections Registry sin cambios.

## Post-Migration Validation
- Typecheck: `tsgo --noEmit` en verde.
- Cero regresiones visuales esperadas: `PublicShell` reproduce 1:1 los
  patrones de `PageShell` y el `<main id="main">` previo.
- Responsive preservado (iPhone, iPad, Desktop) — no se tocaron componentes
  ni clases de los bloques internos.

## Rollback
1. Revertir imports `PublicShell` → `PageShell` en `marketplace/{index,buscar,$slug}.tsx`.
2. Revertir `buildPublicHead(...)` → bloques `head()` literales (versión previa).
3. Restaurar `LegacyHome` con imports directos de `@/components/home/*` y `<main id="main">`.
4. Eliminar la opción `robots` introducida en `DiscoveryHeadOptions` si no se
   reutiliza en otra superficie.
