# 18 · H3·A3.b · Media Intelligence Integration — Completion Report v1.0

> "IA propone. Humano decide. El resolver decide qué se muestra."

## 0 · Behavioral Change Statement

Antes: los renderizadores leían `media_assets.alt_text` directamente. Sin
idioma, sin traducciones, sin política de prioridad humano/IA.

Después: **ningún componente público** lee `alt_text`. Toda resolución
pasa por `resolveMediaAlt()`, la única fuente oficial de idioma,
prioridad humano/IA, fallback y traducciones.

## 1 · Founder Policy registrada

> A partir de H3·A3.b, ningún componente público puede leer `alt_text`.
> El resolver `resolveMediaAlt()` es la única fuente oficial para
> idioma, prioridad humano/IA, fallback, traducciones y futuras reglas
> de negocio.

Enforcement: `scripts/assert-resolve-media-alt.sh` (guard). Excepciones
documentadas: `src/lib/**`, `src/components/cms/**`,
`src/routes/_authenticated/cms/**`, `src/routes/_authenticated/portal/**`,
`src/components/portal/**`, tipos generados (`src/integrations/supabase/**`).

## 2 · Arquitectura definitiva

```text
┌──────────────────────────────────────────────────────────────┐
│ media_assets                                                 │
│  · alt_text            (humano canónico · idioma primario)   │
│  · alt_text_ai         (propuesta IA · idioma primario)      │
│  · alt_text_source     none|ai_pending|ai|human              │
│  · review_state        unreviewed|ai_suggested|approved|…    │
│  · title, entity_kind, usage_context, intelligence(jsonb)    │
└──────────────────────────────────────────────────────────────┘
          │ 1:N locale
          ▼
┌──────────────────────────────────────────────────────────────┐
│ media_asset_translations (locale ∈ es,en,fr,de,it,pt)        │
│  · alt_text, alt_text_ai, title, caption, description        │
│  · source, review_state, intelligence(jsonb)                 │
└──────────────────────────────────────────────────────────────┘

          Public / SSR path
          ─────────────────
   marketplace-reads  ─┐
   studio-media       ─┼─►  join media + translations  ─►  resolveMediaAlt(media, {locale, fallback})  ─►  DTO.alt (string)
   *-media reads      ─┘                                              │
                                                                      ▼
                                                        <img alt={dto.alt} …/>

          Editorial path (CMS)
          ────────────────────
   /cms/media  ──►  suggestMediaAlt (por locale)   ──►  columnas *_ai / traducciones
                     saveMediaMetadata (humano)    ──►  columnas canónicas
                     approveMediaTranslation      ──►  review_state = approved
                     suggestMediaAltBatch          ──►  concurrencia 3, ≤25 media
```

## 3 · Diagrama del flujo de resolución

```text
resolveMediaAlt(media, { locale, fallback })

 1. Traducción locale · source=human           → alt_text
 2. Traducción locale · review_state=approved  → alt_text | alt_text_ai
 3. Traducción locale · IA pendiente           → alt_text_ai
 4. Traducción es (default) · source=human     → alt_text
 5. Traducción es (default) · approved         → alt_text | alt_text_ai
 6. Primario · alt_text_source=human           → alt_text
 7. Primario · review_state=approved           → alt_text | alt_text_ai
 8. Primario · alt_text (cualquier origen)
 9. Primario · alt_text_ai (última red)
10. media.title
11. fallback provisto por el llamador
12. "" (accesible por defecto para imágenes decorativas)
```

Determinismo garantizado: sin llamadas externas, sin fecha, sin
aleatoriedad. Función pura, testable, apta para SSR y edge.

## 4 · Matriz completa de fallbacks

| Nivel | Fuente                              | Cuándo aplica                                    | Calidad devuelta      |
|------:|-------------------------------------|-------------------------------------------------|-----------------------|
| 1     | Traducción locale (humano)          | El editor guardó ALT humano en ese idioma       | `human_locale`        |
| 2     | Traducción locale (IA aprobada)     | IA propuso, editor aprobó                        | `ai_locale_approved`  |
| 3     | Traducción locale (IA pendiente)    | IA propuso, no revisado                          | `ai_locale_pending`   |
| 4     | Traducción `es` (humano)            | Idioma solicitado ≠ es y no hay traducción       | `human_default`       |
| 5     | Traducción `es` (IA aprobada)       | Idem, sin humano en default                      | `ai_default`          |
| 6     | Primario humano                     | `media_assets.alt_text_source='human'`           | `human_primary`       |
| 7     | Primario IA aprobada                | `review_state='approved'`                        | `ai_primary`          |
| 8-9   | Primario crudo                      | Cualquier `alt_text` / `alt_text_ai`             | `raw_primary`         |
| 10    | `media.title`                       | Sin ALT alguno, hay título                       | `title`               |
| 11    | Fallback del llamador               | Nombre de entidad (producto, negocio, destino)   | `fallback`            |
| 12    | `""`                                | Ninguna evidencia — imagen decorativa            | `empty`               |

`inspectMediaAlt()` expone el nivel usado — apto para dashboards de
auditoría SEO/a11y y para métricas de cobertura de traducciones.

## 5 · Cobertura de renderizadores

La resolución se aplica **en el borde DTO** de los server reads. Los
renderizadores (`Gallery`, `ExperienceHero`, `ExperienceSection`,
`ExperienceGallery`, `CardGrid`, `product-blocks`, `MediaPickerDialog`,
`ProductAdvancedPanel`, `StaticMap`) siguen leyendo `.alt` — el
contrato del DTO garantiza que ese `.alt` ya vino resuelto.

| Read/DTO                                             | Resolver aplicado | Locale |
|------------------------------------------------------|:-----------------:|:------:|
| `getMarketplaceProductBySlug` (media & cover)        | ✅                | input `locale` (default `es`) |
| `listStudioMediaLibrary` (Experience Builder picker) | ✅                | `es`   |
| `listBusinessMedia` / `listDestinationMedia` / `listZoneMedia` / `listProductMedia` (CMS panels) | reads editoriales; consumen `alt_text` con conocimiento del editor — fuera del contrato público | n/a |

Enforcement automático: `bash scripts/assert-resolve-media-alt.sh`
bloquea futuros accesos directos desde superficies públicas.

## 6 · Cobertura multilenguaje

- 6 idiomas oficiales: `es`, `en`, `fr`, `de`, `it`, `pt`.
- Persistencia: `media_asset_translations` (unique `media_id + locale`).
- CMS: `MediaTranslationsSheet` — 6 pestañas por media, con acciones
  `Sugerir IA`, `Guardar como humano`, `Aprobar` (llama
  `approveMediaTranslation`).
- Batch: `suggestMediaAltBatch({ mediaIds ≤25, locales ≤6 })` con
  concurrencia 3 y protección `media_alt_protect_human`.
- Fallback determinista al `es` cuando el idioma solicitado no tiene
  traducción — nunca se devuelve vacío si hay alternativa válida.

## 7 · APIs oficiales

```ts
import { resolveMediaAlt, inspectMediaAlt, hasMeaningfulAlt } from "@/lib/media/resolve-alt";

resolveMediaAlt(media, { locale: "en", fallback: product.name }); // → string, nunca null
```

Server functions:

| Function                      | Rol                                           |
|-------------------------------|-----------------------------------------------|
| `suggestMediaAlt`             | Propuesta IA por locale (idioma primario o traducción). Nunca sobrescribe humano. |
| `saveMediaMetadata`           | Guardado humano (idioma primario o traducción). |
| `listMediaTranslations`       | Lectura CMS de traducciones por media.        |
| `suggestMediaAltBatch`        | Lote controlado (≤25 media × ≤6 locales, concurrencia 3). |
| `approveMediaTranslation`     | Aprobación IA→humano (idioma primario o traducción). |

## 9 · Impacto SEO y accesibilidad

- Cada `<img>` público recibe ALT semántico en el idioma correcto
  antes de renderizar — sin llamadas cliente.
- Fallback determinista elimina `alt=""` en imágenes con evidencia
  disponible.
- Base sólida para JSON-LD `image` con `ImageObject.caption` en
  próxima ola.
- Multilingüe listo para hreflang (política **Founder
  Internationalization SEO**) — mismo `media_id`, ALT distinto por
  idioma.

## 10 · Outcome Validation (5 preguntas)

1. ¿Qué comportamiento humano nuevo habilitó esta entrega?
   El editor puede curar ALTs multilenguaje sin miedo a que la IA sobrescriba
   su trabajo; el visitante recibe ALT en su idioma real.

2. ¿Cuál es la métrica observable?
   `% de imágenes con ALT resuelto ≠ "" por idioma`
   (calculable con `inspectMediaAlt` en un job de auditoría).

3. ¿Qué se rompería si esto no existiera?
   SEO multilingüe, accesibilidad y consistencia visual pública.

4. ¿Qué depende ahora de esta pieza?
   H3·A4 (WebP/AVIF), JSON-LD ImageObject, hreflang, futuros bloques
   editoriales, Alux Espacial (menciona lugares con imagen).

5. ¿Cuál es el próximo paso natural?
   H3·A4 · Physical Optimization sobre una arquitectura semántica
   consolidada.
