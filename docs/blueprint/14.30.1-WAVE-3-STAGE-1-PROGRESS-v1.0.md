## 1. Objetivo de la Etapa

Habilitar la infraestructura mínima del Portal Empresarial:

1. Subtree de rutas aislado bajo `_authenticated/portal/*`.
2. Layout con selector de empresa persistido en `localStorage`.
3. Server function autenticada para listar empresas accesibles al
   usuario actual (filtradas por `business_users` activo).
4. Server function de verificación de acceso por empresa
   (`has_business_access`) reutilizable por etapas siguientes.

## 10. Decisiones documentadas

- **Persistencia del selector:** `localStorage`, clave
  `valladolidmx.portal.activeBusinessId`. No se usa cookie ni query
  param (evita exponer IDs en URLs compartibles).
- **Reconciliación defensiva:** si la empresa activa ya no aparece en
  la lista de empresas accesibles, el layout cambia automáticamente
  a la primera disponible.
- **No se introduce contexto de ruta tipado para Portal en esta etapa.**
  Las subrutas en etapas siguientes consumirán `listMyBusinesses` y
  `getMyBusinessAccess` directamente o se promoverá un
  `useRouteContext` cuando exista necesidad probada.
