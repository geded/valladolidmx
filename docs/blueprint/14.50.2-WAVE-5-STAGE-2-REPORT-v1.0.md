## 1. Objetivo

Activar el primer canal del Unified Notification Center (**in-app**),
introducir el **Notification Router** idempotente y entregar una vista
inicial del **Intelligent Activity Center** de sólo lectura por rol.

## 2. Alcance entregado

### 2.1 Notification Router (in-app)
- RPC `unc_publish_in_app(event_id, event_type, recipient_user_id, audience, category, payload_ref)`
  - `SECURITY DEFINER`, `search_path = public`, `EXECUTE` revocado por defecto, otorgado a `authenticated`.
  - Idempotencia: `ON CONFLICT (event_id, channel, recipient_user_id)`.
  - Respeta `notification_preferences` para `operational` y `marketing`. `transactional` y `security` son siempre entregadas.
  - Estados: `sent` (entregada in-app) o `skipped` (silenciada por preferencia) — ambos quedan auditados.
- Server function `publishInAppNotification` (`requireSupabaseAuth`) — invocada por otros server-fn de negocio; no expuesta al cliente.

### 2.2 Catálogo inicial de suscripciones
Inserción idempotente en `notification_subscriptions` para los eventos BEA activos:
- `Business.Order.Created` → traveler / owner (in-app, transactional / operational)
- `Business.Order.Confirmed` → traveler / owner
- `Business.Payment.Succeeded` → traveler / owner
- `Business.Payment.Failed` → traveler / admin
- `Business.Webhook.SignatureInvalid` → super_admin (in-app, security)

### 2.3 Intelligent Activity Center (lectura)
RPCs `SECURITY DEFINER` con verificación de rol/pertenencia interna:
- `unc_activity_admin(limit)` — agrega `system_alerts` (Etapa 7), `orders` y `payment_events` recientes. Sólo admin / super_admin.
- `unc_activity_business(business_id, limit)` — órdenes y pagos de la empresa para owner/staff (validado vía `business_users`) o admin/super_admin.
- `unc_activity_traveler(limit)` — órdenes propias, pagos asociados e in-app feed del propio usuario.

Server functions: `getAdminActivity`, `getBusinessActivity`, `getTravelerActivity` (en `src/lib/notifications/activity.functions.ts`).

### 2.4 UI administrativa
- Ruta `/cms/actividad` — vista del Activity Center para administradores, integrada al shell `/cms` existente.
- Sin cambios visuales en Portal Empresarial ni Cuenta del Viajero en esta etapa (sus vistas dedicadas se cubren en Etapa 6).

## 3. Reutilización

- `system_alerts`, `marketplace_search_metrics` y RPCs admin de la Etapa 7 — sin duplicación de fuentes.
- Tablas y enums del UNC creados en Etapa 1 — sin migraciones de esquema adicionales.
- BEA y contratos existentes — sin modificación.
