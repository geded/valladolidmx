# CV7 · Travel Memory Loop — Travel Passport
**Versión:** 1.0 · **Estado:** APROBADO (Blueprint oficial) · **Roadmap:** v2.0 · Carril A · Programa Traveler Continuity
**Depende de:** CV0 (Travel Plan Contract), CV5 (Workspace), CV6 (Live Destination Companion), Founder Travel Memory Loop.

---

## 1. Visión

CV7 convierte el Travel Plan en el **Travel Passport del viajero Alux**: un activo personal que evoluciona con cada viaje y hace más inteligente el siguiente. No es un historial retrospectivo; es la memoria útil que Alux consulta para descubrir, decidir, organizar, comprar, disfrutar y regresar mejor.

> Cada experiencia vivida debe aumentar la capacidad de Alux para ayudar al viajero en su próximo viaje, sin crear nuevas fuentes de verdad y reutilizando exclusivamente el Travel Plan como memoria canónica.

## 4. Contratos técnicos

### 4.1 Dominio puro (fuente única · Travel Plan)

```
src/lib/traveler/passport.ts
  derivePassport(input: {
    plans: TravelPlanSnapshot[];   // reutiliza contratos CV0
    now: Date;
  }): Passport
```

`Passport` contiene: `visitedDestinations[]`, `completedExperiences[]`, `memories[]`, `derivedPreferences`, `stats`, `nextTripSeed`, y por cada entrada `sources[]` + `rationale` (Explainable by Default).

### 4.2 UI stateless

```
src/components/traveler/PassportSurface.tsx
src/components/traveler/PassportMemoryTimeline.tsx
src/components/traveler/NextTripSeedCard.tsx
```

100% presentacional. Consume el `Passport` derivado. Auto-Hide sin viajes completados.

### 4.3 Bridge con Alux

Alux consume `Passport` como contexto de sólo lectura vía Context Engine, siguiendo Founder Proactive Copilot Principle. Prohibido escribir al Passport desde Alux.

---

## 5. Guardrails arquitectónicos

- **Fuente única:** Travel Plan (`travel_plans` + `travel_plan_items` + snapshots inmutables CV0). Prohibido crear tablas, columnas persistentes o modelos paralelos para CV7.
- **Derivación pura:** `derivePassport()` es determinista, sin efectos secundarios ni I/O.
- **Backward compatible:** el contrato del Travel Plan permanece congelado; CV7 sólo lee.
- **Auto-Hide:** ninguna superficie de CV7 aparece sin evidencia útil (Founder Experience First).
- **Reversibilidad:** el viajero puede ocultar cualquier entrada del Passport sin borrar el Travel Plan de origen.

---

## 6. Validación obligatoria por sub-ola

Cada sub-ola sólo puede abrirse si responde SÍ a las 5 preguntas del Founder Travel Passport Principle:

1. ¿Reutiliza exclusivamente el Travel Plan como fuente canónica?
2. ¿Genera valor para el siguiente viaje?
3. ¿Evita crear memoria paralela?
4. ¿Puede explicar completamente el origen de cada dato mostrado?
5. ¿Permite evolucionar sin refactorizar el contrato existente?

---

## 7. Definition of Done (por sub-ola)

- Dominio puro con tests unitarios (Auto-Hide, determinismo, sources/rationale).
- UI stateless integrada en `NowNextLaterSurface` o `/mi-viaje`.
- Cero migraciones, cero nuevas tablas, cero columnas nuevas en Travel Plan.
- Completion Report con matriz de trazabilidad Política → Archivo → Prueba.
- Typecheck limpio, sin regresiones CV0/CV5/CV6.

---

## 8. Puerta de evolución

CV7 evoluciona de forma **aditiva y backward compatible**: futuras capacidades (recompensas, referidos, álbumes, estadísticas comparativas, badges de viajero) extienden `derivePassport()` sin modificar el Travel Plan ni introducir persistencia paralela. Cambios incompatibles requieren ADR + aprobación Founder.
