## 4. Workflow editorial cubierto

La máquina oficial (Serie 13.4) se mantiene íntegra:

```
draft → in_review → approved → published → archived
         ↘ draft        ↘ draft       ↘ draft
```

Toda transición:

1. Verifica rol editorial server-side (`is_editor_or_admin`).
2. Valida `from → to` contra `ALLOWED_TRANSITIONS`.
3. Persiste `status`, `updated_by` y (si aplica) `published_at`.
4. Registra evento `transition` en `content_audit_log` con `from_status`,
   `to_status`, actor y notas opcionales.

Las operaciones `created` y `updated` también se registran como acciones,
homogeneizando la bitácora editorial.

  editorial; quedan en logs para inspección posterior.
