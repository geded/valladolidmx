# H3·A4 · M1 · Migration Validation Report **v1.1 (ERRATA FORMAL)**

> **Nota**: Este documento sustituye a `v1.0` respecto a los conteos de
> `source_checksum` sobre `media_asset_variants`. El resto del reporte
> (columnas añadidas, `variant_key` generada, índices únicos parciales

> permanece vigente.

## 1. Motivo de la errata
El reporte v1.0 describía la población histórica de `source_checksum` con la
cifra "24 filas con NULL o '-'" agregando dos causas distintas en un solo
número. La cifra correcta desagregada es:

| Cohorte | Filas | Origen |
|---|---|---|
| `source_checksum IS NULL` (piloto v1.0 · placeholder no calculado) | **24** | Filas creadas por el harness M1 v1.0 antes de existir el cálculo determinista. Se conservan para historia; NO son elegibles para el read path M2 hasta backfill autorizado. |
| `source_checksum = '-'` (defecto legacy previo a M1) | **0** | La restricción legacy `UNIQUE (asset_id, format, width, engine)` nunca requirió checksum; el marcador `'-'` NO se usó en esta base. |
| `source_checksum` real (piloto v1.1) | **9** | Filas del asset autorizado `642cb15f-0a13-410c-8027-c4ab92034bf5` con checksum determinista. Elegibles. |

## 2. Corrección aplicada
- Se reemplaza "24 filas con NULL o '-'" por "**24 NULL + 0 '-'**" en toda
  referencia de v1.0.
- Se documenta que la elegibilidad real del canary M2.1 son **9 variantes**
  pertenecientes a **1 asset** (allowlist), NO 24.
- La `variant_key` generada usa `COALESCE(source_checksum, '-')`, por lo que
  las 24 filas NULL producen `variant_key` con segmento `'-'` (histórico,
  no reproducible). El evaluador shadow descarta variantes con checksum
  faltante emitiendo `fallback_reason='variant_key_missing'`.

## 3. Impacto
- **Ninguno en producción**: `media_pipeline_enabled=false`, no hay lectores.
- **M2.1 (Shadow Eval)**: la elegibilidad efectiva se documenta correctamente
  en el Blueprint M2 v1.1 §Elegibilidad Real y en `SHADOW_ALLOWLIST`.
- **Backfill**: cualquier operación futura para recalcular `source_checksum`
  de las 24 filas NULL debe autorizarse expresamente. NO se ejecuta en M2.1.

## 4. Resto del reporte
Los apartados §2..§9 y §10 (PASOS 1–3) de `v1.0` permanecen sin cambios.
`v1.1` se declara la versión canónica.

## 5. Firmas
