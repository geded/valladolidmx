## 1. Fuente canónica
- Única fuente de horarios: **`public.business_hours`** (columnas `business_id`, `day_of_week`, `opens_at`, `closes_at`, `is_closed`).
- No se crearon tablas ni modelos nuevos. No hay horarios hardcodeados. No hay lógica horaria en superficies.

## 2. Arquitectura entregada
- **Evaluador puro** `src/lib/business/hours-status.ts` → `evaluateHoursStatus(rows, { timezone, now, soonMinutes })`. Determinístico, sin I/O, unit-testable. Timezone LOCAL del destino (default `America/Merida`).
- **Hours Contributor efectivo** `src/lib/traveler/destination-context/contributors/hours.ts`:
  - Sólo lee `business_hours` para las entidades del scope (`scope.entities[type='business']`).
  - Normaliza cada entidad a `DestinationSignal<HoursSignalPayload>` con `explain` (fuente, franjas evaluadas, tz, momento de evaluación, rationale, freshness TTL 5 min).
  - No decide UI, no toca itinerario, no llama a Alux, no depende de Weather/Traffic (Guardrail).
- **Contrato de scope extendido** (aditivo, sin refactor de consumidores): `DestinationSignalScope.entities?: Array<{ id, type, timezone? }>` en `src/lib/traveler/destination-context/types.ts`.
- **Server fn** `resolveTravelerDestinationContext`: input schema acepta `entities` (Zod, máx 50). Pasa el arreglo al scope.
- **NowNextLaterSurface**: extrae entidades únicas del día en curso (fase `onsite`) y las inyecta en el server fn. Sigue sin acceder directo a `business_hours` (Guardrail cumplido).

## 3. Estados normalizados (contrato oficial)
`open_now` · `closing_soon` (≤60 min) · `closed_now` (reabre hoy > 60 min) · `opening_soon` (≤60 min) · `closed_today` (no reabre hoy) · `hours_unknown` (sin filas / inválidas / faltantes).

## 4. Correlación centralizada (Decision Center)
`src/lib/traveler/decision-center-destination.ts` traduce señales a tarjetas:
- `closing_soon` → “Cierra pronto” (warning, prioridad 72, slot `now`).
- `opening_soon` → “Abre pronto” (info, prioridad 58, slot `next`).
- `closed_now` / `closed_today` → “Está cerrado ahora” / “Cerrado hoy” (critical, slot `next`) con acción `view_alternative`.

Toda decisión/rationale/priorización vive aquí — el Contributor sólo emite señales.

## 5. Comportamiento degradado
- Sin filas / filas inválidas → `hours_unknown` → el Contributor **omite** la señal → el Decision Center no genera tarjeta → NowNextLaterSurface se auto-oculta (`center.empty`).
- Sin `entities` en el scope → contributor devuelve `[]` inmediatamente.
- Error de DB → `catch` silencioso → `[]`. Nunca bloquea Now·Next·Later.

## 6. Auto-Hide (Founder Experience First)
- **Cuándo aporta valor:** durante `phase === "onsite"`, cuando alguno de los negocios del día está cerca de cerrar, ya está cerrado, o reabre pronto.
- **Condición de auto-ocultamiento:** fuera de `onsite`, sin entidades, sin filas, o cuando ninguna señal cruza el umbral accionable.
- **Acción concreta al viajero:** “Cierra pronto → acelera la visita”, “Abre pronto → acércate”, “Cerrado → ver alternativas”.
