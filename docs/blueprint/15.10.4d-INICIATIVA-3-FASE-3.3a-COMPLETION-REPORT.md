## Corrección al plan 3.3 (importante)

La radiografía de Fase 3.3 dijo "cero call-sites externos". **Fue incorrecto.** El guard reveló dos consumidores activos preexistentes:

- `src/routes/l.$slug.tsx` → `ebResolvePublicPage` + `applyVariantOverrides` (landings v2).
- `src/routes/preview.$token.tsx` → `ebResolvePreview` (preview firmado v2, distinto del preview v1 en `src/routes/preview/composition.$token.tsx`).

Ambos apuntan a tablas `eb_*` con 0 filas, por lo que en producción devuelven `null` / fallback, pero el import existe. Se añadieron al **whitelist** del guard (el freeze prohíbe *nuevos* consumidores, no los preexistentes) y quedan marcados para retirar en Fase 3.3b junto con el borrado del código v2.

Implicación para Fase 3.3b: antes de borrar `eb-studio.functions.ts` / `eb-public.functions.ts` hay que migrar o eliminar esas dos rutas:
- `/l/$slug` — evaluar si debe reescribirse sobre v1 (`getPublishedCompositionBySlug`) o simplemente retirarse (redirect a `/p/$slug`, que ya usa v1).
- `/preview/$token` — el flujo de preview canónico ya es `/preview/composition/$token` sobre v1; la ruta v2 puede retirarse con redirect.
