## 🔴 Hallazgo H-1 · Desync de secreto Vault ↔ worker env

**Descripción:** el endpoint desplegado lee `process.env.MEDIA_SIGNATURE_RENEW_HMAC` (Cloudflare env). El trigger firma con `vault.decrypted_secrets['media_signature_renew_hmac']`. Durante el ciclo, la firma emitida por el trigger no coincidió con la esperada por el endpoint → 401 en cada intento. **El verificador HMAC funciona correctamente**; el hallazgo es de **procedimiento de sincronización**, no de código.

**Impacto:** el UPSERT monotónico (`masu_upsert_monotonic`) y la persistencia de firmas reales **no se ejercitaron en runtime**. Todo el resto del contrato (kill switch, sanitización, no-op, rollback, allowlist, bucket) sí quedó validado.

**Cero exposición:** ninguna URL firmada llegó a cliente, logs, ni respuesta HTTP. Ambos flags OFF al cierre. Datos placeholder eliminados.

**Remediación propuesta (fuera de este ciclo, requiere aprobación separada):**
1. Definir procedimiento oficial de rotación: mismo valor en Vault (`masu_bootstrap_secret`) y en env (`update_secret` / `set_secret`) — actualmente sin herramienta única atómica.
2. Alternativa recomendada: crear un server fn `masu_sync_secret_from_env` (super_admin) que lea `process.env.MEDIA_SIGNATURE_RENEW_HMAC` y lo empuje a Vault via `masu_bootstrap_secret` — la env queda como fuente única, Vault se sincroniza.
3. Re-ejecutar ciclo E2E acotado (mismo alcance y ventana) para validar UPSERT real.
