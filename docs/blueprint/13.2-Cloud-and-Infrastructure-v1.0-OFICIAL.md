
# 13.2 – Cloud & Infrastructure
**Versión:** 1.0
**Estado:** Aprobado para ejecución
**Blueprint:** Valladolid.mx v2.0

# 1. Propósito

Definir la infraestructura oficial sobre la que operará Valladolid.mx.
Este documento establece las capacidades mínimas que deberá implementar Lovable.
La infraestructura debe ser escalable, reproducible y desacoplada del destino Valladolid.

# 2. Principios

- La infraestructura debe soportar múltiples regiones.
- Todo recurso será declarativo y versionado.
- Ningún dato crítico dependerá del frontend.
- Seguridad por defecto.

# 3. Componentes

## Plataforma
- Lovable Cloud
- PostgreSQL
- Storage
- Authentication
- Edge Functions (cuando aplique)

## Base de datos

Requisitos:
- UUID como identificador principal.
- Migraciones versionadas.
- Integridad referencial.
- Índices para búsqueda.
- Auditoría.

## Storage

Buckets mínimos:
- logos
- hero
- destinations
- companies
- products
- gallery
- documents
- temporary

Todos los archivos deberán registrar propietario, fecha y metadatos.

# 4. Seguridad

- Row Level Security habilitado.
- Acceso por roles.
- Políticas por organización.
- Secretos fuera del repositorio.

# 5. Migraciones

Toda modificación estructural deberá:
1. Crear migración.
2. Ser reversible.
3. Documentarse en el Blueprint.

# 6. Backups

- Respaldo automático.
- Restauración validada.
- Pruebas periódicas de recuperación.

# 7. Observabilidad

Registrar:
- Errores.
- Eventos críticos.
- Auditoría de cambios.
- Métricas de rendimiento.

# 8. Rendimiento

Objetivos:
- Consultas optimizadas.
- Carga progresiva.
- Caché cuando aplique.
- Preparado para crecimiento regional.

# 9. Entregables

- Infraestructura desplegada.
- Migraciones.
- Configuración documentada.
- Evidencias de pruebas.
- Checklist de seguridad.

# 10. Definition of Done

El bloque concluye cuando:
- Infraestructura reproducible.
- Base de datos operativa.
- Storage funcional.
- RLS validado.
- QA aprobado.
- Documentación sincronizada.
