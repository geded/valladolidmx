# CV6.8 · Live Recap & Handoff — Blueprint oficial v1.0

Ola CV6 · Live Destination Companion · Sub-ola 8
Fase objetivo: `onsite` · sub-fase `post-day` (end-of-day).

---

## 1. Directiva del Founder (vinculante)

CV6.8 cierra cada día en destino con un **recap útil** y prepara el **handoff** al siguiente momento del ciclo (mañana, siguiente destino o post-viaje). Nunca representa el "final" del viaje.

### Políticas oficiales (5)

1. **Founder Continuity Principle** — el recap siempre construye un puente hacia el siguiente momento del ciclo (mañana / siguiente destino / post-viaje). Prohibido presentar el recap como final.
2. **Recap as Derived View** — el recap es vista completamente derivada. Prohibido crear modelo persistente, tabla, cache o estado paralelo.
3. **Single Source Timeline Principle** — todos los eventos provienen exclusivamente de fuentes oficiales existentes (Decision Center, Travel Plan, Concierge, Context Engine). El recap nunca se convierte en línea de tiempo independiente.
4. **Explainable Summary Principle** — cada highlight, pendiente y recomendación debe declarar `sources` y `rationale`. Ningún elemento por inferencia opaca.
5. **No AI Memory Duplication Principle** — CV6.8 no introduce memoria adicional para IA ni historiales paralelos. La continuidad reutiliza el Founder Memory Principle ya existente.

Registradas en `mem://policies/founder-recap-continuity.md`.

## 2. Alcance funcional

Fuentes reutilizadas EXCLUSIVAMENTE:
- `LiveDayContext` (CV6.1) · trigger `livePhase === "post-day"`.
- `DecisionCenter` (CV6.2/6.3) · pendientes vigentes al cierre.
- `ResolvedDestinationContext` (CV6.4/6.5) · contexto operativo del cierre.
- `AluxSpatial` (CV6.6) · propuestas activas.
- `OnTripConciergeState` (CV6.7) · caso vigente en el handoff.
- `travel_plans` + `travel_plan_items` · ítems de hoy y mañana.

Prohibido: nueva tabla, nuevo modelo, nueva ruta, nuevo Provider, nueva fuente.

### Estados
| Estado | Cuándo |
|---|---|
| `hidden` | phase ≠ onsite · o livePhase ≠ post-day · o sin valor real |
| `preparing` | end-of-day pero sin evidencia suficiente todavía |
| `ready` | recap del día con highlights o pendientes |
| `handoff-tomorrow` | ready + existen ítems para día siguiente dentro del viaje |
| `handoff-post-trip` | ready + es el último día del viaje |

## 3. Contratos técnicos

### 3.1 Dominio puro (`src/lib/traveler/live-recap.ts`)
```ts
export type LiveRecapState =
  | "hidden" | "preparing" | "ready"
  | "handoff-tomorrow" | "handoff-post-trip";

export interface LiveRecapHighlight {
  id: string;
  label: string;
  rationale: string;
  sources: string[];
}

export interface LiveRecapPending {
  id: string;
  label: string;
  rationale: string;
  sources: string[];
}

export interface LiveRecapTomorrowPreview {
  day: number;
  itemsCount: number;
  firstStartAt: string | null;
  rationale: string;
  sources: string[];
}

export interface LiveRecap {
  state: LiveRecapState;
  visible: boolean;
  headline: string;
  rationale: string;
  highlights: LiveRecapHighlight[];
  pendingItems: LiveRecapPending[];
  tomorrowPreview: LiveRecapTomorrowPreview | null;
  reversible: true;
  sources: string[];
  explain: { rules: string[] };
}

export function deriveLiveRecap(input: {
  phase: TripPhase;
  liveDay: LiveDayContext;
  planItems: LiveDayItemInput[]; // TODOS los ítems del plan (para día+1)
  endDate?: string | null;       // ISO date · para determinar último día
  decisionCenter?: { now: DecisionCard[]; next: DecisionCard[]; later: DecisionCard[]; empty: boolean };
  assistance?: { state: string; visible: boolean };
  at: Date;
}): LiveRecap;
```

### 3.2 UI stateless (`src/components/traveler/LiveRecapSurface.tsx`)
```tsx
export function LiveRecapSurface(props: {
  recap: LiveRecap;
  onOpenTomorrow?: () => void;
}): JSX.Element | null;
```
Props → render. Sin stores, polling, timers, efectos ni llamadas a APIs. Auto-Hide si `!recap.visible`.
