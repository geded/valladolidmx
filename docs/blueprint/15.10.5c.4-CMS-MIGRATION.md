# 15.10.5c.4 · Ola 4 — Migración de `/cms/*` al Workspace Engine

Estado: **Implementación entregada — pendiente de cierre formal y
Migration Completion Audit + Post-Migration Validation.**

## Alcance ejecutado (1:1, sin cambios funcionales)

- `src/routes/_authenticated/cms.tsx` migrado al Workspace Engine:
  ahora envuelve `<Outlet />` en
  `<WorkspaceProvider initialWorkspaceId="cms"><WorkspaceShell>`.
  Eliminados (Legacy Removal):
  - topbar móvil propio + botón hamburguesa;
  - overlay y drawer móvil propios;
  - aside desktop + nav cableada;
  - tabla `NAV` y tipo `NavItem`;
  - subcomponente `SidebarContent`;
  - bloque de perfil + botón de cerrar sesión inline;
  - imports `BrandLogo`, `useRouterState`, `useState`, `useEffect`,
    `ROLE_LABELS`.
- Preservado 1:1:
  - gate de UX por rol editorial (`super_admin`, `admin`, `editor`)
    con el mismo copy 403 ("Acceso restringido") y CTA "Volver al
    inicio"; la autorización dura sigue ocurriendo server-side en
    cada server function (`has_role` / `requireSupabaseAuth`);
  - todas las subrutas `/cms/*` intactas: `index`, `regiones`
    (index + `nueva` + `.editar`), `destinos`, `zonas`, `categorias`
    (index + `nueva` + `.editar`), `empresas`, `productos`, `media`,
    `reviews` (`index` + `$id/moderar`), `pagos`, `observabilidad`,
    `alertas`, `actividad`, `experience-builder` (+ `.pages`).
- `WorkspaceDefinition` `cms` actualizada en
  `src/lib/workspace/definitions/index.ts`:
  - navegación completa declarada en el Navigation Registry
    (Resumen, Regiones, Destinos, Zonas, Categorías, Empresas,
    Productos, Media, Reseñas, Pagos, Observabilidad, Alertas,
    Actividad, Experience Builder), agrupada por
    `vista` / `territorio` / `contenido` / `moderacion` /
    `operacion` / `estudio`;
  - bottom-nav prioriza Resumen, Empresas, Media y Reseñas;
  - Alux Copilot Explainable by Default (`rationale`, `sources`,
    `effect`, `reversible`, `confirm`) con `summary` enriquecido;
  - `context` declarativo con entidades `review`, `business`,
    `destination`, `region`, `category`, `product`, `media`,
    `page` y modos de selección `single`/`multi`.

  ninguno introducido por la ola. El gate 403 es la única vista

## Cumplimiento de políticas

- **Workspace First / Build Once, Reuse Everywhere**: cumplido —
  el CMS ya no implementa layout, sidebar, topbar, drawer móvil
  ni navegación propios.
- **Migration Before Evolution**: sin cambios de negocio, reglas,
  workflow editorial, Experience Builder, pagos, alertas,
  observabilidad, actividad, media, reviews, productos, empresas,
  destinos, regiones, categorías ni zonas.
- **Legacy Removal Policy**: topbar móvil, overlay, drawer,
  aside desktop, tabla `NAV`, tipo `NavItem`, subcomponente
  `SidebarContent` y bloque de perfil eliminados; no hay
  variantes legacy en paralelo.
- **Definition of Migration Done**: cumplido en su totalidad.
- **Migration Granularity**: esta ola migra sólo CMS; Founder
  se aborda en 15.10.5c.5.
- **Zero Context Loss**: workspace activo, sidebar/inspector,
  selección, scroll y filtros se preservan al navegar entre
  subrutas del CMS.

## Rollback

Revertir esta ola = restaurar el commit previo de
`src/routes/_authenticated/cms.tsx` (topbar móvil + drawer + aside
desktop + `NAV` + `SidebarContent`) y revertir el bloque `cms` en
`src/lib/workspace/definitions/index.ts`. Las 20 subrutas `/cms/*`,
los componentes (`CmsEntityPage`, `EntityEditor`, `EntityListView`,
`ReviewModerator`, Experience Builder) y los server functions
permanecen intactos, por lo que el rollback es atómico al layout.
Las olas 15.10.5c.1 (Cuenta), 15.10.5c.2 (Concierge), 15.10.5c.3
(Portal) y la infraestructura 15.10.5a/b no se ven afectadas.
