# 15.10.5d.4 — Migración de superficies públicas restantes → Public Shell + buildPublicHead

## Alcance
Migración 1:1 de todas las superficies públicas restantes del Blueprint
Discovery Layer al `PublicShell` (15.10.5d.1) y `buildPublicHead`
(15.10.5d.1). Cierra la migración de la experiencia pública sobre la
infraestructura canónica de Discovery.

Rutas migradas:
- `/empresas`
- `/experiencias`
- `/hoteles`
- `/restaurantes`
- `/eventos`
- `/oriente-maya`
- `/oriente-maya/$destino`
- `/arma-tu-viaje`
- `/alux`

## Cambios aplicados

### Layout
- Reemplazo de `PageShell` por `PublicShell` en todas las rutas
  listadas. La API (eyebrow / title / description / crumbs) es
  equivalente, por lo que la paridad visual se preserva 1:1.
- `PageShell` permanece exclusivamente en superficies de autenticación
  (`/auth`, `/reset-password`), fuera del alcance del Discovery Layer.

### SEO (`SEO is Architecture`)
- Toda la metadata pública se construye mediante `buildPublicHead`.
- Quedan eliminados los bloques manuales `meta:[…]` con `title`,
  `description`, `og:title`, `og:description` literales.
- `canonical`, `og:url`, `og:site_name`, `og:locale`, `twitter:card`
  ahora se emiten uniformemente por construcción.
- `/oriente-maya/$destino` adopta `ogType: "place"` para reflejar la
  semántica del documento (entidad geográfica).

## Migration Completion Audit
- ✅ Paridad funcional 1:1: ningún componente interno fue modificado
  (solo el shell envolvente y el contrato de `head`).
- ✅ Experience Builder preservado: la Home y las superficies que
  consumen el Sections Registry no se tocan en esta adenda.
- ✅ Cards Registry intacto.
- ✅ Política `Experience Builder as Content Layer` respetada: el
  shell, navegación y SEO permanecen bajo el Public Shell.
- ✅ `Performance First`: no se introducen dependencias ni patrones
  que alteren SSR o CWV; el render sigue siendo SSR-first.
- ✅ Legacy Removal Policy: `PageShell` ya no es consumido por
  ninguna superficie pública del Discovery Layer.

## Post-Migration Validation
- Typecheck: `tsgo --noEmit` en verde.
- Cero regresiones visuales esperadas: `PublicShell` reproduce 1:1 el
  patrón previo de `PageShell` (mismo `Container`, mismo header de
  página, mismos breadcrumbs).
- Responsive preservado (iPhone, iPad, Desktop) — no se modificó
  ninguna clase de los bloques internos.
- SEO: canonical y og:url ahora self-referencian cada ruta migrada.

## Rollback
1. Restaurar imports `PublicShell` → `PageShell` (mismo API).
2. Restaurar bloques `head()` literales eliminados (revertir a la
   versión previa de cada archivo).
3. Re-añadir `og:title`/`og:description` literales donde aplicaba.
4. No requiere cambios en infraestructura compartida.
