| Principio | Cumplimiento |
|---|---|
| Renderer único | `composition-renderer.tsx` se usa idéntico en Canvas (Studio), `/preview/$token` y publicación. Cero renderers paralelos. |
| `surface_type` como metadato | El Studio jamás condiciona lógica por `kind`. Solo se muestra como badge. Wizard inicial pide `kind` pero solo persiste el valor. |

| Preview fiel a producción | `PreviewFrame` y `/preview/$token` montan el mismo `CompositionRenderer` con el mismo `variableContext`. Sin componentes alternativos. |
| Server functions + RPCs SECURITY DEFINER | 13 server functions nuevas en `eb-studio.functions.ts`. Toda escritura llama RPC `eb_*_upsert`/`eb_page_*`/`eb_theme_*`/`eb_variant_*`/`eb_preview_token_issue`. Cliente no escribe a tablas. |

## Catálogo inicial de Variables Dinámicas

| Scope | Campos |
|---|---|
| `business` | name, tagline, city, phone_public |
| `destination` | name, region |
| `experience` | title, duration |
| `product` | name, price_label |
| `event` | title, date_label |
| `user` | display_name, language |
| `i18n` | cta_label, learn_more |
| `currency` | code, symbol |
| `site` | name, tagline |

Reglas:
- Solo tokens registrados se resuelven; tokens fuera del catálogo permanecen intactos (no se interpretan).
- Nunca se exponen tokens, emails internos, direcciones privadas, IDs internos u otros campos sensibles.
- `registerVariableScope(scope, descriptors)` permite extender el catálogo sin modificar el motor.

## Flujo demo de cierre (validado)

1. Admin abre `/cms/experience-builder/pages` y pulsa **Nueva** (slug, nombre, kind).
2. La página queda en `draft`. El Studio carga Biblioteca + Canvas + Inspector.
3. Arrastra **Hero** y luego **Sección Empresas** desde la Biblioteca (búsqueda / favorito / recientes operan).
4. Selecciona el Hero. Edita "Título" con el picker `{ }` insertando `${site.name}` y `${i18n.cta_label}`. El Canvas muestra los valores demo en tiempo real (`Valladolid.mx`, `Reservar`).
5. Cambia el **Tema** desde el panel lateral. Los tokens del tema se aplican como CSS variables al contenedor del canvas (paridad con producción).
6. Pulsa **Guardar versión** con nota "v1 — hero + empresas".
7. Pulsa **Preview ↗** — el Studio emite token vía `eb_preview_token_issue` y abre `/preview/$token` en pestaña nueva. El visitante ve el mismo render con variables resueltas.
8. Pulsa **Publicar**. La RPC `eb_page_publish` registra la versión y marca `status=published`. El badge cambia en la lista.
9. Toda la sesión queda auditada en `eb_audit_log` con `action`, `actor`, `scope` y `payload_diff`.

## Deuda técnica documentada (NO se modifica software fuera de Fase 2)

1. **Enum `eb_page_kind` limitado a 4 valores** (`landing | institutional | campaign | site_section`). El lineamiento estratégico exige soportar también `destination`, `business`, `product`, `event`, `wedding`, `promo`, `microsite`, `ai_generated`. Propuesta para 15.10.4b Fase 3 o 15.10.4c: ampliar el enum vía migración aditiva (no rompe filas existentes) y mapear los presets del wizard a los nuevos valores. *Se documenta sin implementar para no contradecir la regla de "proponer antes de cambiar arquitectura aprobada".*
2. **Drag & drop nativo HTML5 sin @dnd-kit**: la Fase 2 entrega ordenar/duplicar/eliminar desde la toolbar y "click-to-add" desde la Biblioteca. La experiencia de arrastrar con vista previa flotante se incorpora en 15.10.4b Fase 3 (no bloquea la demo de cierre).
3. **Selectores `media` y `reference`** del Auto-Inspector están como inputs de URL/ID. El combobox conectado a `media_assets`/`destinations`/`businesses`/etc. queda en 15.10.5 (Smart Pickers).
4. **Editor enriquecido (`rich_text`)** es un `<textarea>` con tokens. Tiptap u otro WYSIWYG se evalúa en 15.10.4c.
5. **A/B Variants** entregadas como CRUD de overrides. Resolución por audiencia/`predicate` en SSR llega en Fase 3.
6. **Autosave** debounced 1 s vía `eb_page_upsert` reescribe `tree`. Si se requiere un endpoint dedicado `eb_page_save_draft` para evitar reescribir SEO/OG en cada tecla, se añade en Fase 3 (decisión deferida, sin impacto funcional ahora).
