# H-03 · U1.5 — Tourism Component Library Hardening · Completion Report

**Estado:** Cerrada
**Carril:** B (Hardening)
**Fecha:** 2026-07-06

## Objetivo

Consolidar la Biblioteca Oficial de Componentes Turísticos como
infraestructura estable: una sola familia por dominio, guardarraíles
automáticos, cero duplicidades y unificación del Hero de producto bajo
`vmx.experience.hero`.

## Alcance ejecutado

1. Auditoría global de la Tourism Component Library.
2. Guardarraíl automático `scripts/tourism-guard.ts`.
3. Unificación `vmx.product.hero` → `vmx.experience.hero`.
4. Matriz oficial de componentes turísticos.
5. Verificación de compatibilidad y no regresión.

### Fuera de alcance (dependencia técnica registrada)

La delegación completa del **Home Hero** (`src/components/home/Hero.tsx`)
queda registrada como dependencia del **Studio CMS**: requiere edición
inline (`data-eb-field`) sobre contratos oficiales. Se abordará en una
iniciativa específica de evolución del Studio. No bloquea U1.5.

## 2) Guardarraíl automático

`scripts/tourism-guard.ts` bloquea:

- Sufijos prohibidos (`-pro`, `-v2`, `-next`, `-lite`) dentro de
  `vmx.experience.*`.
- Cualquier nuevo `vmx.<dominio>.hero` fuera de `vmx.experience.hero`
  (excepto shims legacy explícitos en el allowlist).

Ejecución: `bun scripts/tourism-guard.ts` → exit 0 = OK.

## 3) Unificación `vmx.product.hero` → `vmx.experience.hero`

Adapter oficial:
`src/components/experience-builder/blocks/experience-hero/ExperienceHeroFromProduct.tsx`.

- Mapea `MarketplaceProductDetail` al `ExperienceHeroDTO` v1.1.0
  (variant `editorial`, eyebrow = `product_type`, title = `name`,
  description = `tagline`).
- Reutiliza el slot oficial `extensionsSlot` para inyectar
  `FavoriteButton` + `AddToTravelPlanButton` sin duplicar lógica.
- `composition-renderer.tsx` (STUDIO_PREVIEW_MAP y PRODUCTION_COMPONENT_MAP)
  ahora resuelve `vmx.product.hero` con el adapter. El export legacy
  `ProductHeroBlock` deja de referenciarse; se conserva en el archivo
  para no romper regresiones/snapshots.
- `block-library.ts` etiqueta `vmx.product.hero` como shim legacy.

## 5) Founder Directive · cierre

Tras U1.5, la **Tourism Component Library** queda declarada como
infraestructura estable del producto. Las siguientes evoluciones deberán
enfocarse en capacidades de negocio (Carril A), no en reconstruir la
biblioteca. Toda evolución de un componente turístico se hace por
`variant` / `capabilities` / `extensions[]` / `config`, respetando la
Evolutionary Compatibility Rule.

## Rollback

1. Revertir el cambio en `composition-renderer.tsx`
   (`vmx.product.hero` → `ProductHeroBlock`).
2. Eliminar `ExperienceHeroFromProduct.tsx`.
3. Eliminar `scripts/tourism-guard.ts`.
4. Eliminar este documento.

El resto de la biblioteca oficial no requiere rollback (auditoría y
etiquetados de descripción son no destructivos).
