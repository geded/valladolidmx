## Capa 1 · Presentación (`ExperienceProducts.tsx`)

Componente 100% puro. Cuatro variantes:

- `grid` — tarjetas responsivas 1/2/3/4 columnas.
- `list` — filas densas mobile-first.
- `carousel` — scroll horizontal con snap.
- `featured` — 1 destacado + 3 secundarios.

Sub-componentes internos (`ProductCard`, `ProductRow`, `FeaturedLayout`).
Interactividad se recibe por slot `renderItemActions` — la presentación
no importa `FavoriteButton`, `ProductActions` ni ningún contexto.

## Capa 2 · Contenido (`contract.ts`)

```
ExperienceProductsConfig {
  source: manual | business | destination | region | category | context
  variant, heading, subheading, emptyMessage, columns, maxItems, groupBy
  items: ExperienceProductItem[]
  filters: { productTypes[], businessId }
  capabilities: {
    showPrice, showFavorite, showActions, showBusiness, showMedia,
    compact, contextAware*, livePricing*, liveAvailability*
  }
  contextRefs: { destinationSlug, regionSlug, categorySlug,
                 businessSlug, locale, tripType, aluxSessionId }
  extensions: [{ kind, config }]
}
```

`*` = reservado para evolución (Ola I3 / futura inteligencia contextual).
Cambiarlos NO requiere nuevo bloque — sólo bump minor del contrato.

## Capa 3 · Comportamiento (`ExperienceProductsBlock.tsx`)

- `source: "manual"` → usa `config.items`.
- `source: "business"` → hidrata desde `BusinessSurfaceContext` y
  construye un `interactiveMap` (id → `ProductActionsProduct`) que
  se inyecta vía `renderItemActions`. La presentación permanece pura.
- `source: destination|region|category|context` → **reservado** para
  I2.d y Ola I3 (Context Engine / Alux). Contrato ya lo acepta.

Exporta `marketplaceProductToItem` como utilidad de mapeo reutilizable
por adapters de otras superficies.

## Refactor Business Mother Template

`src/components/surfaces/BusinessSurface.tsx`: eliminado el `<ul>` de
products y toda referencia directa a `ProductActions` / `FavoriteButton`
para el listado. Ahora se declara:

```tsx
<ExperienceProductsBlock
  config={{
    source: "business",
    variant: "grid",
    heading: variant.productsHeading,
    emptyMessage: variant.productsEmpty,
    columns: 2,
  }}
/>
```

La plantilla vuelve a ser lo que la Regla de Orquestación exige:
**declaración de composición**, no lógica visual.
