# H-03 · Ola I3.a — DestinationSurface: Migración a Orquestador Puro

**Versión:** v1.0 · **Estado:** CERRADA (pendiente validación Founder)

## Objetivo
Consolidar `DestinationSurface` como **segunda Plantilla Madre de
referencia** — tras `BusinessSurface` (I2.d) — aplicando el mismo
patrón arquitectónico aprobado por el Founder:

1. Provider de contexto (`DestinationSurfaceProvider` + `DestinationSurfaceContext`).
2. Adaptadores de datos (`destination-to-blocks.ts`).
3. Composición de bloques oficiales del Experience Builder.
4. Cero lógica visual propia (con excepción documentada, ver abajo).
5. Orquestación pura.

## Bloques consumidos
| Bloque                          | Fuente     | Rol                                     |
|---------------------------------|------------|-----------------------------------------|
| vmx.experience.hero             | adapter    | Encabezado territorial (immersive/editorial) |
| vmx.experience.sub-navigation   | adapter    | Scroll-spy dinámico entre anclas activas |
| vmx.experience.section          | adapter    | "Sobre {destino}" — descripción larga    |
| vmx.experience.info-grid        | adapter    | "Lo esencial" derivado de `highlights[]` |
| DiscoveryNavigatorBlock         | contexto   | Panel aside — descubrimiento territorial |
| vmx.experience.cta-bar          | adapter    | Barra persistente móvil — ir a marketplace |

## Excepción arquitectónica documentada (transitoria)
La sección "Empresas del destino / Eventos próximos / Productos
destacados" se mantiene temporalmente como JSX dentro del componente
aislado `LegacyRelatedComposition`. Motivo:

> La Biblioteca Oficial **no expone todavía** un bloque
> `vmx.experience.related-collection` capaz de renderizar colecciones
> heterogéneas (empresas + eventos + productos) reutilizando tiles del
> marketplace. Crear ese bloque debe ser una evolución cuidadosa (no
> duplicar `vmx.experience.products` ni fabricar `-events`), por lo que
> se pospone a **H-03 · Ola I3.b** tras aprobación explícita del
> Founder. Mientras tanto, el JSX transitorio queda aislado, marcado
> con TODO y no contamina el árbol de orquestación principal.

Esta excepción cumple la nueva regla del Founder (post-I2.d):
> "Si durante futuras implementaciones aparece JSX visual dentro de
> una Plantilla Madre, deberá justificarse explícitamente y
> documentarse como excepción arquitectónica."

## Cumplimiento
- **Regla de Herencia de Plantillas:** OK — DestinationSurface replica
  1:1 el patrón de BusinessSurface (provider + adapters + composición).
- **Regla de Orquestación:** OK — cero inteligencia de negocio;
  presentación 100% delegada a bloques oficiales (excepto excepción
  documentada arriba, aislada en `LegacyRelatedComposition`).
- **Regla de Compatibilidad Evolutiva:** OK — cero bloques duplicados;
  todo se resuelve vía `variant` / `capabilities`; la necesidad
  detectada (colecciones heterogéneas) queda registrada como
  candidato a evolución en I3.b, no como duplicación.
- **Regla de Protección del Producto:** OK — paridad visual con la
  ficha actual (`/oriente-maya/{slug}`); todas las secciones,
  eventos, productos, discovery navigator y CTA territorial se
  conservan.
- **Single Source of Truth:** OK — adapters oficiales + contexto único.

## Reutilización futura del patrón
DestinationSurface queda declarado, junto con BusinessSurface, como
**modelo de referencia** para las próximas Plantillas Madre:
`ProductSurface`, `RegionSurface`, `EventSurface`, `HotelSurface`,
`RestaurantSurface`, `ExperienceSurface` — todas heredarán este mismo
patrón (provider + adapters + composición de bloques oficiales).

## Preparación para I3.b (dirección evolutiva)
- Diseñar contrato Zod v1.0.0 de `vmx.experience.related-collection`
  con `entityKind: business|product|event`, `variant: grid|list|carousel`
  y `source: manual|destination|region|context` (Regla Contextual).
- Migrar `LegacyRelatedComposition` a composición 100% declarativa vía
  este bloque, retirando el JSX transitorio de la superficie.
- Evaluar si `vmx.experience.events` merece nacer independiente o
  vivir como variante del bloque colecciones.
