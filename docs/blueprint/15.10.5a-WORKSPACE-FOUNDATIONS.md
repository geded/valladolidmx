# 15.10.5a — Workspace Foundations

**Estado:** Implementación inicial entregada
**Fecha:** 2026-06-30
**Alcance:** Cimientos del Workspace Engine v2.0 (no migra aún superficies existentes — eso es 15.10.5c).

---

## 1. Principios aprobados (rectores de toda la implementación)

1. **Un único Design System** — `src/styles.css` (@theme).
2. **Un único Workspace Engine** — `src/components/workspace/*`.
3. **Workspace extensible** mediante `WorkspaceDefinition` declarativas.
4. **Navigation Registry** como única fuente de navegación.
5. **Alux Copilot Layer** transversal del Workspace Engine.
6. **Mobile First · Touch First · PWA First.**
7. **Cero regresiones** en superficies existentes.

---

## 2. Contratos públicos

### 2.1 `WorkspaceDefinition` (`src/lib/workspace/types.ts`)
Declaración completa de un workspace:
- `id`, `label`, `icon`, `accent`, `rootPath`, `roles?`
- `navigation: NavItem[]` — items declarativos (sidebar/bottom/palette/switcher).
- `commands?: CommandEntry[]` — acciones contribuidas al Command Palette.
- `alux?: AluxContext` — narrativa + acciones de alto impacto.
- `aluxCapabilities?: AluxCapability[]` — lecturas de contexto expuestas a Alux.

Cualquier nuevo dominio (futuro o externo) se incorpora con:

```ts
import { registerWorkspace } from "@/lib/workspace/workspace-registry";
registerWorkspace(myWorkspaceDefinition);
```

**El núcleo del Workspace Engine no contiene una sola referencia a un workspace concreto.**

### 2.2 Navigation Registry (`src/lib/workspace/navigation-registry.ts`)
API:
- `contributeNavItem(item)` / `removeNavItem(id)`
- `getNavItemsForWorkspace(workspaceId, surface)` — `"sidebar" | "bottom" | "palette" | "switcher"`.
- `groupNavItems(items)` — agrupa por `group` preservando `order`.

`WorkspaceSidebar`, `WorkspaceBottomNav` y `CommandPalette` **solo** leen de aquí. **Prohibido hardcodear menús.**

### 2.3 Alux Copilot Layer (`src/lib/workspace/alux-registry.ts`)
API:
- `getAluxContext(workspaceId)`
- `getAluxCapabilities(workspaceId)`
- Resolvers async: `resolveAluxHeadline`, `resolveAluxSummary`, `resolveAluxActions`.

Consumido por `AluxCopilotPanel` (transversal). Cada workspace declara su contexto/capacidades en su `WorkspaceDefinition`.

## 3. Componentes UI entregados

| Componente | Responsabilidad |
|---|---|
| `WorkspaceProvider` | Estado del shell (workspace activo, sidebar, inspector, densidad, palette). Persiste en `vmx.workspace.prefs`. Atajo global ⌘K. |
| `WorkspaceShell` | Layout responsive (3 planos + inspector + bottom-nav móvil). |
| `WorkspaceSidebar` | Sidebar adaptable: `expanded` (256) · `rail` (64) · `hidden` · `floating`. |
| `WorkspaceBottomNav` | Bottom-nav móvil de 5 destinos + sheet "Más", safe-area aware. |
| `WorkspaceTopbar` | Header 56/64px con switcher móvil, búsqueda ⌘K, conectividad, toggle Copiloto. |
| `WorkspaceInspector` | Panel derecho · drawer · bottom-sheet según breakpoint. |
| `AluxCopilotPanel` | Capa Copiloto transversal sobre el Alux Registry. |
| `WorkspaceSwitcher` | Dropdown del workspace activo. |
| `CommandPalette` | Única superficie ⌘K. Lee del Navigation Registry. |
| `BottomSheet` | 3 snaps (peek/half/full) sobre `vaul`. |
| `ConnectivityIndicator` | Estado online/reconectando/offline, `aria-live="polite"`. |
| `EmptyState` | 4 variantes (`first-run | filtered | permission | error`). |
| `Skeleton{Card,List,Grid,Line}` | Con shimmer y respeto a `prefers-reduced-motion`. |
| `EntityCard / MetricCard / ActionCard` | Card system canónico (targets ≥44px, estado pressed). |
| `useWorkspaceBreakpoint` | Hook responsive `xs | sm | md | lg | xl | 2xl`. |

## 4. Tokens añadidos al Design System

Sin renombrar tokens existentes. Aditivos en `src/styles.css`:

- **Superficies:** `--surface`, `--surface-raised`, `--surface-overlay`, `--border-strong`, `--divider`, `--scrim`.
- **Estados:** `--success/-foreground`, `--warning/-foreground`, `--info/-foreground`.
- **Sombras:** `--shadow-card`, `--shadow-raised`, `--shadow-overlay`, `--shadow-focus`.
- **Motion:** `--motion-{instant|fast|base|slow|deliberate}`, `--ease-standard`, `--ease-emphasized`.
- **Densidad:** `data-density="compact"` sobre `<html>`.
- **Utilities:** `ws-shadow-card`, `ws-shadow-raised`, `ws-shadow-overlay`, `ws-shimmer`.

## 6. Criterios de aceptación

- ✅ Tokens y utilidades aditivos — sin colisión con superficies actuales.
- ✅ Cero hardcoded en componentes nuevos (`text-white`, `#hex`).
- ✅ Registries son única fuente de workspaces, navegación y Alux.
- ✅ `WorkspaceShell` re-fluye sin pérdida de estado entre breakpoints.
- ✅ ⌘K / Esc operan globalmente sin interferir con inputs editables.
- ✅ Preferencias persisten en `localStorage` (acceso sólo cliente).
- ✅ Indicador de conectividad reacciona a `online/offline`.
- ✅ Sin regresiones: Discovery Experience y rutas existentes intactas.
- ✅ Showcase entregado en `/lovable/workspace-foundations`.
