# 15.10.6.1 — Service Worker Base + Installability

Estado: **CERRADA**

## Alcance
- Registro real del Service Worker generado por `vite-plugin-pwa` (`generateSW`).
- Estrategia `autoUpdate`, `injectRegister: null`, wrapper único en `src/pwa/register-sw.ts`.
- Installability completa (manifest existente + SW activo en producción).
- Kill-switch `?sw=off` operativo y desregistro automático de SWs huérfanos en
  preview/dev/iframe Lovable.
- Observabilidad: eventos `pwa:registered`, `pwa:installed`, `pwa:updated`,
  `pwa:activated`, `pwa:controlling`, `pwa:error`, `pwa:skipped`, `pwa:unregistered`.

## Estrategias de caché (Cache Integrity + Security Before Offline)
- HTML navegacional: `NetworkFirst` (timeout 4s, TTL 24h) — nunca cache-first.
- Assets hasheados (`js/css/woff2`): `CacheFirst` (TTL 30 días).
- Imágenes estáticas same-origin: `CacheFirst` (TTL 30 días).
- **Denylist** de navigation fallback y runtime caching:
  `/~oauth`, `/api/`, `/auth`, `/cms`, `/portal`, `/admin`, `/cuenta`,
  `/concierge`, `/empresa`.
- No se cachea ninguna respuesta autenticada, token, sesión ni dato sensible.

## Governance 15.10.6 aplicada
- **Offline is Progressive**: el registro falla en silencio; la app sigue funcionando.
- **Cache Integrity**: HTML siempre `NetworkFirst`; nada crítico servido sólo desde caché.
- **Security Before Offline**: rutas autenticadas y APIs fuera del scope cacheable.
- **Update Safety**: `autoUpdate` + `skipWaiting:false` + `clientsClaim:true`;
  el usuario nunca queda atrapado entre versiones inconsistentes.
- **Observability First**: eventos `pwa:*` emitidos en cada transición.

## Compatibilidad
- Preview Lovable (`id-preview--*`, `*.lovableproject.com`, `beta.lovable.dev`):
  jamás registra SW — sólo limpia huérfanos.
- Discovery Layer: rutas públicas cacheadas como HTML `NetworkFirst`.
- Workspace Engine: rutas `/cms`, `/portal`, `/admin`, `/cuenta`, `/concierge`,
  `/empresa` excluidas del fallback y del caché.

## Rollback
1. En `vite.config.ts`, eliminar el plugin `VitePWA(...)` del array `vite.plugins`.
2. Sustituir el contenido de `src/pwa/register-sw.ts` por el wrapper inerte
   anterior (sólo `unregisterStale()`), o anteponer `?sw=off` para limpieza
   inmediata en sesiones activas.
3. `bun run build` y publicar; los clientes existentes desregistran al
   próximo render porque `register-sw.ts` siempre intenta limpiar huérfanos
   en contextos no autorizados.
