## 3. Server-Side Shadow Evaluator

### 3.1 Autorización (multi-factor)
`shadowAuthorization()` exige simultáneamente:

1. `process.env.MEDIA_SHADOW_INTERNAL_SECRET` definido (server-only).
2. Header `x-vmx-shadow` presente y comparado en **tiempo constante**
   (`timingSafeEqual`).
3. Host de la request **fuera** de `PRODUCTION_HOSTS`
   (`valladolidmx.lovable.app`, `www.quehacerenvalladolid.com`,
   `quehacerenvalladolid.com`).
4. `assetId ∈ SHADOW_ALLOWLIST` (hoy: 1 elemento — el asset del pilot v1.1).

Cualquier fallo devuelve `{authorized:false, reason}` sin emitir evento
(evita ruido y exfiltración por conteo).

### 3.2 Lógica de decisión
`evaluateMediaSourceShadow(asset, opts, ctx)`:

1. Autoriza (arriba).
2. Obtiene variantes `is_current=true` y `status='ready'` (fetcher
   inyectable en tests).
3. Filtra por `usage_context`, elige preferido AVIF → WebP → JPEG y ancho
   más cercano al target (`pickPreferredVariant`).
4. Si el chosen carece de `variant_key` → `fallback_reason='variant_key_missing'`.
5. Firma la URL con `supabaseAdmin.storage.createSignedUrl` con timeout duro
   de **2s** y TTL solicitado de 3600s. La firma se **descarta**
   inmediatamente (sólo se lee `length` para forzar materialización). Ante
   timeout → `storage_unreachable`; ante error → `signed_url_error`.
6. Emite un único evento `media_shadow_decision` sanitizado.

**En ningún caso** la firma se devuelve al cliente, se persiste en DB, se
loguea en el evento, ni se propaga al render público.

### 3.3 Evento sanitizado
Campos permitidos (test D los enumera exactamente):

```
kind, ts, env, asset_id, decision, variant_key, format_preferred,
width_chosen, fallback_reason, latency_ms, signed_url_latency_ms,
signed_url_ok
```

Prohibidos (verificado en test): `http:`, `https:`, `token=`, `cookie`,
`authorization`, `referer`, `supabase.co`, `signedUrl`, `bucket`, `path`.

## 4. Endpoint interno

`POST /api/dev/media-shadow-eval`

- Responde **404** si `NODE_ENV==='production'`.
- Responde **404** si `host` es un dominio productivo (defensa en profundidad).
- Responde **403** si `x-vmx-shadow` no coincide.
- Nunca devuelve URLs firmadas: sólo la decisión formal (Test F verifica el shape).
- Importa `client.server` dinámicamente dentro del handler para evitar
  contaminar el bundle cliente (regla `tanstack-supabase-import-graph`).

## 6. Prueba SSR byte-idéntica (test E, aproximación funcional)

No se realiza diff binario del HTML SSR completo (fuera de alcance M2.1).
En su lugar, el test E valida invariancia estructural del **único punto**
por donde el render obtiene fuentes: `resolveMediaSource()`. Los campos
opcionales nuevos permanecen `undefined` mientras no se activen desde
M2.2+. Ningún componente los lee todavía. Combinado con el hecho de que
el evaluador shadow vive en un módulo separado y sólo se invoca desde
`/api/dev/media-shadow-eval`, se garantiza que el HTML público es
byte-idéntico al de M1.
