## 1. Objetivo

Cerrar la continuidad de navegación iniciada en N1/N2. Persistir la cadena
territorial resuelta por el Context Engine en `sessionStorage` para que
las superficies inteligentes (Alux, Related, Arma tu Viaje, futuras) puedan
recuperar el contexto del visitante en:

- navegación interna,
- refresh del navegador,
- back/forward,
- apertura de deep-links en pestaña nueva.

Todo sin activar Fase 2 de 301, sin dependencia de rutas concretas y sin
impacto SEO.

## 2. Alcance implementado

| Módulo | Ruta | Rol |
|---|---|---|
| `session-context.ts` | `src/lib/navigation/session-context.ts` | Store SSR-safe con TTL 30 min, key versionada `vll:nav:session:v1`, snapshotFromContext, hook `useNavigationSession` con sincronización cross-tab (`storage`) e intra-tab (`vll:nav:session`). |
| `live-context.ts` | `src/lib/context-engine/live-context.ts` | Pub/sub in-memory: el `ContextEngineProvider` publica `ResolvedContext` en cada cambio de `canonical`. |
| `NavigationSessionBridge.tsx` | `src/components/navigation/NavigationSessionBridge.tsx` | Dos modos: local (dentro de un provider) y `GlobalNavigationSessionBridge` (suscriptor único al pub/sub). |
| `__root.tsx` | `src/routes/__root.tsx` | Monta `GlobalNavigationSessionBridge` una única vez para toda la app. |

No se creó lógica específica por ruta ni por superficie. Ningún módulo
compone URLs manualmente — el snapshot conserva únicamente `slug`,
`label` y `href` que ya emitió el Context Engine a través del Navigation
Contract v1.0.

## 3. Contrato N3

```
NavigationSessionSnapshot {
  v: 1,
  at: epoch_ms,
  canonical: string,        // path canónico del último contexto
  region?, destination?, category?, business?, product? : { slug, label, href? }
}
```

- **TTL:** 30 min. Snapshots expirados se descartan de forma perezosa.
- **Escritura:** sólo cuando `snapshotFromContext` produce un snapshot con
  ancla territorial (`destination` presente). Home, Blog, Contacto, etc.,
  no borran el último contexto — la continuidad sobrevive a rutas
  no-territoriales.
- **Idempotencia:** `writeNavigationSession` compara con el valor previo
  y omite writes redundantes.
- **Cross-tab / intra-tab:** cada tab escribe su propio snapshot al
  renderizar; el evento `storage` mantiene coherencia cross-tab y
  `vll:nav:session` permite reactividad sin polling en la misma tab.

## 6. Riesgos residuales

- **Snapshots en pestañas paralelas divergen momentáneamente** hasta que
  el evento `storage` reconcilia (comportamiento estándar del navegador).
  Aceptable — cada tab escribe su propio contexto al renderizar.
- **Consumidores aún no acoplados:** Alux, Related y Arma tu Viaje
  todavía no leen `useNavigationSession()`. Se aborda en olas siguientes
  al conectar cada superficie inteligente.
- **TTL fijo de 30 min:** suficiente para descubrimiento; revisar si
  aparecen sesiones más largas en analytics.

## 7. Decisiones consolidadas

- El Navigation Contract v1.0 sigue siendo la única fuente autorizada.
- El pub/sub `live-context` es la vía canónica para consumidores globales
  del Context Engine que no pueden colocarse dentro de un provider.
- `sessionStorage` es la única capa de persistencia autorizada para
  contexto territorial (no cookies, no localStorage, no history state).
- Fase 2 de 301 permanece deshabilitada.
