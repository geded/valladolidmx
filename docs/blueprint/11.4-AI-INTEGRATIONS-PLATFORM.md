# Valladolid.mx Platform Blueprint

# 11.4 – AI, INTEGRATIONS & PLATFORM
## Arquitectura de Alux, Integraciones y Servicios de Plataforma

**Versión:** 1.0
**Depende de:** 11.0, 11.1, 11.2 y 11.3

---

# Objetivo

Definir la arquitectura técnica para Alux, las integraciones externas y los servicios compartidos de la plataforma.

---

# Principios

- Alux es una capa transversal de inteligencia.
- Toda integración debe ser desacoplada.
- Las credenciales nunca se almacenan en el frontend.
- Cada proveedor debe poder sustituirse sin rediseñar el modelo.

# Storage

Todo archivo se almacena fuera de PostgreSQL.

Buckets sugeridos:

- destinations
- businesses
- products
- users
- reviews
- documents

---

# Logs

Registrar:

- llamadas a IA
- errores
- sincronizaciones
- webhooks
- eventos críticos

---

# Reglas

- Las API Keys permanecen únicamente en el backend.
- Los servicios deben ser reemplazables.
- Ninguna integración debe bloquear la plataforma completa.

---

# Criterios de aceptación

La plataforma debe permitir:

- cambiar proveedores IA
- activar o desactivar integraciones
- administrar configuración desde CMS
- soportar nuevas integraciones sin rediseño
