# 16.CV8.4 · Visitor Intelligence Segmentation — Blueprint v1.0

**Estado:** Autorizado por el Founder.
**Depende de:** CV8.0 (Contracts), CV8.1 (Ingesta append-only), CV8.2 (Projection), CV8.3 (Visitor Intelligence Center).
**Ruta:** `/cms/visitor-intel` — sección "CV8.4 · Segmentación".

---

## 1 · Behavioral Change Statement

- **Antes:** el equipo mira KPIs agregados y asume que todos los viajeros recorren el Journey igual.
- **Después:** ante cada sesión, el equipo identifica qué grupo (idioma, destino, capacidad) presenta diferencias significativas y qué decisión toma para acompañarlo mejor.
- **Cómo se comprueba:** cada segmento publicado declara Hallazgo → Transición → Decisión → Riesgo de sesgo.

## 2 · Founder Ethical Segmentation Principle

La segmentación no persigue personas. Comprende diferencias relevantes para mejorar producto, comunicación y atención. Datos mínimos, legítimos, explicables, útiles. Prohibido inferir atributos sensibles.

## 3 · Reglas obligatorias

- Consumir exclusivamente `visitor_intel.events` y contratos v1.0.0. Sin nueva fuente de verdad.
- Umbral mínimo `MIN_SEGMENT_POPULATION = 25` sujetos activos por segmento; menores → "Otros".
- Cero identidades individuales. Sólo agregados.
- Snapshots recomputados; nunca persistidos.
- Cada segmento responde 5 preguntas: diferencia · transición afectada · capacidad influyente · decisión concreta · riesgo de sesgo.

## 4 · Dimensiones v1.0

| Dimensión | Fuente | Estado |
|---|---|---|
| `locale` | `subject.locale` | ✅ operativa |
| `destination` | `context.destination_id` | ✅ operativa |
| `capability` | prefijo de `context.surface` | ✅ operativa |
| `country` | `context.country` (no en contrato) | ⏳ contract_pending |
| `channel` | `context.channel` (no en contrato) | ⏳ contract_pending |
| `device` | `context.device` (no en contrato) | ⏳ contract_pending |

Las dimensiones pendientes se activarán con evolución aditiva del contrato v1.x en CV8.1+.

## 5 · Contrato de salida

```ts
interface JourneySegmentSnapshot {
  dimension: SegmentDimension;
  window_days: 7 | 30 | 90;
  computed_at: string;
  status: "ok" | "contract_pending" | "empty";
  pending_reason?: string;
  baseline: { active_subjects: number; jpr: number };
  buckets: SegmentBucket[];   // active/progressed/jpr/Δ/intent/T1..T9
  others: SegmentBucket | null;
  min_population: 25;
  contract_version: "1.0.0";
}
```

## 6 · Matriz Segmento → Hallazgo → Transición → Decisión → Riesgo

La consola devuelve por bucket: JPR, Δ vs baseline, intención, T2/T5/T7. La lectura editorial de cada hallazgo es responsabilidad del operador (Regla de Accionabilidad).

## 7 · Fuera de alcance

- Alertas por umbral (CV8.5).
- Recomendaciones IA por segmento (CV8.7+).
- Persistencia de snapshots segmentados.
- Ampliación aditiva del contrato de contexto (país/canal/dispositivo): CV8.1.x posterior.
