# 15.10.4R — Matriz de Trazabilidad Blueprint ↔ Implementación

**Versión:** 1.2 (post Paso D · documento vivo)
**Última actualización:** 2026-06-30
**Mantenimiento:** se actualiza en cada ola y en cada cierre documental.

## Cambios v1.2 (Paso D — Cableado fino del sidebar)

- Sidebar `/admin` enlaza ahora `/cuenta/perfil` y un nuevo grupo
  **Vistas públicas** con `/`, `/alux`, `/marketplace`, `/marketplace/buscar`,
  `/arma-tu-viaje`, `/empresas`, `/experiencias`, `/hoteles`, `/restaurantes`,
  `/eventos`, `/oriente-maya`.
- Validación: 100% de enlaces apuntan a rutas existentes en `src/routes/`.
- Cobertura accesible desde `/admin`: **≈58% → ≈68%**.
- Sin server fns nuevas, sin RPCs nuevas, sin cambios de permisos.
- Detalle en `15.10.4R-STEP-D-REPORT-v1.0.md`.

## Cambios v1.1 (Paso C — Admin Composition Layer)

Se incorporan seis sub-rutas thin-shell del Panel Fundador como **capas de
composición pura** sobre dominios existentes (cero lógica nueva, cero server
functions nuevas, cero RPCs nuevas, un único componente nuevo: `AdminHub`):

| Sub-ruta | Archivo | Dominios consumidos |
| -------- | ------- | ------------------- |
| `/admin/empresas` | `routes/_authenticated/admin/empresas.tsx` | Portal · CMS · Pagos · UNC |
| `/admin/turistas` | `routes/_authenticated/admin/turistas.tsx` | Cuenta · Marketplace · Concierge · UNC |
| `/admin/concierge` | `routes/_authenticated/admin/concierge.tsx` | Concierge · Alux · UNC |
| `/admin/operaciones` | `routes/_authenticated/admin/operaciones.tsx` | Pagos · Marketplace · UNC · Observabilidad |
| `/admin/ia` | `routes/_authenticated/admin/ia.tsx` | Alux · UNC · Experience Builder |
| `/admin/sistema` | `routes/_authenticated/admin/sistema.tsx` | Observabilidad · UNC · CMS · Experience Builder |

Cobertura accesible desde `/admin`: **41% → ≈58%**.
Detalle completo en `15.10.4R-STEP-C-REPORT-v1.0.md`.

## Convenciones de estado

- ✅ **Implementado y accesible desde `/admin`** — capacidad construida y enlazada en el Panel Fundador.
- 🟡 **Implementado sin acceso desde `/admin`** — capacidad construida pero no enlazada en `/admin` (objetivo central de 15.10.4R).
- 🟠 **Parcial** — construido a medias (falta UI completa, falta CRUD, etc.).
- ⏳ **Pendiente** — referenciado en Blueprint pero aún no implementado.
- ⛔ **Fuera de alcance 15.10.4R** — pertenece a otra ola.
