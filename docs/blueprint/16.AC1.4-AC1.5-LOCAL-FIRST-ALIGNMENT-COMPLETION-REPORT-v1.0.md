## 1. Resultado

AC1 queda alineado bajo un único contrato operativo:

- antes del registro, favoritos, destinos e ítems viven exclusivamente en `AnonymousTravelDraft` del dispositivo;
- las interacciones anónimas no crean cuentas, filas DB, RPCs, Realtime ni llamadas de red;
- `guest-queue` deja de ser un modelo activo; sólo existe una lectura local de compatibilidad que lo convierte una vez y elimina su clave después de persistir el nuevo contrato;
- después de autenticar, `importAnonymousDraft` fusiona el contenido con el Travel Plan canónico;
- el contenido local sólo se elimina tras una respuesta exitosa de la importación;
- no se añadió migración, tabla, índice, función SQL ni secreto.

## 2. Implementación

| Área                | Resultado                                                                                                                                   |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| Store local         | Cache compartido entre superficies, suscripción reactiva, reconciliación IDB/localStorage por `updatedAt` y debounce global de 400 ms.      |
| Compatibilidad      | Conversión pura desde `travel_plan_guest_queue`; el módulo paralelo anterior fue retirado.                                                  |
| Superficies         | Favoritos, “Agregar a Mi Viaje”, preview público y Arma tu Viaje consumen el mismo draft.                                                   |
| Acceso persistente  | Con contenido local, el shell público muestra “Tu viaje” con contador, resumen, acceso a Arma tu Viaje y CTA explícito de registro.         |
| Registro progresivo | Sólo guardar permanentemente, otro dispositivo, compartir, recordatorios y límite duro abren el prompt AC1.                                 |
| Importación         | Server fn autenticada, RLS, targets deduplicados, notas con UUID determinista, favoritos por upsert y traza sin PII en `travel_plans.meta`. |
| Recuperación        | Error remoto conserva el draft y ofrece reintento; éxito limpia local e invalida Travel Plan/Favoritos.                                     |

## 3. Idempotencia y privacidad

- Targets: dedupe por `(plan_id, item_kind, target_id)`.
- Notas: identificador estable derivado de `draftId + sourceKey` para que un reintento no duplique.
- Favoritos: `upsert` aditivo; nunca se usa `toggle` durante importación.
- Plan: fechas y tamaño de grupo sólo completan campos todavía vacíos.
- Traza: máximo 50 referencias de importación con `draft_id` e `imported_at`, sin contenido, PII ni identidad anónima.
