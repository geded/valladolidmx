## 1. Qué se entregó

Bloque nuevo `vmx.experience.hero` (versión de contrato `1.0.0`),
registrado en `block-library.ts` y `composition-renderer.tsx`
(Studio + producción), disponible desde ya en
`/cms/experience-builder` para composición inmediata.

### 3 capas separadas y documentadas

| Capa            | Archivo | Responsabilidad |
|-----------------|---------|-----------------|
| **Presentación** | `src/components/experience-builder/blocks/experience-hero/ExperienceHero.tsx` | Componente React puro, sin fetching. 3 variantes (`immersive` / `compact` / `editorial`). Mobile-first, WCAG AA, focus visible, botones ≥ 44 px. |
| **Contenido**    | `src/lib/experience-builder/blocks/experience-hero/contract.ts` | Contratos Zod versionados (`contractVersion`, `capabilities`, `extensions[]`) para `config` y `DTO`. |
| **Comportamiento** | `src/components/experience-builder/blocks/experience-hero/ExperienceHeroBlock.tsx` | Parseo defensivo del `config`, hidratación desde `BusinessSurfaceContext` cuando `source === "business"`, fallback a preview neutral, hook para Protected Actions vía `onCtaAction`. |

### Registro en el Experience Builder

- Ficha `experienceHeroBlock` en `INITIAL_BLOCK_LIBRARY`.
- `STUDIO_PREVIEW_MAP["vmx.experience.hero"]` → `ExperienceHeroPreview`.
- `PRODUCTION_COMPONENT_MAP["vmx.experience.hero"]` → `ExperienceHeroBlock`.
- `constraints.unique_per_page = true`; sin `constraints.surfaces` (reutilizable en todas las plantillas).

### Demo funcional

Ruta interna `noindex,nofollow`:
`/lovable/experience-hero-preview` — muestra las 3 variantes con datos
realistas y confirma paridad visual entre Studio y producción.

## 3. Cumplimiento de la Regla de Compatibilidad Evolutiva

- `contractVersion` semver: `1.0.0`.
- Capacidades futuras opt-in dentro del mismo bloque via
  `config.capabilities`: `ratingBadge`, `liveStatus`, `video`,
  `immersive360`, `ar`. Todas defaultean a `false` y no exigen
  migración de composiciones existentes.
- Slot `extensions[]` reservado para módulos futuros (booking widget,
  contador live, sponsor stripe, etc.) sin cambiar el `blockType`.
- Set de iconos Lucide administrado dentro de la Presentación —
  ampliable sin tocar el schema.
- **No se creó** `experience.hero-pro`, `-v2` ni `-next`.

## 4. Checklist de admisión (6/6 verde)

1. ¿Reutilizable en al menos dos plantillas? **Sí** — sin
   `constraints.surfaces`. Válido en business, product, event,
   destination, region, landing, home.
2. ¿Aporta valor al visitante? **Sí** — jerarquía clara + acciones
   inmediatas + señales de confianza.
3. ¿Mejora el descubrimiento? **Sí** — eyebrow contextual, badges y
   meta chips que sitúan el destino.
4. ¿Mejora la conversión? **Sí** — CTA primario y secundario visibles
   desde el primer scroll, con `action` semántica para futuras
   integraciones de Protected Actions.
5. ¿Compatible con móviles? **Sí** — layout responsive, texto
   escalado, botones ≥ 44 px, imagen `object-cover` con overlay.
6. ¿Respeta Context Engine? **Sí** — no altera rutas, breadcrumbs ni
   provider; sólo lee `BusinessSurfaceContext` de manera opt-in.
