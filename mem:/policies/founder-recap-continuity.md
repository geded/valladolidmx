CV6.8 · Live Recap & Handoff. Reglas vinculantes:

1. Founder Continuity Principle — el recap nunca es el final del viaje; siempre construye un puente al siguiente momento (mañana, siguiente destino o post-viaje).
2. Recap as Derived View — el recap es vista completamente derivada. Prohibido crear modelo persistente, tabla, cache o estado paralelo para almacenar resúmenes.
3. Single Source Timeline Principle — todos los eventos provienen exclusivamente de fuentes oficiales existentes (Decision Center, Travel Plan, Concierge, Context Engine). El recap nunca se convierte en una línea de tiempo independiente.
4. Explainable Summary Principle — cada highlight, pendiente y recomendación declara `sources` y `rationale`. Prohibidas inferencias opacas.
5. No AI Memory Duplication Principle — CV6.8 no introduce memoria adicional para IA ni historiales paralelos. La continuidad reutiliza el Founder Memory Principle ya existente.

Alcance permitido: recap y handoff. Fuera de alcance (requiere nueva épica): notificaciones, automatizaciones, IA generativa, cronologías persistentes, asistentes predictivos, nuevas superficies.
