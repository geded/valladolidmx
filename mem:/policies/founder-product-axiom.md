**Todo nuevo módulo que se incorpore a ValladolidMX deberá responder primero a la pregunta:**

> "¿Cómo aporta valor al Travel Workspace del viajero?"

- Si una funcionalidad no fortalece el Travel Workspace, debe justificar explícitamente su existencia.
- El Travel Workspace es el centro funcional del producto y la referencia principal para la evolución del Destination Operating System.
- Filosofía oficial: ValladolidMX organiza viajes, no únicamente reservas. Ciclo: Inspiración → Planificación → Decisión → Reserva → Experiencia → Seguimiento.
- Consumer Journey oficial: Explora → Descubre → Guarda → Alux recomienda → Plan evoluciona → Concierge recibe → Propuesta → Aceptación → Pago → Viaje → Reseña → Regresa.
- Visión arquitectónica futura: Travel Workspace integrará progresivamente Travel Plan + Alux + Concierge + Drawer + Propuestas + Pagos + Timeline + Seguimiento. El Travel Plan sigue siendo la entidad canónica.
