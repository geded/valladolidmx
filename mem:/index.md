# Project Memory

## Core
Founder Experience First Rule (CV6, vinculante): toda capacidad Live Destination Companion debe responder "¿qué ayuda concreta recibe el viajero ahora?". Prioriza recomendaciones accionables sobre información pasiva; se oculta automáticamente cuando no aporta valor. Ver `mem://policies/founder-experience-first.md`.
Founder Decision Center Principle (CV6.2, vinculante): Now·Next·Later es Centro de Decisiones, no timeline. Jerarquía por tarjeta: acción → motivo → contexto → secundarias. Priorización automática por impacto experiencial. Auto-hide obligatorio. Ver `mem://policies/founder-decision-center.md`.
Founder Destination Context Engine Principle (CV6.4, vinculante): toda fuente de contexto del destino se integra como `DestinationContextContributor` publicando al Context Engine. Consumidores (Alux, Mi Viaje, Concierge, Decision Center) SÓLO leen del Context Engine. Prohibido consumir proveedores directo desde superficies. Ver `mem://policies/founder-destination-context-engine.md`.
Architectural Guardrail — Destination Context Contributors (CV6.4+, vinculante): Contributors independientes; sin dependencias cruzadas; sin lógica de negocio. Correlación/priorización/Explain centralizadas en Decision Center sobre `ResolvedDestinationContext`. Consumidores nunca acceden a Contributors individuales. Ver `mem://policies/founder-destination-contributors-guardrail.md`.
Founder Travel Passport Principle (CV7, vinculante): CV7 es el Travel Passport del viajero. Cada experiencia hace más inteligente el siguiente viaje. Reutiliza EXCLUSIVAMENTE Travel Plan como memoria canónica; prohibido historiales/perfiles/modelos paralelos. Todo Explainable, Backward Compatible, Reversible y Derivado. Ver `mem://policies/founder-travel-passport.md`.
Founder Travel Companion First Principle (2026-07-13, vinculante): Alux NUNCA pide permisos (ubicación, notificaciones, etc.) antes de demostrar valor. Cuatro etapas oficiales: Exploración · Planeación · Pre-viaje · En destino · Post-viaje. Cada etapa declara valor de hoy, motivo de retorno mañana y permisos permitidos. Ubicación se solicita sólo en Pre-viaje/En destino con beneficio evidente. Daily Companion obligatorio en toda épica CV6+. Ver `mem://policies/founder-travel-companion-first.md`.
Founder Travel Lifecycle Principle (2026-07-13, vinculante): ciclo oficial del viajero — Inspiración → Exploración → Planeación → Pre-viaje → En destino → Post-viaje → Inspiración. Post-viaje NO es final: abre el siguiente ciclo. Helper único `src/lib/traveler/journey-stage.ts`. Prohibidas taxonomías paralelas de fases/estados. Ver `mem://policies/founder-travel-lifecycle.md`.
Founder Daily Value Principle (2026-07-13, vinculante): cada etapa del Journey ofrece UNA misión diaria principal, derivada, que responde "¿Para qué abro Alux hoy?" antes de que el viajero pregunte. Toda nueva capacidad debe responder por escrito "¿Qué motivo tendrá el viajero para volver a abrir Alux mañana?". Fuente única `getDailyMission(stage)`. Ver `mem://policies/founder-daily-value.md`.
Founder Journey First Principle (2026-07-13, vinculante · cierre CV6.O1): la etapa del viaje es el contexto primario. Toda funcionalidad Carril A resuelve `TravelStage` antes de decidir información, prioridad, permisos, recomendaciones o CTA. Daily Mission Consistency: la misión diaria no puede ser desplazada sin justificación explícita en Blueprint. Restricción de evolución: reutilizar EXCLUSIVAMENTE `TravelStage`/`deriveTravelStage`/`getDailyMission`/`stageAllowsPermission` (contratos v1.0.0 congelados); prohibido motores/estados/lógica paralelos; evolución aditiva. Ver `mem://policies/founder-journey-first.md`.
