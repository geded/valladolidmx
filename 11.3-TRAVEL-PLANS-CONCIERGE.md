# Valladolid.mx Platform Blueprint

# 11.3 – TRAVEL PLANS & CONCIERGE
## Arquitectura de Arma tu Viaje y Concierge

**Versión:** 1.0
**Depende de:** 11.0, 11.1 y 11.2

---

# Objetivo

Definir la arquitectura funcional y de datos para el módulo **Arma tu Viaje** y el sistema de **Concierge Humano**.

La plataforma no funciona como un carrito de compras. El objetivo es construir un expediente inteligente del viaje que posteriormente será atendido por un concierge.

---

# Principios

- Arma tu Viaje es el núcleo funcional de la plataforma.
- El viajero puede planificar sin reservar inmediatamente.
- Alux ayuda a organizar el viaje, pero no sustituye al concierge.
- El concierge recibe expedientes completos para asesorar y cotizar.

---

# Flujo general

1. El visitante descubre destinos.
2. Guarda lugares, hoteles, restaurantes y experiencias.
3. Alux propone mejoras.
4. El usuario confirma su plan.
5. Se genera un expediente.
6. El expediente se asigna a un concierge.
7. El concierge continúa la atención por la plataforma, correo o WhatsApp.

---

# Tabla: travel_plans

Campos sugeridos:

- id
- user_id
- title
- destination_id
- travel_dates
- travelers
- status (draft, submitted, in_review, quoted, confirmed, cancelled)
- notes
- created_at
- updated_at

---

# Tabla: travel_plan_items

Cada elemento agregado al viaje.

Tipos permitidos:

- destino
- empresa
- producto
- evento
- ruta

Campos:

- id
- travel_plan_id
- entity_type
- entity_id
- day_number
- order_index
- notes

---

# Tabla: concierge_requests

Expediente enviado al concierge.

Campos:

- id
- travel_plan_id
- concierge_id
- priority
- status
- summary
- created_at
- assigned_at
- closed_at

---

# Tabla: concierge_notes

Notas internas del concierge.

Campos:

- id
- request_id
- author_id
- note
- visibility
- created_at

---

# Tabla: quotations

Cotizaciones asociadas a un expediente.

Campos:

- id
- request_id
- total_amount
- currency
- status
- valid_until
- created_at

---

# Estados del expediente

- Draft
- Submitted
- Assigned
- In Review
- Quoted
- Accepted
- Rejected
- Closed

---

# Reglas

- Un usuario puede tener múltiples planes.
- Un plan puede contener múltiples elementos.
- Un expediente pertenece a un solo plan.
- Un expediente puede generar varias cotizaciones.
- Todo cambio importante debe registrarse en auditoría.

---

# Integración con Alux

Alux puede:

- sugerir experiencias
- detectar huecos en el itinerario
- resumir el expediente
- preparar información para el concierge

Alux nunca confirma reservas ni modifica cotizaciones.

---

# Integración con empresas

Las empresas no ven el expediente completo.

Solo podrán recibir solicitudes cuando el concierge o el flujo autorizado las comparta.

---

# Criterios de aceptación

El módulo debe permitir:

- construir viajes paso a paso
- mantener historial
- generar expedientes
- facilitar el trabajo del concierge
- integrarse con Alux sin duplicar funciones
