# Filosofía

La base de datos representa el conocimiento estructurado del destino turístico.

No almacena únicamente información: almacena relaciones, contexto y reglas que permitirán a la plataforma evolucionar durante años.

---

# Principios arquitectónicos

## CMS First
Todo el contenido público debe provenir del CMS.

## Region First
Jerarquía mínima:

País → Estado → Región Turística → Destino → Categoría → Empresa → Producto

## AI Ready
Toda entidad relevante debe poder ser utilizada por Alux.

## Multilenguaje
Toda entidad pública debe admitir traducciones sin duplicar tablas.

## Escalabilidad
Agregar nuevas regiones, idiomas, productos o integraciones no debe requerir rediseñar el modelo principal.
